## What it is

After every commit, the hook automatically fires a deploy to Vercel. It supports:
- **Environment-specific strategies** (dev → staging → prod based on branch)
- Automatic **rollback on failure**

Hook event: `PostToolUse`.

## How to install

```bash
npx claude-code-templates@latest --hook automation/vercel-auto-deploy
```

Prerequisites:
- Vercel CLI installed and authenticated
- Project linked (`vercel link`)
- Tokens configured

For Claude:

```
I installed the automation/vercel-auto-deploy hook. Verify that I have
the vercel CLI authenticated. Make a commit and it should trigger a deploy
with my branch's strategy.
```

## What you can do with it

### Continuous deploy without touching the dashboard

Commit → deploy. No manual steps.

### Per-branch strategies

`main` → production. `develop` → staging. `feature/*` → preview.

### Automatic rollback

If the deploy fails healthchecks, roll back to the previous deployment. Fast mitigation.

### Combo with [Vercel Environment Sync](/boveda/hook-vercel-environment-sync)

Auto Deploy publishes. Env Sync keeps env vars consistent between local and Vercel.

### Combo with [Deployment Health Monitor](/boveda/hook-deployment-health-monitor)

Auto Deploy launches. Health Monitor watches post-deploy.

## When it's NOT a good fit

<Callout type="danger" title="Before auto-deploying to production">

- **Without CI tests first:** auto-deploy without tests = shipping broken code to prod. Combine it with [Run Tests After Changes](/boveda/hook-run-tests-after-changes).
- **For repos without Vercel:** the hook is Vercel-only. For Netlify, Railway, and others, look at alternatives.
- **If you don't want auto-deploy:** override whenever you need human review.
- **Broad-scope tokens:** a Vercel token with production access is serious. Restrict it.

</Callout>
