## What it is

Statusline + alert system for Vercel. When it detects deployment failures or build issues:
- **Statusline** shows the error state
- **Desktop notification** automatically
- **Tracking** persisted for analysis

Anti "the deploy failed 30 min ago and I'm only finding out now".


## What you can do with this

### Catch failures instantly

While you're coding, a deploy fails → desktop notification → you go fix it.

### For long sessions with auto-deploys

If you have auto-deploy + a productive session, this setting catches the problems without breaking your focus.

### Combo with [Vercel Deployment Monitor](/boveda/setting-vercel-deployment-monitor)

Deployment Monitor for normal status. Error Alert for anomalies. But they're alternatives (one statusline).

### Combo with [Deployment Health Monitor Hook](/boveda/hook-deployment-health-monitor)

The setting covers the statusline. The hook covers execution-time. A double net.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **Without Vercel:** doesn't apply.
- **If you disabled OS notifications:** the alert won't arrive.
- **For infrequent deploys:** less useful.

</Callout>
