## What it is

DeepGraph React analyzes your React project as a navigable graph. Unlike [DeepGraph Next.js](/boveda/mcp-deepgraph-nextjs), which is Next-specific, this MCP covers pure React projects: CRA, Vite, Remix, Astro with React, component libraries, and so on.

What it maps:
- **Component hierarchy** — who renders whom
- **State flows** — useState, useReducer, context propagation
- **Dependencies** — imports between components

The result: when Claude thinks about your codebase, it doesn't guess; it queries the graph.

## How to install

```bash
npx claude-code-templates@latest --mcp deepgraph/deepgraph-react
```

For Claude:

```
I installed the deepgraph/deepgraph-react MCP. Show me the graph of my
project: the component tree from the root and where the main state lives
(useState/useReducer/context).
```

## What you can do with it

### Component hierarchy map

"Draw me the tree from `<App>` down to the leaves. Identify the components with the most direct children." Handy for spotting monolith components worth breaking up.

### State flow analysis

"Where is the state defined that ends up affecting `<ProductCard>`? Is it prop-drilled or does it arrive via context?" Without this, you'd read files until you find it.

### Impact of a refactor

"If I change the signature of `useUser()`, which components need updating?" An exact list.

### Detect prop drilling

"Find me cases where a prop travels 4+ levels. Those are candidates for context or a global store."

### Dead code

"Components that are defined but nobody imports." A pre-release cleanup.

## Prerequisites

- **A React project** accessible to the MCP
- **Node.js + Claude Code** up and running

## When it's NOT a good fit

<Callout type="warning" title="Before you install">

- **If you use Next.js:** [DeepGraph Next.js](/boveda/mcp-deepgraph-nextjs) covers Server/Client Components specifically.
- **If your stack is Vue:** [DeepGraph Vue](/boveda/mcp-deepgraph-vue).
- **For deep TypeScript type analysis:** [DeepGraph TypeScript](/boveda/mcp-deepgraph-typescript) adds on top of React.
- **Small projects** (< 30 components): you don't need a graph, just read the files.

</Callout>
