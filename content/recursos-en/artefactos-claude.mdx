## What they are

When you ask Claude for something complex (a website, dashboard, game), instead of just replying with text, **it opens a panel on the right with something WORKING**.

The buttons work. The charts move. The forms submit. It's as if Claude built you **a mini-app in real time**.

| Feature | Detail |
|----------------|---------|
| Platform | `claude.ai` (web) |
| Installation | None |
| Plan required | Free works |
| Output type | Functional HTML/JS/React |

---

## What makes them special

### It's not an image, it's a real app

You click buttons, type into fields, **you really interact**.

### It updates live

If something isn't to your liking, you tell Claude and it adjusts on the spot without starting over.

### Zero installation

You don't download anything. Everything runs in your browser in the same Claude window.

## The superpower — Claude inside Claude

Here comes the **trick nobody talks about**. You can create Artifacts that **use the Claude API internally**.

That means: chatbots with real AI running in your browser, inside the Artifact.

### How it works

In your prompt, you ask Claude to use `window.claude.complete()` to call Claude from the Artifact.

### Example — custom chatbot

```
Create a therapist-style chatbot. Use `window.claude.complete()`
to respond with the real Claude.

Bot system prompt:
"You are an empathetic therapist. Listen without judging. Ask
open-ended questions. DO NOT give direct advice — guide the person
to find their own answer."

UI: classic chat, bubble messages, auto-scroll,
typing indicator while waiting for a response.
```

It opens a chatbot **that's functional with real AI**. No backend, no API key of your own, nothing.

## Anti-patterns

### 1. Expecting production-grade

Artifacts are for **prototypes / demos / internal tools**. For serious production, export the code and deploy it properly.

### 2. Asking for complex backend features

Artifacts live in the browser. No backend = no persistent DB, no real auth. For real auth → [mobile app stack](/boveda/stack-app-movil-ia) or [Claude Web Builder](/boveda/claude-web-builder).

### 3. Forgetting that the superpower chatbot uses your quota

Every call to `window.claude.complete()` **spends your plan**. Sharing your Artifact with lots of people = they all use your quota.

### 4. Not iterating visually

The first version is rarely perfect. **Iterate with feedback**: "shrink the padding", "change the main color to green", "add an animation on hover".

### 5. Asking for many features at once

Better: 1 feature → 2 features → 3 features. **Incremental**. If you ask for everything at once, Claude gets confused.

## How to share Artifacts

### Share the link

In the Artifact panel there's a "Share" button. It generates a public URL that anyone can open.

### Sharing limitations

- Whoever opens it sees the result
- They do NOT see the conversation with Claude
- If it uses the "superpower" (Claude inside Claude), it **consumes your quota**

### Embed

Some Artifacts can be embedded in other sites. Handy for portfolio showcases.

## When it's NOT a good fit

| Case | Why |
|------|-----|
| Serious production with many users | No backend, doesn't scale |
| App with real auth + persistent DB | For that you need a real backend |
| Pixel-perfect design for a premium client | Artifacts are good but not perfect |
| App with specific dependencies | Limited sandbox |
