## What it is

`CLAUDE_CODE_ENABLE_TELEMETRY=1` — Claude Code sends usage analytics to Anthropic.

For users who explicitly opt in. Anthropic uses that data to identify bugs, usage patterns, and feature improvements.

The opposite of [Disable Telemetry](/boveda/setting-disable-telemetry).

## How to install it

```bash
npx claude-code-templates@latest --setting telemetry/enable-telemetry
```

For Claude:

```
I installed the telemetry/enable-telemetry setting. Confirm that telemetry
is active and show me exactly what kind of data gets sent (high level).
```

## What you can do with it

### Contribute to improvements

Your usage pattern helps Anthropic prioritize features.

### Explicitness over defaults

If your policy is "opt in to everything explicitly," this setting documents that telemetry is intentionally active.

### For Anthropic API users with no privacy concerns

If your workflow has no privacy constraints, telemetry is a positive.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your workflow requires privacy:** use [Disable Telemetry](/boveda/setting-disable-telemetry).
- **If your org blocks data sharing:** this setting clashes with policy.
- **For highly sensitive data:** don't even enable it.

</Callout>
