## Qué es

Configura cómo Claude Code maneja telemetry:

- `CLAUDE_CODE_ENABLE_TELEMETRY=0` — desactiva default
- `OTEL_METRICS_EXPORTER=custom` — usa OTel exporter custom

Útil para:
- **Privacy:** desactivar telemetry totalmente
- **Enterprise observability:** mandar a tu Grafana/Datadog/Logfire propio
- **Compliance:** controlar dónde van los datos

## Qué podés hacer con esto

### Privacy first

`CLAUDE_CODE_ENABLE_TELEMETRY=0` evita que info de uso vaya a Anthropic.

### Observability enterprise

Mandás telemetry a tu OTel pipeline propio. Tu Dynatrace/Grafana/Logfire ve metrics de Claude Code junto al resto.

### Compliance regional

Algunos países requieren que metrics queden en data centers locales. Custom exporter lo permite.

### Combo con [Logfire MCP](/boveda/mcp-logfire)

Logfire MCP para consultas. Custom Telemetry para mandar metrics a Logfire. Combo natural.

### Combo con [Dynatrace MCP](/boveda/mcp-dynatrace)

Si tu stack es Dynatrace, mismo patrón.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para uso individual:** no necesitás observability seria.
- **Sin OTel pipeline configurado:** custom exporter necesita destino.
- **Para Anthropic feature improvements:** desactivar telemetry les complica ver patterns. Trade-off.
- **Solo 32 installs:** nicho enterprise. Validá implementación.

</Callout>
