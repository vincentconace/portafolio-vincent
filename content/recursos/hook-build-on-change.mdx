## Qué es

Cuando Claude edita un archivo source, este hook detecta el build tool de tu proyecto (npm, pnpm, yarn, vite, tsc, etc.) y corre el comando de build correspondiente automáticamente. Hook event: `PostToolUse`.

Útil para feedback rápido durante development asistido: editás → build corre → ves si rompiste algo sin tener que pedirlo.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook automation/build-on-change
```

Para Claude:

```
Instalé el hook automation/build-on-change. Probá: editá un archivo TS
y verificá que dispara el build. Si falla, mostrame los errores.
```

## Qué podés hacer con esto

### Feedback continuo de errores

Cada edit dispara build. Si Claude introduce un type error o syntax error, lo ves inmediatamente.

### Combo con type checking

Para proyectos TS estrictos, el build corre `tsc --noEmit`. Catch type issues en el momento, no en CI.

### Watch sin watch

Para proyectos sin watch mode armado, este hook simula uno: edit → build automático.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Builds lentos:** si tu build tarda 30+ segundos, cada edit traba la sesión. Mirá si conviene editar varios archivos antes de buildear.
- **Para edición masiva:** Claude edita 20 archivos en una pasada = 20 builds. Probablemente querés debouncing manual.
- **Para no-build languages:** Python interpretado, Bash, etc. — el hook no aporta.

</Callout>
