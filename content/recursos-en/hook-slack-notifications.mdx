## What it is

Minimal version of the Slack pipeline. When Claude or a subagent finishes, it sends a message to the channel. No context, no filters — just "done".

Hook events: `Stop` + `SubagentStop`.

Prerequisite: the `SLACK_WEBHOOK_URL` env var (created from Slack App settings → Incoming Webhooks).

## How to install it

```bash
npx claude-code-templates@latest --hook automation/slack-notifications
```

Configure it:

```bash
export SLACK_WEBHOOK_URL="https://hooks.slack.com/services/.../..."
```

## What you can do with it

### Remote alert when it finishes

When you're away from the editor, Slack lets you know.

### Subagent completions

If your workflow uses subagents, it notifies you for each one. Great for long orchestrations.

### Team activity channel

The team sees completions automatically in `#claude-runs`.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you don't use Slack:** obviously.
- **Spam:** frequent short sessions get overwhelming.
- **No webhook configured:** it won't work.
- **If you want more detail:** use the Detailed version.

</Callout>
