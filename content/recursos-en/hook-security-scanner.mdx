## What it is

**#4 in the entire catalog (1,365 installs).** After every file modification, it scans for:

- **Vulnerabilities** — insecure code patterns
- **Secrets** — hardcoded API keys (similar to [Secret Scanner](/boveda/hook-secret-scanner) but broader)
- **Other security smells**

It uses multiple security tools under the hood for broad coverage.

Hook event: `PostToolUse`.

## What you can do with it

### Defense in depth post-edit

Every one of Claude's modifications goes through the scan. If it introduces a security smell (SQL injection pattern, eval with unvalidated input, etc.), the hook warns you.

### Combo with [Secret Scanner](/boveda/hook-secret-scanner)

Secret Scanner pre-commit, Security Scanner post-edit. Redundant layers.

### Combo with [Dependency Checker](/boveda/hook-dependency-checker)

Dep Checker for supply chain (1,729 installs — the top one). Security Scanner for your own code. Complete coverage.

### Combo with [Codacy MCP](/boveda/mcp-codacy)

Codacy for deep platform analysis. Security Scanner for immediate feedback. They pair well.

### For autonomous sessions

A long `/goal`: Claude won't introduce vulns without you noticing.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **Latency:** a multi-tool scan can be slow. Every edit triggers it.
- **False positives:** some flagged patterns may be legitimate. Validate the findings.
- **No scanners installed:** the hook depends on external tools.
- **If you already have a security pipeline in CI:** running it locally too can be overhead.

</Callout>
