## What it is

You tell your computer: **"make me a 30s video announcing my product"** and it does it on its own.

**Editor Pro Max** is an open source tool that turns Claude Code into an AI video editing studio. Describe it in plain English → Claude writes Remotion code → you export a professional MP4 for any platform.

Repo: `Hainrixz/editor-pro-max`. Open source.

| Feature | Detail |
|----------------|---------|
| Processing | 100% local (nothing goes to the cloud) |
| Stack | Remotion 4 + FFmpeg + Whisper.cpp + React 19 |
| Components | 25+ |
| Templates | 9 (TikTok, Reels, YouTube, etc.) |
| Output platforms | TikTok, IG, YouTube, LinkedIn, Twitter, FB |
| Cost | Free |

---

## Prerequisites

- **Claude Code** installed
- **Node.js 20+**
- **FFmpeg** (optional, to process existing video)
- **Whisper.cpp** (optional, for automatic captions)

---

## Setup — 4 commands

```bash
# 1. Clone
git clone https://github.com/Hainrixz/editor-pro-max.git

# 2. Install
cd editor-pro-max && npm install

# 3. Open Claude
claude

# 4. Start preview
npm run dev
```

Open `http://localhost:3000` and you'll see the **Remotion Studio** with a live preview.

### FFmpeg and Whisper are optional

The core creation flow works without them. You only need them for:
- FFmpeg → processing existing video (cutting silences, jump cuts)
- Whisper → automatic captions

```

  Style: modern and dynamic.
  Include:
  - Animated text with the product name
```

## Anti-patterns

### 1. Vague prompts for video

❌ "Make me a nice video"
✅ "30s TikTok, Linear style, dark mode, 3 benefits with icons + CTA"

Specificity wins.

### 2. A single iteration

The first version is 70%. Iterate:
- "The text at second 5 comes in too fast, slow it down"
- "Change the CTA color to green"
- "Add a 0.5s pause before the logo"

### 3. Expecting pixel-perfect creative work

Editor Pro Max is **excellent for programmatic video**. For creative editing with lots of shots (filmed footage, subtle visual play), After Effects still wins.

### 4. Ignoring the preview

`localhost:3000` shows you the video **before rendering**. Rendering takes minutes. **Iterate in the preview**, and render only when you're ready.

### 5. Rendering at low quality to save time

If you're going to publish on YouTube/Reels, **render at maximum quality**. The extra seconds are worth it.
