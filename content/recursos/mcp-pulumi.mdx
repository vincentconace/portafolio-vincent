## Qué es

[Pulumi](https://www.pulumi.com) es la alternativa a Terraform que usa lenguajes reales (TypeScript, Python, Go, .NET, Java) en lugar de HCL. Mismo enfoque IaC: declarás recursos, Pulumi los aprovisiona y mantiene state.

Este MCP **oficial** de Pulumi habilita IaC development capabilities para agents. Te deja:
- Inspeccionar stacks y resources
- Preview cambios
- Aplicar updates
- Manejar state y backends

Útil para equipos que tomaron Pulumi en vez de Terraform y querés que Claude opere infra desde el editor.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/pulumi
```

Pre-requisitos:
- Pulumi CLI instalado
- Login a Pulumi Cloud (`pulumi login`) o backend self-hosted
- Stack(s) Pulumi en el proyecto

Para Claude:

```
Instalé el MCP devtools/pulumi. Listame los stacks del proyecto actual.
Para el stack `prod`, hacé preview de cambios sin aplicar y mostrame
qué recursos cambiarían.
```

## Qué podés hacer con esto

### Preview seguro de cambios

"Hacé `pulumi preview` del stack X y mostrame qué recursos van a crearse, modificarse y eliminarse. Para los `delete`, validá si son esperados". Patrón anti accidental destroy.

### Inspección de state

"Para el stack `production`, listame los resources actuales agrupados por tipo. Identificá los más críticos (databases, network)".

### Sync code ↔ infra

"En `main.ts`, agregué un S3 bucket nuevo. Hacé preview y validá que el plan matchea. Si OK, aplicalo a `dev` (no a `prod`)".

### Multi-stack ops

"Comparame los stacks `dev`, `staging` y `prod`. Identificá divergencias en config o resources que no deberían existir".

### Genera código IaC desde lenguaje natural

"Generame Pulumi code (TypeScript) que aprovisione: VPC con 3 AZs, subnet pública/privada, RDS Postgres, ALB con HTTPS desde ACM". Combinás generación + revisión.

## Pre-requisitos

- **Pulumi CLI** instalado y configurado
- **Login** a backend (Pulumi Cloud o self-hosted)
- **Cloud credentials** correctas (AWS, GCP, Azure según provider)
- **Proyecto Pulumi** existente

## Cuándo NO conviene

<Callout type="danger" title="Antes de aplicar a producción">

- **Operaciones destructivas:** `pulumi destroy`, `delete resource`. Catastrófico si sale mal. Hooks de aprobación SIEMPRE para prod.
- **Permisos cloud amplios:** las credentials cloud que usa Pulumi heredan todo. Restringí a IAM mínimo necesario.
- **Si usás Terraform:** este MCP es Pulumi-only. Para Terraform, mirá [Agent Skills HashiCorp](/boveda/plugin-agent-skills).
- **State conflicts:** si varios devs / agents operan el mismo stack en paralelo, podés corrupt state. Coordinen.
- **Para learning:** el MCP no enseña Pulumi. Si recién arrancás, leé los tutoriales primero.

</Callout>
