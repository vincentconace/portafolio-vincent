## Qué es

Cuando Claude intenta `git push` a `main` o `develop`, el hook bloquea. Enforce el patrón Git Flow: cambios entran via PR/MR, no via push directo.

Hook event: `PreToolUse` (Bash).

317 installs lo posicionan como protección sólida para teams que ya trabajan con branch protection.

```bash
npx claude-code-templates@latest --hook git/prevent-direct-push
```


## Qué podés hacer con esto

### Local enforcement de Git Flow

Antes de que tu push pegue contra branch protection en GitHub/GitLab, el hook ya lo paró local.

### Para teams que enforcement es importante

PR es mandatory en tu workflow. El hook actúa como recordatorio local.

### Sesiones autónomas

`/goal` largos: Claude no rompe el flow porque no puede pushear a main.

### Combo con [Force Push Blocker](/boveda/hook-force-push-blocker)

Prevent Direct Push para branches protegidas. Force Push Blocker para force pushes. Triple red.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si trabajás solo y mergeás directo a main:** el hook te bloquea.
- **Para hotfixes urgentes:** override necesario.
- **Si tu branch protection en server ya bloquea:** este hook duplica.
- **Si tus branches productivas son otras** (production, release/*): configurá las correctas.

</Callout>
