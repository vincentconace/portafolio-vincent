## Qué es

Es el marketplace oficial de plugins para Claude Code, mantenido directamente por Anthropic. 15k stars no son un accidente: es el lugar donde la propia empresa publica sus plugins de referencia y donde curan integraciones de terceros que pasaron su filtro de calidad.

Si recién instalaste Claude Code y querés saber por dónde empezar sin tener que evaluar marketplaces de comunidad, esta es la respuesta corta: instalá este primero, mirá qué hay, y después capás con cosas más específicas.

119 plugins en total. La división mental útil es: **plugins firmados por Anthropic** (~16+) y **integraciones de terceros verificadas** (~100+, marcadas como "external" o "community-managed").

## Cómo se instala

```bash
# 1) Marketplace
/plugin marketplace add anthropics/claude-plugins-official

# 2) Empezá con lo oficial Anthropic + un par de integraciones
/plugin install claude-code-setup@anthropics-claude-plugins-official
/plugin install commit-commands@anthropics-claude-plugins-official
/plugin install pr-review-toolkit@anthropics-claude-plugins-official
/plugin install hookify@anthropics-claude-plugins-official
```

O `/plugin` → Discover.

Para Claude:

```
Instalá el marketplace anthropics/claude-plugins-official. Listame solo los
plugins firmados por Anthropic. Después recomendame qué integraciones de
terceros me convienen según mi stack [tu stack].
```

## Qué podés hacer con esto

### Setup base de Claude Code en 10 minutos

`claude-code-setup` escanea tu repo, te recomienda hooks/skills/MCPs específicos para ese proyecto. `hookify` te deja crear los hooks desde markdown sin escribir código. `pr-review-toolkit` cubre la salida.

### Frontend grado producción sin aesthetic genérico

`frontend-design` está pensado específicamente para que el código que genera no se vea "hecho por IA". Es el plugin de referencia si te pasaste 6 meses peleando con UIs feas autogeneradas.

### Mantener `CLAUDE.md` vivo

`claude-md-management` audita tu `CLAUDE.md`, captura aprendizajes de sesión y los suma. La memoria del proyecto deja de envejecer sola.

### Conectar herramientas reales sin escribir MCPs

Asana, Notion, Linear, Slack, GitHub, Sentry, etc. — todos verificados por Anthropic. Esto evita el riesgo de instalar un MCP de comunidad que filtre tokens.

### Probar Stagehand para browser automation

[Stagehand](https://www.aitmpl.com/plugins/playwright-skill) es uno de los enfoques más prometedores en browser automation con LLMs. Está en el marketplace, listo para probar.

### LSPs oficiales (mejor que adivinanza)

12 LSPs firmados por Anthropic. Equivalente a [Claude Code LSPs](/boveda/plugin-claude-code-lsps) pero con la garantía de Anthropic detrás.

## Beneficios reales

- **Mantenido por la fuente** — Anthropic releases plugins primero acá.
- **Curación seria** — los terceros pasan filtros antes de aparecer listados.
- **119 plugins** — cubre lo más importante sin overlap caótico.
- **Updates frecuentes** — sale feature nueva en Claude Code, sale plugin que la aprovecha.
- **Es el "default" del ecosistema** — si una guía te dice "instalá X plugin", probablemente está acá.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **No hay "instalá todo":** son 119, varios solapan (`code-review` Anthropic vs CodeRabbit vs autofix-bot). Elegí.
- **Algunos plugins de terceros tienen su propio sub-marketplace** (`atomic-agents`, `superpowers`) — instalar este abre otra rabbit hole.
- **No tiene los plugins comunidad más experimentales** — para eso vas a `cc-marketplace`, `buildwithclaude`, o `jeremylongshore/claude-code-plugins-plus-skills`.
- **Si querés MCPs de Docker:** el marketplace oficial los tiene puntuales (chrome-devtools, playwright). Para el catálogo completo de 199+, [buildwithclaude](/boveda/plugin-buildwithclaude) tiene `mcp-servers-docker`.

</Callout>
