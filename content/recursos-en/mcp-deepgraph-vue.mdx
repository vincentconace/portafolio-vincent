## What it is

DeepGraph Vue is the vertical version built for Vue.js projects. The neat part is that it understands Vue-specific concepts: composition API, options API, refs and computed, watchers, reactive stores (Pinia, Vuex), and the relationship between the template and script of a `.vue` file.

When Claude asks "what triggers this watcher" or "which components consume this store", it queries the graph instead of iterating over files.

432 installs is respectable for a more niche category (Vue vs React/Next).


```
I installed the deepgraph/deepgraph-vue MCP. Map the graph of my Vue

```

## What you can do with it

### Component composition tree

"Give me the whole tree from `<App>` down to the leaves, distinguishing Composition API vs Options API." Mixed Vue is common, and seeing it as a graph helps.

### Reactive data flow

"Where does the `userPreferences` prop in `<ProfileSettings>` come from? Trace the chain back to the source." Useful when the flow runs through composables and stores.

### Pinia / Vuex stores

"List all the stores and which components consume them. Flag stores with a single consumer (candidates for local state)."

### Template ↔ Script

"In `<ProductCard>`, which refs from the script are used in the template? Are there any refs defined that aren't used?" Detects dead reactive state.

### Global watchers

"Find all the `watch` and `watchEffect`. Flag the ones that trigger changes across multiple components." A source of bugs in complex reactivity.

## Prerequisites

- **A Vue.js project** you can access
- **Node.js + Claude Code**

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If your stack is React:** [DeepGraph React](/boveda/mcp-deepgraph-react) or [Next.js](/boveda/mcp-deepgraph-nextjs).
- **For strict TypeScript analysis on Vue:** combine it with [DeepGraph TypeScript](/boveda/mcp-deepgraph-typescript).
- **Small Vue projects** (< 20 components): you don't need a graph.
- **If you use Nuxt:** most of the analysis applies, but some Nuxt concepts (server routes, middleware) aren't specifically covered.

</Callout>
