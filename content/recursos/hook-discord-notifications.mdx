## Qué es

Versión minimal de Discord notifications. Cuando Claude o un subagent termina, manda un mensaje al canal. Sin detalles, sin filtros de errores — solo "terminó".

Hook events: `Stop` + `SubagentStop`.

Pre-requisito: env var `DISCORD_WEBHOOK_URL` (creás el webhook desde Server Settings → Integrations → Webhooks).

## Cómo se instala

```bash
npx claude-code-templates@latest --hook automation/discord-notifications
```

Configurá el webhook URL:

```bash
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/.../..."
```

## Qué podés hacer con esto

### Aviso remoto cuando termina

Te avisa al canal Discord cuando Claude completa una tarea. Útil cuando no estás en el editor.

### Detección de subagent completions

Si tu workflow usa subagents, te avisa cuando cada uno termina. Para tracking de orquestaciones largas.

### Canal de actividad de equipo

Si varios devs corren Claude Code, todos los completions van al mismo canal. Visibility instant.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Discord:** obvio. Para Slack o Telegram hay otros patterns o [Claude Notifications Go](/boveda/plugin-claude-notifications-go).
- **Para sesiones muy frecuentes:** spam de notificaciones.
- **Sin webhook configurado:** no hace nada.
- **Si querés saber qué exactamente pasó:** usá la versión Detailed.

</Callout>
