## Qué es

Statusline simple con reloj. Muestra:
- **Model** name
- **Directory** del proyecto
- **Current time** (hora actual)

Útil para sesiones largas donde perdés noción del tiempo, o para trackear duration.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/time-statusline
```

Para Claude:

```
Instalé el setting statusline/time-statusline. Mostrame la hora
actual visible.
```

## Qué podés hacer con esto

### Anti deep-work-pierdes-tiempo

Si entrás flow y se te pasan las horas, el clock ayuda.

### Session duration tracking

Combinado con start time, calculás duración.

### Para meetings programados

Te recuerda check time antes de meeting.

### Combo con [Deadline Countdown](/boveda/setting-deadline-countdown)

Time muestra ahora. Deadline muestra hasta cuándo. Son alternativos (un statusline).

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu OS ya muestra reloj:** mac/Linux/Windows tienen reloj system. Redundante.
- **Si querés más info:** [Context Monitor](/boveda/setting-context-monitor) tiene clock + más.
- **Para focus deep:** ver tiempo puede romper flow.

</Callout>
