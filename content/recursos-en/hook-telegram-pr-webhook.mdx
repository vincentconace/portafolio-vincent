## What it is

When Claude creates a PR via `gh pr create` (GitHub CLI), this hook notifies Telegram with:
- URL of the created PR
- Associated Vercel preview URL (if the project is on Vercel)

Hook event: `PostToolUse`.

Useful for teams where Telegram is the channel and you want a quick review of the preview deploy before merging.

## How to install

```bash
npx claude-code-templates@latest --hook automation/telegram-pr-webhook
```

Prerequisites: TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, `gh` CLI configured, project on Vercel so the preview can be detected.

For Claude:

```
I installed the automation/telegram-pr-webhook hook. Configure the creds.
Ask Claude to create a PR with `gh pr create` — a notification should
arrive with both URLs.
```

## What you can do with it

### Quick cross-platform review

You get the PR + preview on Telegram → you open the preview on mobile → you decide on the merge.

### Distributed team on Telegram

A notification with links lets anyone on the team open the preview without hunting for it.

### For PR-heavy workflows

If you push several PRs a day, automating the notify step saves friction.

### Combo with [Vercel Auto Deploy](/boveda/hook-vercel-auto-deploy)

Vercel auto-deploys. This hook lets you know when the preview is ready.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you don't use GitHub CLI or Vercel:** the hook needs both.
- **For repos without previews:** the hook works but delivers less value.
- **If your Telegram channel is noisy:** adding PR notifications can saturate it.
- **Only 69 installs:** validate the implementation in a sandbox.

</Callout>
