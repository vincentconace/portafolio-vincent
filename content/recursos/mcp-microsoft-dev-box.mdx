## Qué es

Microsoft Dev Box son entornos de desarrollo cloud-hosted (Windows VMs preconfiguradas) que muchas empresas usan en lugar de máquinas físicas para devs. La sacada: nuevo developer se onboardea en minutos con un entorno listo en vez de pasar 2 días configurando local.

Este MCP da operación natural language sobre Dev Boxes: crear, suspender, configurar, manejar pools. Útil para admins de Dev Box y para devs que querés operar tu box sin abrir el Azure portal.

## Qué podés hacer con esto

### Manejo de Dev Boxes personales

"Suspendéme mi Dev Box principal (no lo voy a usar el finde) y restaurémelo el lunes a las 9 am". Operaciones bursátiles para ahorrar.

### Onboarding asistido

"Aprovisioná un Dev Box nuevo para [nombre], con el pool `frontend-team`. Una vez listo, mandame el RDP/URL para que se conecte". Operación admin.

### Audit de uso y costos

"Listame Dev Boxes inactivos (sin uso por más de 14 días). Identificá el costo estimado de cada uno y candidate to delete".

### Gestión de pools

"Para el pool `data-team`, ¿cuál es la utilización actual? ¿Conviene escalar el cap up o down?".

### Configurations

"Crear una nueva configuration para devs Java: incluyendo IntelliJ, Maven, JDK 17 y Docker. Aplícala a un nuevo pool `java-team`".

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Dev Box:** obvio. Para devboxes en otro cloud (GitHub Codespaces, Gitpod, Replit), MCPs distintos o nada.
- **Operaciones destructivas:** delete Dev Box, delete pool. Hooks de aprobación.
- **Permisos amplios:** Dev Box Admin tiene poder serio. Si tu role permite delete de pools enteros, Claude hereda eso.
- **Para devs casuales:** el portal Azure puede ser más cómodo. El MCP brilla en admin tasks o cuando combinás con automatización.
- **Costos:** Dev Box no es gratis. Operaciones que aprovisionen recursos generan facturación.

</Callout>
