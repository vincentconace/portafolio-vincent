## What it is

`forceLoginMethod: console` — Claude Code only allows login with Anthropic Console accounts (API billing). Claude.ai accounts (Pro/Team) are rejected.

The opposite of [Force Claude.ai Login](/boveda/setting-force-claudeai-login). Use this if your org centralizes on API billing instead of Claude.ai plans.

## How to install it

```bash
npx claude-code-templates@latest --setting authentication/force-console-login
```

For Claude:

```
I installed the authentication/force-console-login setting. Confirm that
I'm authenticated with a Console API key and not with a Claude.ai account.
```

## What you can do with it

### Centralize on API billing

Your org has an Anthropic API contract with centralized billing. This setting enforces it.

### For developers with dedicated API access

If each dev has an org API key instead of a personal Claude.ai one, this setting avoids confusion.

### For integration with existing tooling

If your org uses the API directly for other things (internal apps, etc.), centralizing on Console makes sense.

### The opposite of [Force Claude.ai Login](/boveda/setting-force-claudeai-login)

These two are opposites. Only install one.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you use Claude.ai Pro/Team:** this setting blocks you.
- **Only 40 installs:** low adoption. Most people prefer Claude.ai.
- **For devs with a personal Pro account:** no override possible.

</Callout>
