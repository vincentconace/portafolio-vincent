## Qué es

**#3 de statuslines (1,545 installs).** Statusline funcional que muestra:
- **Model** name actual
- **Directory** del proyecto
- **Git branch** activa
- **Change indicators** (count de uncommitted changes)

Sin gimmick. Info útil para devs con workflow git intenso.

## Qué podés hacer con esto

### Git context constante

Saber en qué branch estás y si tenés uncommitted sin tener que `git status` cada vez.

### Anti "ay, estoy en main"

Cuando ves "main" en el statusline, frena de editar antes de hacer un mess.

### Combo con [Validate Branch Name](/boveda/hook-validate-branch-name) o [Prevent Direct Push](/boveda/hook-prevent-direct-push)

Statusline + hooks de protección git = workflow git seguro y visible.

### Combo con [Git Flow Settings](/boveda/setting-git-flow-settings)

Git Flow Settings cubre permissions + hooks + statusline para Git Flow completo. Comparable a este pero más opinionado.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás git:** no aporta.
- **Si querés más info** ([Context Monitor](/boveda/setting-context-monitor) cubre context + cost también).
- **Si tu terminal es muy estrecho:** el statusline puede no caber.

</Callout>
