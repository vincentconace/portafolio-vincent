## Qué es

Es una de las **tres** variantes de Playwright en el catálogo browser_automation:
- [playwright-mcp-server](/boveda/mcp-playwright-mcp-server) (4,419 installs — top) — `executeautomation`
- **playwright-mcp** (este, 1,393 installs) — variante alternativa
- [mcp-server-playwright](/boveda/mcp-mcp-server-playwright) (675 installs) — `automatalabs`

Tres implementaciones distintas del mismo concepto (envolver Playwright como MCP). Compará en el JSON cuál te conviene; para la mayoría de casos, el más adoptado (`playwright-mcp-server`) es la opción default.

## Qué podés hacer con esto

Las mismas operaciones de browser automation con Playwright que las otras variantes:

- Navegar URLs
- Click, type, scroll
- Tomar screenshots
- Ejecutar JS en el browser
- Inspeccionar DOM
- Testing E2E ad-hoc

Para casos de uso concretos, mirá [Playwright MCP Server](/boveda/mcp-playwright-mcp-server) (la variante más adoptada) — la mayoría se transfieren.

## Cuándo elegir cuál

| Caso | Recomendación |
| --- | --- |
| No tenés preferencia | [playwright-mcp-server](/boveda/mcp-playwright-mcp-server) (más adoptado) |
| Querés probar varias y comparar | Probá una en sandbox antes de comprometerte |
| Stack enterprise crítico | Mirá los JSONs y la implementación upstream |

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si ya tenés otro Playwright MCP:** no los instales juntos (colisión de tools).
- **Comunidad reducida vs el top:** el `playwright-mcp-server` tiene 3x más adoption. Más probabilidad de bugs conocidos resueltos.
- **Para sites con anti-bot:** mejor [Browserbase](/boveda/mcp-mcp-server-browserbase) o [Bright Data](/boveda/mcp-brightdata).

</Callout>
