## Qué es

**1,840 installs — uno de los settings más adoptados.** Razón: muchísima gente no quiere el `Co-Authored-By: Claude <noreply@anthropic.com>` que se agrega automáticamente a cada commit.

Setting key: `includeCoAuthoredBy: false`. Resultado: commits limpios sin co-authorship line.

```bash
npx claude-code-templates@latest --setting global/git-commit-settings
```

Para Claude:

```
Instalé el setting global/git-commit-settings. Hacé un commit de prueba
y verificá que NO incluye la línea Co-Authored-By Claude.
```

## Qué podés hacer con esto

### Git history limpio

Si tu equipo prefiere commits sin metadata del modelo, este setting lo logra.

### Compliance / policy de org

Algunas orgs requieren que commits no mencionen tools externos. Setting enforce.

### Para repos públicos OSS

En proyectos OSS donde author attribution matters, podés preferir solo tu nombre.

### Si te molesta visualmente

Para algunos devs, la línea extra ensucia el commit. Estética matter.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu equipo valora transparency sobre AI usage:** Co-Authored-By señala dónde se usó Claude. Removerlo oculta esa info.
- **Para compliance que requiere AI disclosure:** algunos contracts/regulations piden señalar uso de AI en code. Removerlo puede ir contra eso.
- **Para audit:** la línea ayuda a tracking de qué fue generado por AI.

</Callout>
