## What it is

A statusline that shows:
- **Current git branch**
- **Files changed** count
- **Countdown timer** to your deadline (configured with the `DEADLINE_TIME` env var)

Color-coded urgency: green when far from the deadline, yellow when close, red when critical.

Useful for sprints with a hard deadline or demos at a fixed time.

## How to install

```bash
npx claude-code-templates@latest --setting statusline/deadline-countdown
```

Configure it:

```bash
export DEADLINE_TIME="2026-12-31T23:59:59"
```

For Claude:

```
I installed the statusline/deadline-countdown setting. Set the deadline
to the end of the month. Show me the current countdown and the color thresholds.
```

## What you can do with it

### Visual time pressure

If you have a real deadline, the statusline reminds you constantly.

### For demos at a fixed time

"Demo at 4pm" — the countdown tells you exactly how many hours are left.

### Sprint endings

Last day of the sprint: the countdown shows you the remaining hours.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **No real deadlines:** the countdown is irrelevant.
- **If urgency stresses you more than it motivates you:** avoid it.
- **For casual use:** the setting is niche.
- **Only 16 installs:** very niche. Validate the implementation.
- **For deep creative work:** the constant clock can be disruptive.

</Callout>
