## Qué es

Versión zero-config del [Vercel Deployment Monitor](/boveda/setting-vercel-deployment-monitor). En vez de pedirte que setees env vars, detecta automáticamente el Vercel auth token desde el CLI config local (`~/Library/...` en macOS).

Plug-and-play: si `vercel` CLI está autenticado, este setting funciona instantáneamente.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/zero-config-deployment-monitor
```

Pre-requisito: `vercel` CLI instalado y autenticado (`vercel login`).

Para Claude:

```
Instalé el setting statusline/zero-config-deployment-monitor. Verificá
que detecta mi Vercel auth automaticamente y mostrame el deploy status.
```

## Qué podés hacer con esto

### Setup más simple

Si ya usás Vercel CLI, no tenés que configurar nada.

### Para devs que rotan máquinas

Cada máquina con `vercel login` activo funciona out of the box.

### Combo con [Vercel CLI workflows](/boveda/hook-vercel-auto-deploy)

Si ya tenés flow Vercel CLI completo, este setting agrega awareness without overhead.

### Anti vs [Vercel Deployment Monitor](/boveda/setting-vercel-deployment-monitor)

Deployment Monitor pide env vars. Zero Config auto-detecta. Mismo output, setup distinto.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin `vercel` CLI auth:** el auto-detect falla.
- **Para CI sin CLI config:** [Vercel Deployment Monitor](/boveda/setting-vercel-deployment-monitor) con env vars explícitas.
- **Si querés multi-env:** [Vercel Multi Env Status](/boveda/setting-vercel-multi-env-status).
- **Para focus en errors:** [Vercel Error Alert System](/boveda/setting-vercel-error-alert-system).

</Callout>
