## Qué es

**10,792 installs — el #2 absoluto del catálogo entero.** Solo lo supera [Simple Notifications Hook](/boveda/hook-simple-notifications) (6,810... espera — eso es 6.8k, este tiene 10.8k. **Este es el #1**).

Real-time monitor del context window de Claude Code con:
- **Progress bars** visual del context usage
- **Color-coded alerts** cuando se acerca al límite
- **Session analytics**: cost, duración, lines changed
- **Auto-compaction** awareness

Es decir: el statusline serio que casi todo el mundo quiere. Por eso lidera el catálogo entero.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/context-monitor
```

Pre-requisito: `python3` disponible.

Para Claude:

```
Instalé el setting statusline/context-monitor. Mostrame el statusline
en vivo: context usage actual, cost de la sesión y cuándo se va a
auto-compactar.
```

## Qué podés hacer con esto

### Anti "Claude se quedó sin contexto"

Ves el progress bar y sabés cuándo cortás o pedís un resumen antes de que rompa.

### Cost tracking en vivo

Para quienes paguen por tokens, ver el cost acumulado de la sesión es crítico.

### Decisiones informadas sobre /compact

Cuando el context llega a 80%, sabés que conviene /compact.

### Comparado con [Claude HUD Plugin](/boveda/plugin-claude-hud)

HUD es plugin completo con más capabilities. Context Monitor es setting native, más simple, más adoptado. Compará.

### Combo con [Context Timeline Hook](/boveda/hook-context-timeline)

Context Monitor en statusline + Timeline en browser separado = visibility total.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin python3:** verificá `python3 --version`.
- **Si querés statusline minimal:** [Colorful Statusline](/boveda/setting-colorful-statusline) es más liviano.
- **Si ya usás [Claude HUD](/boveda/plugin-claude-hud):** plugin overlap.

</Callout>
