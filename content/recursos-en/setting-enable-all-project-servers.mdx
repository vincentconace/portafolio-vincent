## What it is

`enableAllProjectMcpServers: true` — every MCP server defined in the project's `.mcp.json` is activated without a manual prompt.

Useful for teams that define MCPs as part of the repo (similar to `package.json` for libraries): when someone clones it, the MCPs are already active.

## What you can do with it

### Simplified team onboarding

Dev clones repo → MCPs active with no manual steps.

### Environment reproducibility

Whatever is in `.mcp.json` is the source of truth. Every dev has the same MCP setup.

### Combo with [Disable Risky Servers](/boveda/setting-disable-risky-servers)

Enable All for the default + Disable Risky for specific exceptions. Allowlist + blacklist.

### For CI / sandboxes

CI running Claude Code on a checked-out repo: this setting avoids getting stuck on approval prompts.

## When it's NOT a good idea

<Callout type="danger" title="Before enabling">

- **Trust in .mcp.json:** auto-approve means that if someone adds a malicious MCP to the repo's `.mcp.json`, it activates without you seeing it. Review PRs that touch that file.
- **For projects on untrusted networks:** don't auto-approve MCPs in shared repos without review.
- **For conservative use:** manual approval is safer.
- **Combo with review process:** if your team doesn't do serious PR review, this setting can be an attack vector.

</Callout>
