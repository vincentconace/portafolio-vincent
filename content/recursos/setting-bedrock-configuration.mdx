## Qué es

Setting que apunta Claude Code a usar **Amazon Bedrock** como provider del modelo, en vez del API direct de Anthropic. Útil cuando:

- Tu organización ya tiene AWS billing centralizado
- Necesitás compliance específico de AWS (HIPAA BAA, FedRAMP, etc.)
- Data residency en regiones AWS específicas
- Auditoría unificada con resto de servicios AWS

Configura `CLAUDE_CODE_USE_BEDROCK=1` y `AWS_BEARER_TOKEN_BEDROCK` con tu API key de Bedrock.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting api/bedrock-configuration
```

Después seteás el token real en tu env.

Para Claude:

```
Instalé el setting api/bedrock-configuration. Mostrame qué env vars
necesito setear y la URL endpoint de Bedrock para mi región.
```

## Qué podés hacer con esto

### Centralizar billing en AWS

Tu org ya tiene AWS contract. Bedrock factura ahí, no separado a Anthropic.

### Compliance enterprise

Si necesitás BAA HIPAA con AWS, Bedrock lo permite. API direct de Anthropic puede no aplicar.

### Data residency específica

Bedrock corre en regions AWS específicas. Si necesitás us-east-1 para compliance, tenés control.

### Combo con [AWS Credentials Setting](/boveda/setting-aws-credentials)

AWS Credentials para SSO/credential refresh. Bedrock Configuration para apuntar al provider correcto. Combo enterprise natural.

## Cuándo NO conviene

<Callout type="warning" title="Antes de habilitar">

- **Si no tenés AWS account ni Bedrock activado:** pasos previos necesarios.
- **Para uso individual:** Bedrock puede ser más caro que API direct de Anthropic.
- **Si tu region AWS no tiene Bedrock:** verificá disponibilidad.
- **Models pueden diferir:** Bedrock no siempre tiene la última versión de Claude. Validá lo que necesitás.

</Callout>
