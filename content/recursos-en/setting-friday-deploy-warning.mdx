## What it is

A decorative setting that replaces the spinner verbs (the animated text Claude Code shows while it's processing) with the classic "Don't Deploy On Fridays" reminder.

It's a humorous nod to industry folklore: never deploy to production on a Friday afternoon, because if something breaks, you'll ruin your weekend.

## Summary table

| Field | Value |
| --- | --- |
| aitmpl category | `environment` |
| Installs | 32 |
| Setting key | `spinnerVerbs` (mode: replace) |

## How to install it

```bash
npx claude-code-templates@latest --setting environment/friday-deploy-warning
```

For Claude:

```
I installed the environment/friday-deploy-warning setting. Run a task
and show me what appears in the spinner.
```

## What you can do with it

### Team culture

If your team respects the rule, this setting is a knowing wink.

### Anti-Friday impulse

At 5pm on Friday someone suggests "let's just quickly do this deploy." The spinner reminds you that Monday is better.

### To combo with [Vercel Auto Deploy](/boveda/hook-vercel-auto-deploy)

If you have auto-deploy active, this reminder keeps awareness.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you work in a timezone where Friday isn't "the day":** the culture is US-centric. In other countries the pattern may be different.
- **If your CI/CD is serious:** a good pipeline with feature flags + automatic rollback lets you deploy any day.
- **For individual use without a team:** less fun.
- **Only 32 installs:** very niche.

</Callout>
