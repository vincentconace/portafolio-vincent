## What it is

When Claude tries to `git push` to `main` or `develop`, the hook blocks it. It enforces the Git Flow pattern: changes come in via PR/MR, not via direct push.

Hook event: `PreToolUse` (Bash).

With 317 installs, it stands as solid protection for teams that already work with branch protection.

npx claude-code-templates@latest --hook git/prevent-direct-push
```

```

## What you can do with it

### Local enforcement of Git Flow

Before your push hits branch protection on GitHub/GitLab, the hook has already stopped it locally.

### For teams where enforcement matters

PRs are mandatory in your workflow. The hook acts as a local reminder.

### Autonomous sessions

Long `/goal` runs: Claude doesn't break the flow because it can't push to main.

### Combo with [Force Push Blocker](/boveda/hook-force-push-blocker)

Prevent Direct Push for protected branches. Force Push Blocker for force pushes. A triple safety net.

## When it's NOT a good fit

<Callout type="warning" title="Before you install">

- **If you work solo and merge directly to main:** the hook blocks you.
- **For urgent hotfixes:** you'll need an override.
- **If your branch protection on the server already blocks it:** this hook duplicates it.
- **If your production branches are different ones** (production, release/*): configure the right ones.

</Callout>
