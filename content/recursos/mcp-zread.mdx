## Qué es

zread es una capa adicional del ecosystem Z.AI específicamente para lectura web optimizada por LLMs. Powered by [zread.ai](https://zread.ai), provee output limpio listo para que Claude (u otros clientes MCP-compatibles) procesen contenido.

Forma parte del stack Z.AI MCPs ([Vision](/boveda/mcp-zai-mcp-server), [Web Reader](/boveda/mcp-web-reader), [Web Search Prime](/boveda/mcp-web-search-prime)). Si tu setup ya usa Z.AI para otros servicios, mantener consolidado simplifica.

## Qué podés hacer con esto

### Web reading optimizado para LLM

URL → output limpio. Mismo concepto que [Web Reader](/boveda/mcp-web-reader) y [Firecrawl](/boveda/mcp-firecrawl).

### Diferencia con Web Reader

Ambos son de Z.AI pero pueden tener APIs/output ligeramente distintos. Mirá los JSONs de configuración para saber cuál te conviene en tu caso específico.

### Casos típicos

- Convertir URLs a contenido procesable
- Anti-hallucination en research
- Lectura recurrente automatizada

## Pre-requisitos

- **Cuenta Z.AI / zread.ai** activa
- **API key**
- **Conexión a internet**

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si ya usás [Web Reader](/boveda/mcp-web-reader) del mismo provider:** alta probabilidad de overlap. Elegí uno.
- **Si ya tenés [Firecrawl](/boveda/mcp-firecrawl) o solution Western:** compará valor antes de sumar.
- **Compliance:** servicio chino. Validá tu policy.
- **Solo 67 installs:** comunidad probando reducida.
- **Para data sensible:** el contenido pasa a server Z.AI.

</Callout>
