## What it is

Statusline focused on Git Flow. It shows:
- **Branch type icon** based on prefix:
  - 🌿 feature/*
  - 🚀 release/*
  - 🔥 hotfix/*
- **Sync status** with the remote
- **Change indicators** (uncommitted)

It's the visual version of [Git Flow Settings](/boveda/setting-git-flow-settings) (the all-in-one bundle). If you only want the statusline without permissions/hooks, use this one.

```

```

## What you can do with it

### Quick branch type identification

Without reading the full name, the icon tells you the branch type.

### For teams with many active branches

If you're working on 5 features in parallel, the icons help with quick context.

### Combo with [Git Flow Settings](/boveda/setting-git-flow-settings)

If you want the complete bundle (which includes this statusline), use [Git Flow Settings](/boveda/setting-git-flow-settings).

### Combo with [Validate Branch Name](/boveda/hook-validate-branch-name)

The hook enforces naming. The statusline shows the resulting icon. A natural combo.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you don't use Git Flow:** other workflows don't have the prefixes.
- **If your terminal doesn't show emojis:** the icons appear as weird ?? characters.
- **If your team uses a different convention:** it won't match.

</Callout>
