## Qué es

Statusline que muestra:
- **Git branch** actual
- **Files changed** count
- **Countdown timer** a tu deadline (configurado con `DEADLINE_TIME` env var)

Color-coded urgency: verde lejos del deadline, amarillo cerca, rojo crítico.

Útil para sprints con deadline duro o demos en horario fijo.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/deadline-countdown
```

Configurá:

```bash
export DEADLINE_TIME="2026-12-31T23:59:59"
```

Para Claude:

```
Instalé el setting statusline/deadline-countdown. Configurá deadline
para fin de mes. Mostrame el countdown actual y los colors thresholds.
```

## Qué podés hacer con esto

### Presión visual de tiempo

Si tenés deadline real, el statusline te recuerda constante.

### Para demos en horario fijo

"Demo a las 4pm" — el countdown te dice cuántas horas quedan exactamente.

### Sprint endings

Último día del sprint: countdown te muestra horas restantes.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin deadlines reales:** countdown irrelevante.
- **Si la urgencia te estresa más que motiva:** evítalo.
- **Para uso casual:** el setting es nicho.
- **Solo 16 installs:** muy nicho. Validá implementación.
- **Para trabajo creativo profundo:** el reloj constante puede ser disruptivo.

</Callout>
