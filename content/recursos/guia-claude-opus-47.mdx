## El problema

Te emocionás con el nuevo **Opus 4.7**. Lo activás, escribís 3-4 prompts, y aparece el cartelito:

> "Tu uso está limitado. Regresá en 5 horas."

Tu día, perdido.

¿Por qué pasa? Anthropic configuró Opus 4.7 para que **piense más** por default. Cada prompt te come **el doble** de tu plan. Vos no te das cuenta hasta que ya salió el cartelito.

Esta guía es el mapa para que **no te pase**.

## 6 reglas de prompting (de la doc oficial de Anthropic)

Aplican igual para Sonnet 4.6, Opus 4.6 y Opus 4.7. Si escribís prompts así, te ahorrás la mitad del plan.

### 1. Sé claro y directo

Pensalo como hablarle a un empleado **brillante pero sin contexto** de tu negocio. Mientras más específico, mejor.

❌ "Hacéme un dashboard"
✅ "Hacéme un dashboard de ventas con filtro por mes, gráfica de línea arriba y tabla de productos abajo"

### 2. Dale el porqué

❌ "No uses puntos suspensivos"
✅ "No uses puntos suspensivos porque se va a leer en voz alta y los puntos suspensivos no se pronuncian bien"

Claude **generaliza del porqué** y aplica la regla aun en casos que no pensaste.

### 3. Dale ejemplos

2-3 ejemplos bien hechos valen más que 1000 palabras de instrucciones. Envolvelos en etiquetas:

```
<ejemplo>
Input: ...
Output: ...
</ejemplo>
```

Claude distingue ejemplos del resto del prompt.

### 4. Decile QUÉ HACER, no qué NO hacer

❌ "No uses markdown"
✅ "Escribí en prosa fluida con párrafos completos"

Claude responde **mejor a instrucciones positivas** que a negaciones.

### 5. Dale un rol

Una línea al inicio:

```
Sos experto en Python que explica a principiantes
```

Cambia el tono y la profundidad. **Es lo más barato y lo que más impacta**.

### 6. Si es algo simple, bajá el effort

Por default Opus 4.7 piensa en nivel **xhigh** (el máximo). Si le pedís algo corto o rutinario:

```
Usá effort medium para esta tarea (no xhigh).
```

No quemes tokens de más.

## 4 prompts plantilla (en inglés a propósito)

Estos están en **inglés** porque así vienen de la doc oficial y así rinden mejor (entrenamiento). Copialos y pegalos.

### 1. Respuestas cortas (anti-verbosidad)

Cuando Claude se extiende de más:

```
Provide concise, focused responses. Skip non-essential context,
and keep examples minimal.
```

### 2. Confirmar antes de acciones riesgosas

Para coding agents — que no borre archivos, haga `git push --force`, ni suba cambios sin preguntar:

```
Consider the reversibility and potential impact of your actions.
You are encouraged to take local, reversible actions like editing
files or running tests, but for actions that are hard to reverse,
affect shared systems, or could be destructive, ask the user
before proceeding.
```

### 3. No sobre-ingeniería

Para que no invente archivos, helpers o abstracciones que no le pediste:

```
Avoid over-engineering. Only make changes that are directly
requested or clearly necessary. Keep solutions simple and focused.
Don't add features, refactor code, or make improvements beyond
what was asked.
```

### 4. Investigar antes de responder

Contra alucinaciones de código — lo obliga a **leer archivos reales** antes de afirmar:

```
Never speculate about code you have not opened. If the user
references a specific file, you MUST read the file before
answering. Make sure to investigate and read relevant files
BEFORE answering questions about the codebase.
```

## Anti-patrones específicos de Opus 4.7

### 1. Activarlo "por las dudas"

Si activás Opus 4.7 como default, **cada prompt** te come 2x. En 4-5 prompts en Pro $20, cartelito.

**Default: Sonnet 4.6.** Sube cuando necesites.

### 2. Usarlo para tareas mecánicas

Reformatear código, escribir docs simples, renamear variables → **Sonnet o Haiku alcanzan**. Opus 4.7 te cobra como genio para hacer trabajo de junior.

### 3. Sesiones largas sin contexto management

Opus 4.7 piensa mucho **por turno**. Si tu sesión tiene 30 turnos con Opus 4.7, gastaste como 60 turnos en Sonnet.

Para sesiones largas: **Sonnet 4.6** base + invocás Opus 4.7 solo para los momentos críticos.

### 4. Code generation masivo

Generar 500 líneas de boilerplate → Sonnet. Generar lógica core compleja de 50 líneas → Opus 4.7.

### 5. No bajar effort cuando aplica

Si pedís algo simple en Opus 4.7 y no decís "low effort", piensa al máximo igual. **Explicitá** el effort según tarea.
