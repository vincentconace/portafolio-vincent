## Qué es

Statusline para game devs que compilan a múltiples plataformas simultáneamente. Muestra build progress per platform:
- iOS
- Android
- PC (Windows/Mac/Linux)
- WebGL

Tracking de completion en tiempo real. Para shops que disparan builds en paralelo y querés awareness del status.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `statusline` |
| Installs | 63 |
| Plataformas | iOS, Android, PC, WebGL |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/multiplatform-build-status-statusline
```

Para Claude:

```
Instalé el setting statusline/multiplatform-build-status-statusline.
Mostrame qué builds detecta como activos.
```

## Qué podés hacer con esto

### Builds en paralelo visible

Disparás builds para 4 platforms — ves progress de los 4 en statusline.

### Para shops Unity/Unreal con CI

Si tu game se compila para varios platforms automáticamente, este awareness es útil.

### Combo con [Game Performance Monitor Statusline](/boveda/setting-game-performance-monitor-statusline)

Game Performance para runtime. Multiplatform Build para builds. Combinables si querés ambos contexts.

### Combo con [Asset Pipeline Controller](/boveda/setting-asset-pipeline-controller-statusline)

Asset Pipeline para processing. Multiplatform Build para final builds. Stack game dev completo.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no hacés game dev multi-platform:** no aplica.
- **Si tus builds van por CI cloud:** el statusline puede no verlos locally.
- **Para web dev solo:** [Context Monitor](/boveda/setting-context-monitor) es la opción.
- **Solo 63 installs:** nicho.

</Callout>
