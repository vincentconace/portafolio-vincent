## What it is

Overrides the model to `claude-3-5-haiku-20241022`. Faster and cheaper than Sonnet, less capable for complex tasks.

For tasks where speed or cost is the priority — e.g. simple queries, lookups, small edits.

## How to install it

```bash
npx claude-code-templates@latest --setting model/use-haiku
```

For Claude:

```
I installed the model/use-haiku setting. Confirm that I'm using Haiku
now.
```

## What you can do with it

### Small / cost-conscious tasks

For queries that don't need Sonnet's full capability.

### Faster iteration

Haiku responds faster. Great for quick exploration.

### Significant cost reduction

Haiku is quite a bit cheaper than Sonnet.

### Combo with [Performance Optimization](/boveda/setting-performance-optimization)

Performance Opt limits output. Use Haiku changes the model. Both reduce cost.

### Opposite of [Use Sonnet](/boveda/setting-use-sonnet)

These two are opposites. For complex code work, use Sonnet.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **For complex tasks:** Haiku can give poor answers. Sonnet/Opus are better.
- **For serious code generation:** Haiku has lower output quality.
- **If your workflow is already slow for other reasons:** the speed gain may not make up for the quality loss.

</Callout>
