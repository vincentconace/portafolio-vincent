## Qué es

Después de cada modificación de código, el hook detecta el test runner del proyecto (Jest, Vitest, pytest, etc.) y corre tests rápidos. Si algo se rompe, te avisa inmediatamente.

Hook event: `PostToolUse`. Foco: "quick tests" — unit, no E2E pesados.

1,017 installs es alto para hooks de testing — refleja la importancia de catch regressions on the fly.

## Qué podés hacer con esto

### Catch regresión instantáneo

Si Claude rompe un test al editar, lo ves inmediatamente. No después de 30 ediciones.

### Test-first workflow

Si seguís TDD, este hook valida que tu green stay green durante refactors.

### Combo con [TDD Gate](/boveda/hook-tdd-gate)

TDD Gate fuerza que existan tests primero. Run Tests After valida que sigan green.

### Combo con [Lint On Save](/boveda/hook-lint-on-save)

Lint + tests = doble check de quality. Sin scaffolding extra.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Tests lentos:** si tu suite tarda 30+ segundos, fricción notable.
- **Sin tests existentes:** el hook no inventa tests.
- **Para edits experimentales:** tests verdes son útiles para release, no para WIP.
- **Si tu CI ya corre tests:** local también puede ser overhead.

</Callout>
