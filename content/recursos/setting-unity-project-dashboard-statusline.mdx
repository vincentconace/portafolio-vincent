## Qué es

Statusline focused en Unity dev workflow. Muestra:
- **Scene info** (scene actual)
- **Build target** (iOS/Android/Mac/Windows/etc.)
- **Asset pipeline status**
- **Unity version**

Para devs Unity específicamente. Contexto rich del editor visible en statusline.

## Qué podés hacer con esto

### Scene + build context constante

Sabés en qué scene estás trabajando sin buscar.

### Detection de build target switching

Si cambiás de iOS a Android target, el statusline lo confirma.

### Asset pipeline awareness

Mientras Claude codea, ves si assets están processing en background.

### Combo con [Game Performance Monitor](/boveda/setting-game-performance-monitor-statusline)

Unity Dashboard cubre editor context. Game Performance cubre runtime. Solo podés tener un statusline.

### Combo con [Multiplatform Build Status](/boveda/setting-multiplatform-build-status-statusline)

Unity Dashboard cubre editor. Multiplatform Build cubre builds finales. Diferentes phases del workflow.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Unity:** no aplica. Para Unreal hay otros options.
- **Si tu workflow no toca Unity Editor:** menos util.
- **Para web dev:** otros statuslines.

</Callout>
