## What it is

It's the most sophisticated security hook in the catalog. Before any Bash command runs in a Claude Code session, a lightweight Claude sub-agent evaluates the command against destructive or irreversible patterns. If it spots danger, it blocks it.

How it differs from a traditional blocklist (which only matches exact strings): this one uses **AI judgment**. It detects variations, contexts, intent. A direct `rm -rf` gets caught, but so does `rm -rf $UNDEFINED_VAR/*` or `find . -delete`.

Hook event: `PreToolUse`. Specifically on the Bash tool.

7 installs (just published), but the pattern is promising.


## What you can do with it

### Block accidental destructive commands

Commands like `rm -rf`, `dd if=...`, `mkfs.*`, `> /dev/sda` — blocked before they run. Even when they're buried inside a complex script where they aren't obvious.

### Detect variations

`rm -rf $HOME/*` with a variable that could expand into something bad: the sub-agent catches it. A traditional blocklist would let it through.

### Irreversibility

It's not just about "destructive." It also blocks operations you can't easily undo: `chmod -R 777`, `kill -9` on critical processes, force-push to main.

### Working in autonomous mode

If you leave Claude running with `/goal` or `/loop`, this hook is basically mandatory. Without it, a single wrong decision by the model can be catastrophic.

### Combo with [Claude Forge](/boveda/plugin-claude-forge)

Claude Forge ships 6 security layers. This hook is complementary and more granular, specifically for Bash commands.

## Prerequisites

- **Claude Code** with hooks enabled
- **A Claude plan** with enough tokens (the sub-agent consumes tokens every time it evaluates a command)

## When it's NOT worth it

<Callout type="warning" title="Before you install">

- **Token cost:** every bash command fires the sub-agent. If your workflow runs a lot of commands, it multiplies your spend.
- **Latency:** the AI evaluation adds half a second or more per command. On very fast tasks, you'll notice it.
- **False positives:** the AI can be overly cautious. Legitimate commands may need an override.
- **If you already have deterministic security hooks:** you might get overlap.
- **Only 7 installs:** it's a new pattern. Test it in a sandbox before putting it into production sessions.

</Callout>
