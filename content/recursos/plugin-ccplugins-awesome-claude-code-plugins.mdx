## Qué es

"Awesome Claude Code Plugins" es la versión "awesome list" para Claude Code: una colección curada de slash commands, subagents, MCP servers y hooks. La diferencia clave con otros marketplaces grandes es que acá la curación está más cuidada — no es "todo lo que hay", es "lo que vale la pena".

Lo más importante: incluye plugins oficiales de Anthropic dentro del catálogo. Si querés probar lo que Anthropic publica como referencia sin tener que buscarlo en repos separados, este es el atajo.

Cubre 118 plugins en total, con foco en flujos de PR, code review, debugging, git, planning, security y un set sólido de agents.

## Cómo se instala

```bash
# 1) Agregar el marketplace
/plugin marketplace add ccplugins/awesome-claude-code-plugins

# 2) Instalar el set base "oficial Anthropic"
/plugin install pr-review-toolkit@ccplugins-awesome-claude-code-plugins
/plugin install commit-commands@ccplugins-awesome-claude-code-plugins
/plugin install agent-sdk-dev@ccplugins-awesome-claude-code-plugins
```

O `/plugin` → Discover.

Para Claude:

```
Instalá el marketplace ccplugins/awesome-claude-code-plugins. Mostrame solo
los plugins firmados por Anthropic y dame el orden recomendado para instalarlos.
```

## Qué podés hacer con esto

### Empezar por los oficiales

Si todavía no instalaste nada de la comunidad y querés ir a lo seguro, los tres plugins de Anthropic (`agent-sdk-dev`, `pr-review-toolkit`, `commit-commands`) son el set base más conservador. Después capas con otros si querés.

### PR review con seis ángulos

`pr-review-toolkit` corre seis agents distintos sobre tu PR: uno que revisa comments, otro tests, otro error handling, otro type design, otro code quality, otro simplificación. Es más completo que pedirle a Claude "revisame este PR" en una sola pasada.

### Hook de seguridad pasivo

`security-guidance` se queda en background y te avisa si una edición introduce algo riesgoso (inyección de comandos, XSS, patrones inseguros). No bloquea — alerta. Buen guardarraíl para no enviar a prod cosas obvias.

### Combinar con planning de Galen Ward

`plan` y `explore` arman el patrón Explore → Plan → Execute. Combinalos con `pr-review-toolkit` al final y tenés un workflow guiado de feature → review → merge.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si ya tenés [Claude Code Setup](/boveda/claude-code-setup) corrido,** ya recibiste recomendaciones tipo Anthropic. Este marketplace expande eso pero solapa parcialmente.
- **Solapamiento con `cc-marketplace`:** ambos comparten muchos plugins (los de Anand, Galen, Galpert). Si vas a usar uno, no instales ambos.
- **No esperes una curación temática única.** Es un mix de áreas — ventaja si querés variedad, desventaja si buscás un foco.

</Callout>
