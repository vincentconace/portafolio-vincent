## What it is

monday.com is a work management platform widely used at mid-size and large companies: visual boards, workflows, automations, integrations. More structured than Trello, more opinionated than Notion.

This MCP plugs the monday API into Claude Code. The official description promises to "operate reliably inside real workflows" — the idea is that an agent can not only read state but also update items, move between groups, and assign people.

## How to install it

```bash
npx claude-code-templates@latest --mcp productivity/monday
```

Prerequisites: a monday.com account + a Personal API token from Admin → API.

For Claude:

```
I installed the productivity/monday MCP. Set up MONDAY_API_TOKEN and
list the available boards for me. Show me the items on board X with
status "Working on it" and a due date this week.
```

## What you can do with this

### Item triage

"On the `Current Sprint` board, list the items with status `Stuck` or no assigned owner. For each one, tell me when it was created and what's blocking it."

### Bulk creation

"Import these 50 leads from my CSV as items on the `Pipeline` board. Map the columns: A→Name, B→Email, C→Company, D→Status (default `New`)."

### Programmatic updates

"For every item with the `deploy-ready` label and status `Done`, move it to the `Released` board and add today's date."

### Reporting

"Generate a weekly report: completed items, overdue items, items by owner. Output it as markdown so I can paste it into my update."

### Assisted workflows

"When an item moves to `Code Review`, automatically assign me as reviewer if no one is assigned yet." You combine the MCP with hooks or automations.

### Combined with [Google Workspace](/boveda/mcp-google-workspace)

monday + Calendar + Gmail = a complete pipeline: items in monday, meetings in Calendar, communication in Gmail. Claude correlates them.

## Prerequisites

- An active **monday.com account**
- A **Personal API token** (Admin → API)
- **Permissions** on the relevant boards/workspaces

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you use another platform** ([Trello](/boveda/mcp-mcp-server-trello), Asana, Linear, ClickUp): this MCP is monday-only.
- **Destructive operations:** delete items, delete columns. Use approval hooks.
- **API rate limits:** monday has quotas. Bulk imports can temporarily block you.
- **Broad-scope token:** the token lets Claude operate everything your user can see. Restrict it if possible.
- **If your use is casual read-only:** the MCP adds overhead if you only read boards now and then.

</Callout>
