## Qué es

**#3 absoluto del catálogo (2,076 installs).** El universal formatter pipeline.

Después de cada edit, detecta el lenguaje y aplica el formatter apropiado:
- **JS/TS** → Prettier
- **Python** → Black
- **Go** → gofmt
- **Rust** → rustfmt
- **Y más language-specific**

Reemplaza los hooks individuales ([Format JavaScript Files](/boveda/hook-format-javascript-files), [Format Python Files](/boveda/hook-format-python-files)) con uno solo que cubre todos.

Hook event: `PostToolUse`.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook development-tools/smart-formatting
```

Pre-requisito: los formatters de tus lenguajes instalados.

Para Claude:

```
Instalé el hook development-tools/smart-formatting. Listame qué
formatters detecta como disponibles en mi sistema (prettier, black,
gofmt, etc.).
```

## Qué podés hacer con esto

### Format pipeline universal

Sin importar el lenguaje, format se aplica. No tenés que instalar hook por lenguaje.

### Para repos polyglot

Si tu repo tiene JS + Python + Go (caso común), un solo hook cubre todos.

### Combo con [Lint On Save](/boveda/hook-lint-on-save)

Format primero, lint después. Pipeline limpio cross-language.

### Para sesiones autónomas

`/goal` largo: código siempre queda formateado, sin que tengas que recordar pedirlo.

### Reemplaza varios hooks

Si tenés [Format JS](/boveda/hook-format-javascript-files) y [Format Python](/boveda/hook-format-python-files) por separado, este los reemplaza con uno.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin formatters installed:** el hook falla silently para lenguajes sin tool.
- **Si tu team tiene formatter custom:** puede no estar en la lista soportada.
- **Para lenguajes muy nicho:** verificá compatibility.
- **Si ya tenés [Format JS](/boveda/hook-format-javascript-files) o [Format Python](/boveda/hook-format-python-files):** no instales ambos.

</Callout>
