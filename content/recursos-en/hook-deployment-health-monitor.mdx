## What it is

A hook that continuously monitors:
- **Deployment status** — which deploys are running, which ones failed
- **Error rates** — error spikes post-deploy
- **Performance metrics** — degradations

Specifically focused on Vercel deploys (the hosting mentioned in the description). It sends notifications when something breaks.

Hook events: `PostToolUse` + `SessionStart`.

## How to install it

```bash
npx claude-code-templates@latest --hook automation/deployment-health-monitor
```

Prerequisite: credentials or tokens to access the provider (Vercel or another).

```

```

## What you can do with it

### Post-deploy automation

After Claude pushes changes and a deploy is triggered, the hook validates that everything goes well and notifies you if it doesn't.

### Fast regression detection

If the new deploy introduces a high error rate, the hook warns you before you notice. Useful for autonomous `/goal` runs.

### Performance budget

If the new version is slower than the previous one (degraded LCP, FID), the hook detects it.

### Combo with [Dependency Checker](/boveda/hook-dependency-checker)

Dependency Checker pre-deploy + Deployment Health Monitor post-deploy = a complete supply chain + runtime health cycle.

### Combo with [Sentry MCP](/boveda/mcp-sentry)

Sentry has this kind of monitoring built in. Compare them based on your preference.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **Without Vercel or other supported providers:** check what the hook supports before installing.
- **If your observability lives elsewhere** (Datadog, Sentry, Grafana): this hook may overlap.
- **For infrequent deploys:** the hook shines with continuous deployment. For monthly releases, doing it manually is enough.
- **Notifications config:** if you don't set up a channel (Discord, Slack, email), alerts won't reach you.

</Callout>
