## Qué es

Es la versión **oficial** del Sentry MCP en modo local. Se ejecuta vía `npx` sin tener que armar infra adicional. Comparado con el [Sentry MCP (community)](/boveda/mcp-sentry) (270 installs), este es más reciente y oficialmente respaldado por Sentry — solo 21 installs todavía.

La distinción "local" suele referir a:
- Corre como proceso local de Node
- Conecta a tu Sentry SaaS o self-hosted (URL configurable)
- No requiere setup adicional vs versiones que necesitan server propio

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/sentry-local
```

Pre-requisito: cuenta Sentry + access token.

Para Claude:

```
Instalé el MCP devtools/sentry-local. Configurá SENTRY_AUTH_TOKEN y
SENTRY_URL si self-hosted. Listame los issues más recientes y top
errors del proyecto.
```

## Qué podés hacer con esto

Las mismas operaciones que con el [Sentry MCP community](/boveda/mcp-sentry):

- Triage de issues
- Debugging asistido con stack traces
- Análisis de releases
- Correlación con deploys
- Performance triage

Casos de uso detallados en [Sentry MCP](/boveda/mcp-sentry) — aplican igual.

## Diferencias con la versión community

- **Mantenido por Sentry directamente** — releases sincronizados con producto
- **Menos installs (21 vs 270)** — comunidad probando reducida
- **Soporte oficial** si tenés plan enterprise
- **Updates predictibles**

## Cuándo elegir cuál

| Caso | Recomendación |
| --- | --- |
| Soporte oficial importa | **Sentry Local (oficial)** |
| Más comunidad probando | **Sentry (community)** |
| No tengo preferencia | **Empezá con community por madurez** |
| Stack enterprise crítico | **Oficial** |

## Pre-requisitos

- **Cuenta Sentry** + access token
- **Node.js** para ejecutar via npx
- **No los instales junto con el community** (colisión)

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si ya tenés [`sentry`](/boveda/mcp-sentry) community funcionando:** no migrés a la fuerza. Probá en sandbox.
- **21 installs es bajo:** menos issues conocidos públicamente.
- **Para self-hosted Sentry on-prem:** verificá que el MCP soporte tu setup.

</Callout>
