## Qué es

Configura timeouts y output limits para MCP server operations:

- `MCP_TIMEOUT: 30000` (30s) — timeout de startup del MCP server
- `MCP_TOOL_TIMEOUT: 60000` (60s) — timeout por tool execution
- `MAX_MCP_OUTPUT_TOKENS: 50000` — output cap por tool call

Previene que MCP servers lentos hagan hang a tu sesión.

## Qué podés hacer con esto

### Anti-hang en MCPs lentos

Si un MCP server tiene un bug o se cuelga, no te traba indefinidamente.

### Cap a output

`MAX_MCP_OUTPUT_TOKENS` evita que un MCP genere MB de output que consuma tu context.

### Tunear para tu uso

- Para sistemas rápidos: bajar timeouts (feedback más rápido on hangs)
- Para MCPs complejos (browser automation, big scrapes): subir timeouts

### Combo con [Bash Timeouts](/boveda/setting-bash-timeouts)

Bash Timeouts para shell commands. MCP Timeouts para MCPs. Cubren todos los entry points.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás MCPs:** no aporta.
- **Si tus MCPs son rápidos:** los defaults te alcanzan.
- **Para MCPs que necesitan minutos:** subí timeouts con cuidado para no atascar la sesión.

</Callout>
