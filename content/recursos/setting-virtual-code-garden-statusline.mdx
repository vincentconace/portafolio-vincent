## Qué es

Statusline gamificado tema garden. Tu código es un jardín:
- **Plantas evolucionan** de seeds a trees según activity
- **Weather effects** dynamic (sun, rain, etc.)
- Calmer que [Tamagotchi](/boveda/setting-programmer-tamagotchi-statusline) (no necesitás "feed" nada)

Para users que prefieren gamification calmer y aesthetic vs combat/competitive.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/virtual-code-garden-statusline
```

Para Claude:

```
Instalé el setting statusline/virtual-code-garden-statusline. Mostrame
el state actual de mi garden y el weather.
```

## Qué podés hacer con esto

### Calmer gamification

Si Tamagotchi o RPG son too much, garden es chill.

### Aesthetic nature

Para users que disfrutan slow growth metaphor.

### Para focus sessions

Garden no demanda atención; just creciendo. No es disruptivo.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para trabajo serio:** [Context Monitor](/boveda/setting-context-monitor) es la opción.
- **Si nature theme no resuena:** alternative gimmicky.

</Callout>
