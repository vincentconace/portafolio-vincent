## Qué es

Devplan MCP conecta Claude Code con la plataforma Devplan. La descripción oficial en aitmpl es breve — para detalles concretos de qué expone el server, mirá el [JSON de configuración](https://github.com/davila7/claude-code-templates/blob/main/cli-tool/components/mcps/devtools/devplan-mcp.json) y la página oficial de Devplan.

Si tu equipo ya usa Devplan para tracking de proyectos, planning o agentic workflows, este MCP es el puente para que Claude consulte/actúe contra esa plataforma desde tu editor.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/devplan-mcp
```

Pre-requisito: cuenta y credenciales de Devplan.

Para Claude:

```
Instalé el MCP devtools/devplan-mcp. Decime qué env vars o credenciales
necesito setear y mostrame qué operaciones expone el server.
```

## Qué podés hacer con esto

### Integrar Devplan en tu workflow de Claude

Si tu equipo planifica en Devplan y vos codeás en Claude, este MCP elimina el cambio de contexto entre los dos. Vos preguntás "qué tareas tengo en sprint" desde Claude y el MCP responde.

### Sync entre código y plan

Patrones tipo "cuando termino una tarea de código, actualizo su estado en Devplan automáticamente" se vuelven triviales con el MCP + un hook.

### Onboarding a contexto del proyecto

Cuando arrancás una nueva tarea, Claude puede leer el contexto desde Devplan (descripción, criterios de aceptación, links) antes de tocar código.

## Pre-requisitos

- **Cuenta Devplan** activa
- **API token** o credenciales que el server pida
- **Verificá el JSON de configuración** para entender qué exactamente expone

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Devplan:** obvio. Para Linear hay otros MCPs; para Jira también; para Notion también.
- **Información limitada en aitmpl:** la descripción es genérica. Si tu uso es serio, mirá la doc de Devplan oficial y el JSON antes de comprometerte.
- **Pocos installs (43):** menos comunidad probando. Probá en sandbox antes de uso productivo.

</Callout>
