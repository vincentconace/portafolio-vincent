## Qué es

Antes de cada `git commit` que Claude ejecute, este hook valida que el mensaje siga el formato Conventional Commits:

```
type(scope): description
```

Types soportados: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `ci`, `perf`, `build`.

Si el mensaje no respeta el formato, el commit se bloquea. Hook event: `PreToolUse`.

```bash
npx claude-code-templates@latest --hook git/conventional-commits
```

## Qué podés hacer con esto

### Convention forzada en todo el repo

Ya no importa si Claude tira un commit informal. El hook lo formatea o lo rechaza.

### Changelog autogenerado

Con commits conventional, herramientas como `conventional-changelog` arman el CHANGELOG automáticamente al release.

### Semantic versioning automático

`feat:` → minor bump. `fix:` → patch. `BREAKING CHANGE:` → major. Tu CI puede manejarlo.

### Para teams nuevos en Convention

Forzar el patrón es la forma más rápida de adoptar la convención.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu equipo no usa Convention:** no impongas — primero alineá con el equipo.
- **Para repos personal/experimental:** convention es overhead. Commit a tu gusto.
- **Si tu CI ya valida:** husky + commitlint hacen lo mismo en pre-commit hook tradicional.
- **Quick commits de WIP:** el hook bloquea. Para hotfixes urgentes, override necesario.

</Callout>
