## Qué es

Cuando arrancás Claude Code en modo debug (flag `--debug` o `-d`), este hook abre automáticamente una ventana auxiliar que hace tail -f del debug log en vivo. Cuando termina la sesión, la ventana se cierra sola.

Hook events: `SessionStart` (abre) + `SessionEnd` (cierra).

Útil cuando estás diagnosticando un bug de Claude Code o de tu setup de hooks/MCPs. El log debug tiene mucha info; tener una ventana siempre visible te ahorra el `tail -f` manual.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook development-tools/debug-window
```

Después arrancá Claude con debug:

```bash
claude --debug
```

## Qué podés hacer con esto

### Diagnostic en vivo

Durante una sesión con bugs, vas viendo qué está pasando en debug log sin tener que pausar y abrir terminal nueva.

### Aprender qué hace Claude por dentro

Si querés entender el flujo interno (cómo invoca tools, cómo aplica hooks), tener el debug log a la vista enseña mucho.

### Para developers de plugins/hooks

Si estás desarrollando un plugin o hook custom, el debug window te muestra exactamente cómo se está invocando.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Solo se activa con `--debug`:** sin el flag, no hace nada. No es un hook que aplica siempre.
- **Para sesiones productivas normales:** debug log es ruido. No corras Claude siempre en debug mode.
- **Pantalla pequeña:** la ventana extra compete por espacio.
- **Si no necesitás ver debug en vivo:** podés revisar el log post-mortem.

</Callout>
