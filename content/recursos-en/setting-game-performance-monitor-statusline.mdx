## What it is

A custom statusline built specifically for game development. It shows:
- **FPS targets** and compliance
- **Draw calls** count
- **Memory usage** at runtime
- **Polygon count** and build optimization metrics

Niche: Unity, Unreal, and Godot devs who live with these metrics as part of their routine.

## What you can do with it

### Always-visible metrics

For game dev, metrics like FPS and draw calls are critical. Having them always in the statusline keeps them front of mind.

### Build optimization tracking

While you code features, you can see whether your changes affect the target metrics.

### Combo with [Asset Pipeline Statusline](/boveda/setting-asset-pipeline-controller-statusline)

Asset Pipeline for asset processing. Game Performance for runtime. They cover different angles of the game dev workflow.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you don't do game dev:** the metrics don't apply.
- **If your engine doesn't expose those metrics via an accessible interface:** the setting may not detect them.
- **For web devs:** [Context Monitor](/boveda/setting-context-monitor) is the option.
- **Only 70 installs:** very niche. Validate the implementation.

</Callout>
