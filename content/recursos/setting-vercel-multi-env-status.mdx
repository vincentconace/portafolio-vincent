## Qué es

Statusline que monitorea ambos production y preview environments al mismo tiempo. Color-coded indicators:
- Verde = healthy
- Amarillo = warning
- Rojo = error

Para teams managing multiple targets — production stable + preview testing en paralelo.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `statusline` |
| Installs | 104 |
| Envs monitored | Production + Preview |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/vercel-multi-env-status
```

Para Claude:

```
Instalé el setting statusline/vercel-multi-env-status. Mostrame el
status actual de prod vs preview.
```

## Qué podés hacer con esto

### Awareness cross-env

Sabés si prod sigue healthy mientras testeás en preview.

### Catch divergencias

Si preview pasa pero prod tiene un error, el statusline lo señala.

### Para teams con muchas branches

Cada PR genera preview deploy. El statusline summary te da vista rápida.

### Combo con [Vercel Deployment Monitor](/boveda/setting-vercel-deployment-monitor)

Multi Env (este) cubre ambos. Deployment Monitor cubre uno. Alternativos según preferencia.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si solo tenés un env:** [Vercel Deployment Monitor](/boveda/setting-vercel-deployment-monitor) es suficiente.
- **Sin Vercel:** no aplica.
- **Para tracking de errors específicamente:** [Error Alert System](/boveda/setting-vercel-error-alert-system).

</Callout>
