## What it is

A setting that configures a custom statusline for projects with heavy asset pipelines:

- **Texture processing** (compression, format conversion)
- **Model optimization** (3D meshes, LOD generation)
- **Audio compression** (ogg, mp3, opus)
- **Platform-specific variants** (PC vs mobile vs console)

It shows the processing queue, current status, and metrics.

Niche: game dev, multimedia apps, AR/VR.

## What you can do with this

### Pipeline visibility during development

While you code, you see in the statusline how many textures are left to process and which model is being optimized.

### For projects with lots of assets

Game dev typically has hundreds of textures + models. The statusline gives you awareness without opening external tools.

### Multi-platform tracking

If you build for PC + mobile + Switch, the statusline can show progress per target.

### Combo with [Context Timeline Hook](/boveda/hook-context-timeline)

Statusline in the editor + Context Timeline in the browser = double visibility into what's happening.

### Compared to [Claude HUD](/boveda/plugin-claude-hud)

Claude HUD is generic (context, tools, todos). Asset Pipeline is game dev niche. If your use case is game dev, this one delivers more value.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you don't do game dev / multimedia:** this statusline serves no purpose.
- **If your pipeline lives elsewhere** (CI/CD): the statusline may not reflect it.
- **Only 84 installs:** very niche. Validate the implementation.
- **For JS/TS web projects:** [Claude HUD](/boveda/plugin-claude-hud) is more useful.

</Callout>
