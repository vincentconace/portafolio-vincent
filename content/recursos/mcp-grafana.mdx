## Qué es

Grafana es probablemente el stack más usado del open source para observability: dashboards, queries a Prometheus, Loki, InfluxDB, Tempo, y otros datasources. Este MCP da acceso a esa data desde Claude Code.

En vez de abrir el browser, navegar al dashboard y escribir PromQL/LogQL a mano, le preguntás a Claude. Claude consulta los datasources, te devuelve resultados estructurados.

Soporta self-hosted y Grafana Cloud — útil porque la realidad de empresa suele ser mix.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/grafana
```

Pre-requisitos: URL de tu Grafana + service account token con permisos correctos.

Para Claude:

```
Instalé el MCP devtools/grafana. Después de setear GRAFANA_URL y
token, listame los dashboards en el folder X y mostrame el estado
de alertas activas.
```

## Qué podés hacer con esto

### Consultas a métricas en lenguaje natural

"¿Cuál fue el peor latency p95 del servicio `payments-api` en las últimas 24h? ¿En qué momento ocurrió y qué endpoints estuvieron involucrados?". Claude traduce a PromQL.

### Inspección de logs en Loki

"En Loki, mostrame los logs del namespace `prod` con level ERROR en los últimos 30 min, agrupados por servicio". LogQL generado y ejecutado.

### Estado de alertas

"¿Qué alertas están firing en este momento? Agrupame por severity y por team owner". Útil para incidentes.

### Comparación temporal

"Comparame el RPS del servicio X esta semana vs la semana pasada en el mismo intervalo". Detección de regresiones.

### Cross-reference con deploys

"La latencia se disparó a las 14:30. ¿Qué dashboards tienen anotaciones de deploy alrededor de esa hora?". Cruzar métricas con eventos para root cause.

### Crear dashboards o paneles

Algunas implementaciones del MCP permiten también crear/modificar dashboards. Validá con el JSON qué exactamente expone.

## Pre-requisitos

- **Acceso a Grafana** (self-hosted URL o Grafana Cloud)
- **Service account token** o API key con permisos sobre los datasources
- **Datasources configurados** en Grafana (el MCP es un cliente; no setea Prometheus por vos)

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si usás otra observability platform** ([Datadog, New Relic, Dynatrace](/boveda/mcp-dynatrace)): este MCP es Grafana específico.
- **Sin datasources funcionando:** el MCP consulta lo que ya tenés. Si Prometheus no está scrapeando, el MCP no va a inventarte data.
- **Si tu permisión es read-only y el plugin permite write:** verificá scopes del token para evitar accidentes.
- **Para alertas críticas:** el MCP no reemplaza el routing de PagerDuty / Opsgenie. Es para consulta y triaging, no para reception.

</Callout>
