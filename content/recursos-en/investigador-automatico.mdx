## What it is

A digital detective that **hits 8 different platforms**, reads everything said about your topic over the last 30 days, and hands you a **structured report** as if it were an expert analyst.

Before:
- 30 min on Reddit
- 30 min on X
- 30 min on YouTube
- 30 min cross-referencing info
- 2h total per topic

With Last30Days: **5-8 min**, same output. Topic in, report out.

Repo: `mvanhorn/last30days-skill`. Open source and free.

## Basic usage

```
> /last30days artificial intelligence
```

Claude:
1. Hits the 8 platforms
2. Filters relevant content from the last 30 days
3. Identifies patterns
4. Synthesizes key points
5. Hands you a report with **links to sources**

Time: 2-8 min depending on the topic (more popular = more data = more time).

## Command variants

### Compare 2 topics

```
> /last30days --compare "React vs Vue"
```

Side-by-side output: what each community says about each topic, where they're strong, final comparison.

Ideal for deciding between tools, strategies, niches.

### Quick search

```
> /last30days --quick cryptocurrencies
```

Light version: less depth, more speed. Gives you **the essentials** without the detail.

When: you want a general overview in 1-2 min, not a deep analysis.

### Time window

```
> /last30days --days 7 startups          # last week
> /last30days --days 14 gen AI            # last 2 weeks
> /last30days --days 30 web3             # default
> /last30days --days 90 SaaS market     # last quarter
```

Default: 30 days. For fast-moving topics (crypto, AI), 7-14 days is more useful.

You know what's moving each community before investing 6 months.

### 2. Monitor your industry

```
> /last30days "AI agents for marketing"
```

You find out about new tools, success stories, common problems.

### 3. Pre-research for content creation

```
> /last30days "no-code in LATAM"

Output: what's being discussed, what hurts, what's missing. **Your next post writes itself** with that data.

```

## Anti-patterns

### 1. Topic too broad

❌ `/last30days technology`
✅ `/last30days "AI agents for B2B SaaS"`

More specific = better synthesis.

### 2. Not checking the sources

The report synthesizes but **always check the main sources**. Sometimes Claude weighs things wrong by volume vs quality.

### 3. Trusting 100% without context

Last30Days tells you **what's being said now**. It doesn't tell you if that opinion will hold up for 6 months. For long-term decisions, complement it with more stable sources.

### 4. Using the default 30 days for stable topics

If you're researching something that doesn't change much (e.g. Italian cooking, gardening), 30 days gives you little new data. Bump it up to 90+ days.

### 5. Same topic 5x a day

The platforms have rate limits. If you run it many times in a row with the same topic, you can get throttled.

## When it's NOT a good fit

| Case | Why |
|------|---------|
| Rigorous academic research | You need papers, not opinions |
| Critical legal/financial decisions | Validate with human experts |
| Topics with very little online activity | If nobody's talking, there's no data |
| Strict compliance | The platforms have unverified content |
