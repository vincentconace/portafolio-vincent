## What it is

Pre-authorizes common npm development commands:
- `npm run lint`
- `npm run test`
- `npm run build`
- `npm start`

It's the #2 most adopted permission setting (1,823 installs), just behind [Allow Git Operations](/boveda/setting-allow-git-operations) in a similar category. The reason: an uninterrupted JS/TS workflow is what almost all of us want.

## What you can do with it

### Uninterrupted JS workflow

`npm run lint && npm run test && npm run build` runs without an "allow" prompt on every step.

### For autonomous sessions

A `/goal` that includes build + test: the setting avoids confirmations.

### Combo with [Lint On Save Hook](/boveda/hook-lint-on-save)

The hook triggers lint automatically. The setting authorizes execution without asking permission. Silent pipeline.

### Combo with [Run Tests After Changes Hook](/boveda/hook-run-tests-after-changes)

Setting + hook = tests run without confirmation on every edit.

### For polyglot projects

If your project mixes npm with other tools, this setting covers the JS/TS side.

## When it's NOT a good fit

<Callout type="warning" title="Before enabling it">

- **Destructive custom scripts:** if your `npm run something` is `rm -rf node_modules && reinstall`, authorizing it without thinking may be questionable.
- **No scripts in package.json:** the setting does nothing useful.
- **If you want to review every step:** this setting skips review.
- **For non-npm projects:** Yarn/pnpm may need a separate or equivalent setting.

</Callout>
