## What it is

Override the default model that Claude Code uses. Configure:
- `model` — the primary model (e.g. `claude-3-5-sonnet-20241022`)
- `ANTHROPIC_SMALL_FAST_MODEL` — the secondary model for fast tasks (e.g. Haiku)

Useful for:
- Testing new versions before adopting them
- Using an org-specific model (custom-trained)
- Tuning cost: Haiku for small tasks, Sonnet for complex ones

## How to install it

```bash
npx claude-code-templates@latest --setting global/custom-model
```

Edit your config with the real models:

```json
{
  "model": "claude-3-5-sonnet-20241022",
  "env": {
    "ANTHROPIC_SMALL_FAST_MODEL": "claude-3-5-haiku-20241022"
  }
}
```

For Claude:

```
I installed the global/custom-model setting. Show me the models
available for my API key and suggest which combination works best
for my usage (balancing cost and capability).
```

## What you can do with it

### Cost control

Small tasks to Haiku (much cheaper). Complex tasks to Sonnet/Opus.

### Testing versions

When a new model is released, you can try it in specific projects without migrating everything.

### For teams on different tiers

Dev team on Sonnet, data team on a specific model.

### Combo with [Bedrock Configuration](/boveda/setting-bedrock-configuration)

If you use Bedrock, the model IDs are different. Custom Model lets you specify the correct ID.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If the default is enough for you:** don't add complexity.
- **If you specify a model that doesn't exist:** Claude Code fails. Verify the exact IDs.
- **New models sometimes have breaking changes:** validate compatibility with your workflow before putting it in prod.
- **Cost:** watch out for accidentally switching to a more expensive model.

</Callout>
