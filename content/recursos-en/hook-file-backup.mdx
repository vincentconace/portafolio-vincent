## What it is

Before modifying a file, the hook creates a copy in a `.backups/` folder with a timestamp. The difference from [Backup Before Edit](/boveda/hook-backup-before-edit) (`filename.backup.TIMESTAMP` next to the original): this one groups all the backups into a single folder, which is tidier.

Hook event: `PreToolUse`.

## How to install it

```bash
npx claude-code-templates@latest --hook development-tools/file-backup
```

For Claude:

```
I installed the development-tools/file-backup hook. Edit a file and
show me the contents of `.backups/`.
```

## What you can do with this

### Restore with `.backups/`

If Claude breaks something, you go to `.backups/` and copy the latest timestamp over the original.

### Keep the repo clean

Unlike backups placed next to each file, they all end up grouped in a single folder. Easier to gitignore.

### Audit with ls

`ls -lt .backups/` shows you which files were edited and in what order — handy for timelines.

### Combo with [File Protection](/boveda/hook-file-protection)

File Protection blocks edits to critical files. File Backup acts as a net in case something slips through anyway.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you version everything in git:** git already gives you history. The hook duplicates it.
- **For repos with huge files:** `.backups/` fills up fast.
- **Remember to gitignore:** add `.backups/` to your `.gitignore`.
- **Periodic cleanup:** backups pile up. Consider rotation.

</Callout>
