## Qué es

Override del modelo a `claude-3-5-sonnet-20241022`. Balance entre capability y cost.

Sonnet es el sweet spot para la mayoría de coding work — mejor que Haiku, más barato/rápido que Opus.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `model` |
| Installs | 82 |
| Modelo | claude-3-5-sonnet-20241022 |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting model/use-sonnet
```

Para Claude:

```
Instalé el setting model/use-sonnet. Confirmá que el modelo
activo es Sonnet.
```

## Qué podés hacer con esto

### Balance default

Para mayoría de coding work, Sonnet es la opción correcta.

### Override explicit

Si tu default es Haiku o Opus, este setting fuerza Sonnet.

### Para teams con preferencia común

Si tu team standardiza en Sonnet, este setting lo enforce.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu default ya es Sonnet:** no aporta.
- **Para tasks chicos:** [Use Haiku](/boveda/setting-use-haiku) reduce cost.
- **Para tasks muy complejos:** Opus puede dar mejor output (mirá [Custom Model](/boveda/setting-custom-model)).
- **Sonnet 3.5 vs 4.x:** este setting es 3.5 specific. Si querés version más nueva, [Custom Model](/boveda/setting-custom-model).

</Callout>
