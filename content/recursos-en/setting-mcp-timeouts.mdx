## What it is

Configure timeouts and output limits for MCP server operations:

- `MCP_TIMEOUT: 30000` (30s) — MCP server startup timeout
- `MCP_TOOL_TIMEOUT: 60000` (60s) — timeout per tool execution
- `MAX_MCP_OUTPUT_TOKENS: 50000` — output cap per tool call

Prevents slow MCP servers from hanging your session.

## What you can do with this

### Anti-hang for slow MCPs

If an MCP server has a bug or gets stuck, it won't lock you up indefinitely.

### Output cap

`MAX_MCP_OUTPUT_TOKENS` prevents an MCP from generating MBs of output that eat up your context.

### Tune it for your usage

- For fast systems: lower the timeouts (quicker feedback on hangs)
- For complex MCPs (browser automation, big scrapes): raise the timeouts

### Combo with [Bash Timeouts](/boveda/setting-bash-timeouts)

Bash Timeouts for shell commands. MCP Timeouts for MCPs. They cover every entry point.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you don't use MCPs:** it adds nothing.
- **If your MCPs are fast:** the defaults are enough for you.
- **For MCPs that need minutes:** raise the timeouts carefully so you don't stall the session.

</Callout>
