## Qué es

Statusline focused en Neon (Postgres serverless). Muestra:
- **Project name**
- **Active branch** (Neon supports git-like branching de DBs)
- **Compute state** (active, suspended, etc.)
- **Autoscaling CU range** (compute units)

Usa **solo Neon REST API** — no abre conexiones SQL constantes, por lo que es ligero.

Uno de 3 statuslines Neon de la familia: este (Context), [Dev](/boveda/setting-neon-database-dev), [Resources](/boveda/setting-neon-database-resources).

## Qué podés hacer con esto

### Awareness de branch Neon

Si laburás en branch dev de Neon, el statusline te recuerda — no mandés migrations a main por error.

### Cost awareness rápido

Compute state te dice si estás pagando active o estás suspended. Útil para monitor costos.

### Combo con [Neon Database Dev](/boveda/setting-neon-database-dev)

Context muestra cluster info. Dev muestra runtime info. Combinables (pero uno solo a la vez como statusline).

### Combo con [Neon MCP](/boveda/mcp-neon)

Statusline para context constante. MCP para ops on-demand. Combo natural.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Neon:** no aplica.
- **Si ya tenés otro statusline:** statusline es uno solo, elegí.
- **Solo 26 installs:** poca comunidad. Validá implementación.

</Callout>
