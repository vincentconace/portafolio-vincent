## What it is

**2,329 installs — the second most adopted hook in the aitmpl catalog** (only behind [Dependency Checker](/boveda/hook-dependency-checker)). The reason: it covers the #1 thing you want when AI edits code — that it respects your linter's rules.

After every modification, the hook detects the language and runs the appropriate linter:
- **JS/TS** → ESLint
- **Python** → Pylint
- **Ruby** → RuboCop

Hook event: `PostToolUse`.

## How to install it

```bash
npx claude-code-templates@latest --hook development-tools/lint-on-save
```

Prerequisite: your language's linter installed (`eslint`, `pylint`, `rubocop`).

For Claude:

```
I installed the development-tools/lint-on-save hook. Check that I have
ESLint configured. Then edit a TS file with an obvious error
(unused var) — the lint warning should show up.
```

## What you can do with this

### Catch issues that format doesn't fix

Format fixes whitespace. Lint catches unused imports, magic numbers, complexity issues, security smells.

### Combo with format

Format first ([Format JS](/boveda/hook-format-javascript-files) or [Format Python](/boveda/hook-format-python-files)), lint after. A clean pipeline.

### For teams with strict lint

If your CI rejects PRs with lint errors, this hook warns you before you commit. Catching it locally vs. catching it in CI.

### For autonomous sessions

Long `/goal`: Claude won't introduce lint errors without you noticing.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **No linter configured:** the hook needs a valid config (`.eslintrc.*`, `.pylintrc`, `.rubocop.yml`).
- **If your linter is slow:** every edit triggers lint. For large projects with a heavy linter, noticeable friction.
- **If your workflow uses other linters** (Ruff for Python, Standard for Ruby): check compatibility.
- **For unsupported files:** this hook covers 3 languages. For Go, Rust, etc., other hooks or none.

</Callout>
