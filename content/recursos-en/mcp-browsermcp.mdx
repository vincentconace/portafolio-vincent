## What it is

Browser MCP is one of the base MCPs in the browser_automation catalog. Simpler than [browser-use](/boveda/mcp-browser-use-mcp-server) or [Chrome DevTools](/boveda/mcp-chrome-devtools), its role is to give AI applications basic web capabilities: navigate URLs, fill out forms, interact with elements.

Handy when you need something lighter than browser-use but friendlier than the raw Chrome DevTools Protocol. Its 582 installs make it a recognized option.

## How to install it

```bash
npx claude-code-templates@latest --mcp browser_automation/browsermcp
```

For Claude:

```
Install the browser_automation/browsermcp MCP. Open example.com and give me back
the main text of the page.
```

## What you can do with this

### Basic navigation

Open a URL, wait for it to load, read the content. Tasks like "go to `my-app.com/dashboard` and bring me the cards that show up".

### Fill out simple forms

"Go to the contact form and submit a message with test data". Useful for quick smoke tests.

### Light scraping

For pages without serious anti-bot protection, it's enough. If the page has Cloudflare or captchas, go with [Bright Data](/boveda/mcp-brightdata) instead.

### Post-deploy validation

"Go to the prod URL and verify the home page loads without errors". Lightweight uptime validation + sanity check.

## When NOT to use it

<Callout type="warning" title="Before installing">

- **For complex automations:** [browser-use](/boveda/mcp-browser-use-mcp-server) (1,636 installs) is more mature. [Chrome DevTools](/boveda/mcp-chrome-devtools) (2,685 installs) gives you more control.
- **For serious scraping:** [Apify](/boveda/mcp-apify) or [Bright Data](/boveda/mcp-brightdata).
- **For reproducible E2E tests:** [Playwright Skill](/boveda/plugin-playwright-skill) is better.
- **If you just need an HTTP fetch:** a plain `curl` or `fetch` is simpler. The MCP is for when you need real interaction with the page.

</Callout>
