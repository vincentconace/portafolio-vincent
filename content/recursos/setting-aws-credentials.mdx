## Qué es

Configura cómo Claude Code maneja credentials AWS para usar Claude vía Bedrock (en vez del API direct de Anthropic).

Dos settings clave:
- `awsAuthRefresh` — comando para refresh de auth (ej. `aws sso login --profile myprofile`)
- `awsCredentialExport` — script custom para exportar credentials (`/bin/generate_aws_grant.sh`)

Útil cuando:
- Tu org usa AWS SSO con tokens efímeros
- Vault o STS con rotación automática
- Multi-account con perfiles

```bash
npx claude-code-templates@latest --setting global/aws-credentials
```

Para Claude:

```
Instalé el setting global/aws-credentials. Configurá awsAuthRefresh con
mi profile SSO. Verificá que puedo invocar Claude vía Bedrock con
los creds renovados automaticamente.
```

## Qué podés hacer con esto

### SSO automático

Token SSO expira → setting dispara `aws sso login` → continuar usando Claude.

### Multi-profile

Si tenés varios profiles AWS (dev, staging, prod), configurás el correcto según contexto.

### Vault custom

Si tu org tiene vault propio, `awsCredentialExport` lo conecta.

### Combo con [API Key Helper](/boveda/setting-api-key-helper)

API Key Helper para tokens genéricos. AWS Credentials para Bedrock específico. Pueden coexistir.

## Cuándo NO conviene

<Callout type="warning" title="Antes de habilitar">

- **Si usás Claude direct (no Bedrock):** no aplica.
- **Sin AWS account / SSO:** no tiene caso.
- **Scripts de auth con bugs:** si refresh falla, Claude se queda sin acceso a Bedrock. Test bien.
- **Security:** los scripts pueden ver creds. Asegurate que estén en paths seguros con permisos correctos.
- **Para Bedrock cross-account:** verificá que tu setup soporte el patrón.

</Callout>
