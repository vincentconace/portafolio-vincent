## Qué es

Claude Code por default pregunta antes de ejecutar comandos. Este setting pre-autoriza las git ops comunes:
- `git status`
- `git diff`
- `git add`
- `git commit`
- `git push`
- ...y más (7 ops totales)

Operaciones destructivas siguen requiriendo confirmation. Workflow git fluido sin "permitir / cancelar" cada vez.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting permissions/allow-git-operations
```


## Qué podés hacer con esto

### Workflow git sin interrupciones

Editás, Claude commitea, Claude pushea — sin "permitir" en cada step.

### Para sesiones autónomas

`/goal` que incluye git ops: el setting evita que Claude se trabe pidiendo confirmation.

### Combo con [Conventional Commits Hook](/boveda/hook-conventional-commits)

Setting autoriza commits. Hook valida format. Workflow git automatizado pero seguro.

### Combo con [Prevent Direct Push Hook](/boveda/hook-prevent-direct-push)

Setting autoriza push genérico. Hook bloquea push a branches protegidas. Defense in depth.

## Cuándo NO conviene

<Callout type="warning" title="Antes de habilitar">

- **Para repos críticos:** considerá si querés siempre confirmation.
- **Sin hooks de seguridad:** auto-push sin [Prevent Direct Push](/boveda/hook-prevent-direct-push) puede ser peligroso.
- **Si tu workflow es de review-each-step:** este setting saltea esos pasos.
- **Tokens git con scope amplio:** el setting + tokens amplios = catastrofe potencial.

</Callout>
