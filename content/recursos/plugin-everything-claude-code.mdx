## Qué es

113k stars: es el plugin más estrellado del catálogo aitmpl.com. Por encima de Claude Mem (42k), por encima de Claude HUD (14k), por encima de cualquier marketplace. Cuando una sola pieza junta esa cantidad de respaldo, hay que prestarle atención.

El subtítulo del autor es preciso: "The agent harness performance optimization system". No es un marketplace ni un workflow completo — es un **sistema** que optimiza cómo el agent harness (la capa que conecta el LLM con las tools y el entorno) opera. Trae cinco ejes:

- **Skills** — capacidades específicas
- **Instincts** — comportamientos por default que evitan errores típicos
- **Memory** — persistencia inteligente entre sesiones
- **Security** — guards y validaciones
- **Research-first development** — el patrón de investigar antes de implementar

Funciona en Claude Code, Codex, Opencode, Cursor "and beyond". El autor (affaan-m) tiene una web propia en [ecc.tools](https://ecc.tools) con docs detalladas.

## Qué podés hacer con esto

### Tener un Claude Code que "piense antes de actuar"

El eje **research-first** es probablemente el más impactante: antes de empezar a editar, Claude investiga la zona del código que va a tocar. Menos "implementó algo que no se conecta con el resto" y más "entendió el contexto, después actuó".

### Memoria persistente sin pelearte con el setup

Memoria entre sesiones, igual que [Claude Mem](/boveda/plugin-claude-mem) pero integrada con los otros 4 ejes. Si vas a instalar uno solo, comparalos antes — ambos resuelven el mismo problema con filosofías distintas.

### Instincts que cubren los errores típicos

El concepto de "instincts" es interesante: comportamientos default que Claude aplica sin pedir permiso, basados en lo que la comunidad ya identificó como errores frecuentes. Tipo "antes de borrar un archivo, fijate si está versionado", o "antes de pushear, corre los tests".

### Cross-tool real

Si mañana cambiás de Claude Code a Cursor o Codex, los 5 ejes te siguen. Para equipos heterogéneos esto es valioso.

### Performance optimization

El autor lo llama "performance optimization system" no por casualidad. El sistema reduce iteraciones inútiles, evita re-leer archivos, prioriza skills relevantes, etc. Menos tokens consumidos para el mismo resultado.

## Beneficios reales

- **113k stars** — la comunidad votó masivamente.
- **Cross-tool** — no quedás atado a un harness específico.
- **Filosofía clara** (los 5 ejes) — sabés qué te está dando.
- **Memoria + security + research integrados** — no tenés que combinar 3 plugins.
- **Web propia con docs** ([ecc.tools](https://ecc.tools)) — soporte sólido.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si recién arrancás con Claude Code:** instalar un "sistema completo" sin entender Claude Code primero te puede tapar el aprendizaje básico. Empezá por [Claude Code Setup](/boveda/claude-code-setup) oficial y un par de skills.
- **Solapamiento con `claude-mem`:** los dos cubren memoria. No los instales ambos al toque — elegí.
- **Solapamiento con hooks de seguridad propios:** si ya configuraste hooks de seguridad a mano, los "instincts" pueden duplicar reglas.
- **No es un toolkit, es un sistema opinionado:** vas a heredar las decisiones del autor. Si discrepás, lo vas a tener que tunear.
- **Costo de tokens del research-first:** investigar antes de actuar gasta más. La eficiencia gana a largo plazo, pero la primera corrida es más cara.

</Callout>
