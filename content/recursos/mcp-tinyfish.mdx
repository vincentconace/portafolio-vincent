## Qué es

TinyFish es una plataforma de web agents enfocada en data extraction con dos cosas distintivas:
1. **Real-time progress streaming** — sabés en qué está el agent mientras corre, no solo el resultado final
2. **Stealth browser profiles** — el browser se camufla como tráfico real, no como bot

Pensado para extracción donde otros tools fallan por anti-bot. Comparable con [Browserbase](/boveda/mcp-mcp-server-browserbase) pero con foco más extracción + streaming.

## Qué podés hacer con esto

### Scraping con anti-bot

"Andá a una page que tiene Cloudflare. Con stealth profile, navegá y extraé los listings". Donde browser local fracasa, TinyFish puede pasar.

### Async tasks con progress

"Lanzá la task de scrap de 200 páginas. Mientras corre, mostrame progress cada 30s. Cuando termine, devolveme todos los resultados unificados".

### Multi-page extraction

"Recorre toda la pagination del sitio X y extraé todos los items. Reportame total y errores si los hay".

### Data extraction estructurada

"De esta page de producto, extraé `nombre`, `precio`, `descripción`, `specs`, `reviews count` como JSON".

### Comparado con [Bright Data](/boveda/mcp-brightdata) / [Browserbase](/boveda/mcp-mcp-server-browserbase)

- **Bright Data:** enterprise scale, 60+ tools, más caro
- **Browserbase:** cloud browser + Stagehand, más generalista
- **TinyFish:** focal en extraction + stealth, pricing más accesible probablemente

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para sites sin anti-bot:** [Firecrawl](/boveda/mcp-firecrawl) o un script propio es más simple.
- **Costos:** stealth profiles + cloud browsers cobran. Mirá pricing.
- **Legal:** evadir bot detection puede violar TOS. Responsabilidad legal es tuya.
- **Solo 81 installs:** comunidad probando reducida.
- **Para scrap puntual:** sumar otro servicio puede ser overkill.

</Callout>
