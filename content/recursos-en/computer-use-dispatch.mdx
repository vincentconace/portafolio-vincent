## The combo

Two features that complement each other:

| Feature | What for |
|---------|----------|
| **Computer Use** | Claude controls your computer (mouse, keyboard, screen) |
| **Dispatch** | You give it instructions from your phone |

Combined: **you're on the subway, you tell Claude to process 30 invoices while your Mac mini at home does it on its own**.

---

## What Computer Use is

It gives Claude the ability to:
- ✅ See your screen
- ✅ Move your mouse
- ✅ Click
- ✅ Type
- ✅ Navigate between apps

**You don't need APIs, code, or complicated configurations**. If you can do it with a mouse and keyboard, **so can Claude**.

### Before

You did everything manually: looking up data, copying info, filling out forms, organizing files. **Repetitive and slow**.

### Now

You tell Claude what you need. What takes an assistant 1 hour, **Claude does in minutes**.

See [Computer Use in detail](/boveda/claude-control-remoto) to dive deeper.

---

## Enabling Computer Use — 4 steps

### 1. Download the app

`claude.ai/download` → Claude Desktop for Mac/Windows.

### 2. Open Settings

In the app, go to Settings (gear icon).

### 3. Flip the toggle

Find **"Computer Use"** and enable it.

### 4. Grant permissions

**macOS**: System Preferences → Security & Privacy → Privacy → Accessibility → check Claude.

**Windows**: the system prompts you for permissions on first use.

## What Dispatch is

It lets you **assign tasks to Claude from your phone**. Claude runs them on your computer (which is at home, powered on).

### The Computer Use + Dispatch combo

```
1. You're on the subway / at a café / out for a walk
2. You message Claude from Telegram/Discord/Cowork mobile
3. Claude, on your computer at home:
   - Enables Computer Use
   - Does what you asked (move mouse, fill form, etc.)
4. It reports back when it finishes
```

See [Claude Dispatch](/boveda/claude-dispatch) for the full Dispatch setup.

## Limitations

### Sites with anti-bot protection

CAPTCHAs, 2FA, strict Cloudflare → **Claude stops to ask you**.

### Closed / no-API apps

If it requires a Google login → Claude can't get past 2FA. You need the password saved in a password manager and a specific workflow.

### Latency

Operating the mouse/keyboard is **slower than an API**. If the task has an API available (Gmail MCP, for example), **the API is better**.

### Apps that change their UI

If the app's UI changes (a button moved), Claude **may click where it shouldn't**. For apps that change frequently, consider MCP/API.

---

## Anti-patterns

### 1. Computer Use for everything

If the app has an API, **the API is better**. Use Computer Use for apps **without an API** or as a last resort.

### 2. Critical tasks without supervision

Computer Use can make mistakes with clicks. For critical tasks (sending an important email, making a payment), **review** before auto-confirming.

### 3. No sandbox for testing

If you're going to test a new workflow with Computer Use, **test it in a test account** first. Not in real production.

### 4. Dispatch for urgency

Dispatch latency can be seconds to minutes. For something **urgent and real-time**, it's not the right tool.

### 5. No monitoring

If you leave Claude operating with Computer Use for 1h, **monitor it**. If something breaks, you'll see it quickly.
