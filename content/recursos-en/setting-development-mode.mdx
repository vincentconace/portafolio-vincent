## What it is

An "all-in-one" setting for active development. It combines:
- **Allow:** 18 common operations (git, npm, build, test, lint, fs reads, etc.)
- **Deny:** 4 sensitive paths (.env, secrets, etc.)

Instead of installing [Allow Git Operations](/boveda/setting-allow-git-operations) + [Allow Npm Commands](/boveda/setting-allow-npm-commands) + [Deny Sensitive Files](/boveda/setting-deny-sensitive-files) separately, this one bundles them together.

With 1,983 installs, it ranks as one of the most widely adopted permission settings.

## What you can do with it

### One-command permissions setup

Instead of combining 3 settings, one covers them all.

### Trusted dev environments

If your repo and machine are secure, the balanced preset gives you a frictionless workflow.

### Combined with security hooks

Settings establish a baseline. Hooks ([Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker), [AI Bash Guard](/boveda/hook-ai-bash-guard)) add dynamic layers.

### For autonomous sessions

Long `/goal` runs don't get stuck asking for permission at every step.

## When it's NOT a good fit

<Callout type="warning" title="Before you install">

- **For prod or sensitive repos:** the broad set may be too permissive. A custom config is recommended.
- **If you already have [Allow Git/Npm](/boveda/setting-allow-git-operations) separately:** don't install this on top of it — duplication.
- **If you don't trust your environment:** narrow the scope first.

</Callout>
