## What it is

Before Claude modifies a file with Edit, this hook creates a copy with the format `filename.backup.<timestamp>`. If the edit goes wrong, you restore it by copying the backup back over the file.

It's the basic safety net for sessions where you don't want to rely solely on git (which requires a commit before the change in order to restore) or on Cmd+Z (which only works in the current editor).

Hook event: `PreToolUse` (Edit).

```bash
npx claude-code-templates@latest --hook pre-tool/backup-before-edit
```

```

```

## What you can do with this

### Quick restore without git

Claude edited something and the result is worse than before. You restore with `cp filename.backup.TIMESTAMP filename`. No need to remember which commit to go back to.

### Easy before-and-after

`diff filename.backup.TIMESTAMP filename` shows you exactly what changed. Useful for auditing what Claude did.

### Coverage when git isn't enough

If you're going to make 20 changes in a session, you don't want to commit each one. The backups cover the intermediate points.

### Periodic cleanup

Backups pile up. It's worth having a periodic cleanup script (delete backups older than N days) or adding them to `.gitignore`.

## Prerequisites

- **Claude Code** with hooks enabled
- **Disk space** — backups add up if you edit a lot
- **Edits via the Edit tool**

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you version everything in git:** git already gives you history. The hook duplicates it.
- **For huge files:** backing up 100MB files on every edit fills the disk fast.
- **For clean projects:** the backups create clutter. Add them to `.gitignore` (`*.backup.*`).
- **If you're going to show the directory to others:** timestamped backups look ugly. Consider moving them to `.backups/` (if the implementation allows it).

</Callout>
