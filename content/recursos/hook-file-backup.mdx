## Qué es

Antes de modificar un archivo, el hook crea una copia en una carpeta `.backups/` con timestamp. Diferencia con [Backup Before Edit](/boveda/hook-backup-before-edit) (`filename.backup.TIMESTAMP` al lado del original): este agrupa todos los backups en una sola carpeta, más prolijo.

Hook event: `PreToolUse`.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook development-tools/file-backup
```

Para Claude:

```
Instalé el hook development-tools/file-backup. Editá un archivo y
mostrame el contenido de `.backups/`.
```

## Qué podés hacer con esto

### Restore con `.backups/`

Si Claude rompe algo, vas a `.backups/` y copiás el último timestamp encima del original.

### Mantener el repo limpio

A diferencia de backups al lado de cada archivo, todos quedan agrupados en una carpeta. Más fácil de gitignorear.

### Audit con ls

`ls -lt .backups/` te muestra qué archivos se editaron en qué orden — útil para timelines.

### Combo con [File Protection](/boveda/hook-file-protection)

File Protection bloquea edits a critical files. File Backup hace red si algo igual pasa.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si versionás todo en git:** git ya da history. El hook duplica.
- **Para repos con archivos enormes:** `.backups/` se llena rápido.
- **Acordate de gitignore:** agregá `.backups/` a `.gitignore`.
- **Cleanup periódico:** los backups acumulan. Considerá rotación.

</Callout>
