## What it is

Sentry is the most widely adopted error tracking and APM platform. Issues with stack traces, performance metrics, release tracking, alerts. This MCP connects your account to Claude Code with a specific focus: **human-in-the-loop coding agents and dev workflows**.

What that means in practice: the MCP is designed so that Claude (with you supervising) uses Sentry as context for assisted debugging — reading issues, traces, and error context, and proposing fixes.

## How to install it

```bash
npx claude-code-templates@latest --mcp devtools/sentry
```

Prerequisite: a Sentry account + an API token with the appropriate scopes.

For Claude:

```
I installed the devtools/sentry MCP. Configure the credentials and list
the open issues for project X with > 10 occurrences in the last
24h. For the top 3, read the stack trace and give me the root cause + suggested fix.
```

## What you can do with this

### Issue triage

"List the open issues for the project in production grouped by: critical / unresolved / regression. For the critical ones, give me context and a suggested priority."

### Assisted debugging

"Issue X has 500 occurrences in the last 2 hours. Read the stack trace, request context, and breadcrumbs, and propose what changed and where to fix it. If you need to see the code for function X, paste it."

### Release analysis

"After the v2.1.0 release, which new errors appeared that didn't exist in v2.0? Compare issue rates pre/post-deploy for me."

### Correlation with deploys

"When commit X was deployed, which new errors were introduced? Show me the issues that appeared in that window."

### Performance issues

"List transactions with p95 > 2s in the last day. For the top 3, give me the dominant span and suggest an optimization."

### Resolving issues in bulk

"For all issues with the `legacy` label that haven't reproduced in the last 30 days, group them and propose marking them as resolved."

## Prerequisites

- **Active Sentry account**
- **API token** with the correct scopes
- **Access to the projects** you're going to query

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **Destructive operations:** deleting issues / resolving without context. Add approval hooks.
- **Broad token:** a token with org admin lets Claude see everything. Restrict the scope.
- **Overlap with alternatives:** [Logfire](/boveda/mcp-logfire), [Dynatrace](/boveda/mcp-dynatrace), [Grafana](/boveda/mcp-grafana). Check whether your observability is already covered before adding this.
- **If you use Sentry Self-Hosted on-prem:** verify whether this MCP supports it (some assume Sentry.io).
- **Stack trace privacy:** some stack traces carry PII in variables. Be careful when pasting them to Claude.

</Callout>
