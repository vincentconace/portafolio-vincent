## What it is

Before every `git commit`, the hook scans the staged files looking for secret patterns:

- **Anthropic:** `sk-ant-...`
- **OpenAI:** `sk-...`
- **AWS:** `AKIA...`
- **Stripe:** `sk_live_...`, `pk_live_...`
- **GitHub:** `ghp_...`, `gho_...`
- **Slack:** `xoxb-...`
- ...and 25+ more

If it finds a match, it blocks the commit.

Hook event: `PreToolUse` (Bash with git commit).

## What you can do with it

### Anti-leak before push

Leaking an sk-ant- or AWS key to a public repo just once is enough to drain your account. The hook prevents it.

### For public / open source repos

Especially critical. Once a secret has been committed, it lives in git history forever — even if you later delete it from the file.

### Combo with [Env File Protection](/boveda/hook-env-file-protection)

Env File Protection stops secrets from being written to `.env`. Secret Scanner stops them from being committed in any file. Double safety net.

### Combo with [Security Scanner](/boveda/hook-security-scanner)

Secret Scanner is specific (secrets). Security Scanner is broader (vulns + secrets + more). They can be combined.

## When it's NOT worth it

<Callout type="warning" title="Before you install">

- **For tests with fake keys:** patterns that look real can get flagged. Configure a whitelist.
- **Latency:** the scan adds time to every commit. For very frequent commits, that's friction.
- **If your legitimate workflow hard-codes credentials in specific places:** verify it doesn't break that flow.

</Callout>
