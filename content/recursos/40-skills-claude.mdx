## La idea simple

Una **skill** de Claude es:
- Un archivo `SKILL.md`
- Con frontmatter YAML (`name`, `description`)
- Y un cuerpo con instrucciones
- En una carpeta

Claude **lo lee** automáticamente. Desde ese momento sabe ese trabajo **sin que se lo expliques otra vez**.

> **La diferencia entre Claude prendido y Claude operando tu semana es esa carpeta**.

---

## La pieza clave — `description`

El campo más importante del frontmatter es `description`. Se lee como **gatillo**:

```yaml
description: Úsala cuando el usuario quiera redactar cold email
```

Cada vez que tu prompt encaja, Claude **auto-invoca** la skill.

Por eso buenas skills empiezan con **"Úsala cuando..."** — no es estilo, es contrato con el motor.

---

## Ejemplo mínimo

```markdown
---
name: arquitecto-de-hilos
description: Úsala cuando quieras convertir un tema, artículo o
  idea en un hilo de X.
---

Escribís hilos de X que la gente guarda.

Cuando se invoque:
1. Pedí la idea central si no la dieron
2. Abrí con hook de una línea que pegue
3. 5-9 tweets de cuerpo
4. Closer con CTA
```

Eso es una skill completa. **Construís UNA vez, usás SIEMPRE**.

## Instalación — 3 superficies

### 1. Claude.ai / Desktop / Cowork

1. Crear carpeta con nombre de la skill (ej: `arquitecto-de-hilos`)
2. Adentro pegar `SKILL.md`
3. Comprimir carpeta como `.zip`
4. Settings → Skills → **Add Skill** → cargar zip

### 2. Claude Code (global)

```bash
mkdir -p ~/.claude/skills/[NOMBRE]
# Pegar SKILL.md adentro
```

### 3. Claude Code (por proyecto)

```bash
mkdir -p .claude/skills/[NOMBRE]
# Pegar SKILL.md adentro
```

Aplicá la regla [global vs proyecto](/boveda/skills-global-o-proyecto) — global para transversales, proyecto para específicas.

## Anatomía de skill bien armada

### 1. Frontmatter claro

```yaml
---
name: kebab-case-descriptivo
description: Úsala cuando [TRIGGER ESPECÍFICO]
version: 1.0.0
---
```

### 2. Contexto inicial

Quién eres Claude, qué rol tomás cuando se invoca.

### 3. Pasos numerados

Claude responde mejor a steps que prose.

### 4. Casos edge

Qué hacer si falta info, si input es ambiguo.

### 5. Formato del output

Estructura esperada (bullets, tabla, código, etc.).

Ver [crear claude skills](/boveda/crea-claude-skills) para profundizar en arquitectura.

## Ejemplo de skill — "hook-master"

```markdown
---
name: hook-master
description: Úsala cuando quieras generar hooks para posts, tweets,
  videos o reels.
version: 1.0.0
---

Sos copywriter especializado en hooks que detienen el scroll.

Cuando se invoque:

1. Pedí contexto:
   - ¿Qué plataforma? (X, IG, TikTok, LinkedIn)
   - ¿Qué tema?
   - ¿Quién es la audiencia?
   - ¿Cuál es el objetivo? (engagement, leads, viralidad)

2. Generá 10 hooks. Para cada uno especificá:
   - El hook (texto)
   - Por qué funciona en 1 línea
   - Mejor formato (texto/imagen/video)

3. Variá las técnicas:
   - 2 con pregunta directa
   - 2 con dato contraintuitivo
   - 2 con personal/confesional
   - 2 con contraste (antes/después)
   - 2 con curiosity gap

4. Al final, marcá los 3 con más potencial viral según mi contexto.

NUNCA uses:
- "X cosas que cambiarán tu vida"
- "El secreto que nadie te dice"
- "Vas a quedar sorprendido"
- Otros clichés sobreusados

SIEMPRE que sea posible, citá específicos:
✅ "Llevo 18 meses construyendo SaaS. Solo 2 cosas movieron MRR."
❌ "Aprendí mucho construyendo SaaS"
```

## Anti-patrones

### 1. Instalar 40 sin usar

Cada skill ocupa contexto. Si tenés 40 activas, **Claude se distrae**. Instalá las del **pack que matchea tu rol**, sumá según necesidad.

### 2. Skills con descriptions vagas

```yaml
description: Para escribir bien
```

vs

```yaml
description: Úsala cuando el usuario quiera convertir transcript
  de podcast en post de blog SEO-optimizado.
```

Specificidad = activación correcta.

### 3. Copiar sin entender

Las 40 skills son **baseline**. Aplican principios universales. Pero **adaptá** a tu nicho específico.

### 4. No iterar

Si una skill te tira output regular, **edítala**. Es un archivo, lo modificás. Las skills mejoran con uso.

### 5. Saltarse instalación correcta por superficie

Skills en zip para web ≠ skills en `.claude/skills/` para Code. **Cada superficie tiene su forma**.
