## Qué es

Abre una ventana de browser donde visualizás en tiempo real:
- El **context window** actual (cuánto consume Claude)
- **Subagent activity** y orquestación
- **Tool usage** secuencial
- **Timeline** completo de la sesión

Hook events: `SessionStart`, `PreToolUse`, `PostToolUse`, `Stop`. Captura todo el ciclo.

Comparado con [Claude HUD](/boveda/plugin-claude-hud) (statusline integrado al editor): Timeline es ventana separada con histórico visual completo. HUD es bar inline; Timeline es dashboard.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook monitoring/context-timeline
```

Para Claude:

```
Instalé el hook monitoring/context-timeline. Abrí una sesión nueva y
mostrame qué URL local sirve la visualización.
```

## Qué podés hacer con esto

### Monitor visual de sesiones largas

Para `/goal` o `/loop` autónomos: dejás el browser abierto en otra pantalla y ves qué hace Claude en tiempo real.

### Debugging de orquestación

Si tu sesión usa subagents, ver la timeline ayuda a entender qué dispara qué y cuánto demora cada paso.

### Demos en vivo

Para presentar Claude Code a alguien, la timeline visual es más impactante que mirar la terminal.

### Análisis post-mortem

Después de una sesión, podés revisar el timeline para ver dónde se gastó tiempo o tokens.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si trabajás en pantalla chica:** browser separado + editor compiten por espacio.
- **Para tareas cortas:** no aporta — termina antes de que mires.
- **Si ya usás [Claude HUD](/boveda/plugin-claude-hud):** overlap.
- **Para CI / servers:** Timeline necesita browser. No funciona headless.

</Callout>
