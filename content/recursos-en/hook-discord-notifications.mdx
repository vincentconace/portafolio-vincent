## What it is

The minimal version of Discord notifications. When Claude or a subagent finishes, it sends a message to the channel. No details, no error filters — just "done".

Hook events: `Stop` + `SubagentStop`.

Prerequisite: the `DISCORD_WEBHOOK_URL` env var (you create the webhook from Server Settings → Integrations → Webhooks).

## How to install it

```bash
npx claude-code-templates@latest --hook automation/discord-notifications
```

Set the webhook URL:

```bash
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/.../..."
```

## What you can do with it

### Remote alert when it finishes

It pings your Discord channel when Claude completes a task. Handy when you're away from the editor.

### Subagent completion detection

If your workflow uses subagents, it notifies you as each one finishes. Great for tracking long orchestrations.

### Team activity channel

If several devs run Claude Code, all completions land in the same channel. Instant visibility.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you don't use Discord:** obviously. For Slack or Telegram there are other patterns or [Claude Notifications Go](/boveda/plugin-claude-notifications-go).
- **For very frequent sessions:** notification spam.
- **Without a configured webhook:** it does nothing.
- **If you want to know exactly what happened:** use the Detailed version.

</Callout>
