## Qué es

Variante development-focused del statusline Neon. Muestra:
- **Connection status** (conectado / desconectado)
- **Response time** (latency average)
- **Database activity** (queries en progreso, etc.)

A diferencia de [Neon Context](/boveda/setting-neon-database-context) (project info estático), este es runtime info dinámico.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/neon-database-dev
```

Para Claude:

```
Instalé el setting statusline/neon-database-dev. Configurá creds y
mostrame el dev statusline.
```

## Qué podés hacer con esto

### Health check constante de Neon

Connection status te avisa si la DB cayó.

### Detection de latency issues

Si response time sube, sabés que algo va mal.

### Para daily dev work

Stronger fit que [Neon Context](/boveda/setting-neon-database-context) cuando hacés muchas queries dev.

### Combo con [Neon MCP](/boveda/mcp-neon)

Statusline para awareness constante. MCP para ops puntuales (queries, branches).

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin Neon:** no aplica.
- **Si tu app no toca DB seguido:** menos util que [Neon Resources](/boveda/setting-neon-database-resources) (que cubre costos).
- **Statusline es uno:** elegí entre las 3 variantes.

</Callout>
