## Qué es

Cuando Claude crea un PR via `gh pr create` (GitHub CLI), este hook notifica a Telegram con:
- URL del PR creado
- URL de Vercel preview asociada (si el proyecto está en Vercel)

Hook event: `PostToolUse`.

Útil para teams donde Telegram es el canal y querés review rápido del preview deploy antes de merge.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook automation/telegram-pr-webhook
```

Pre-requisitos: TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, `gh` CLI configurado, proyecto en Vercel para que el preview se detecte.

Para Claude:

```
Instalé el hook automation/telegram-pr-webhook. Configurá las creds.
Pidele a Claude que cree un PR con `gh pr create` — debería llegar
notification con ambas URLs.
```

## Qué podés hacer con esto

### Review rápido cross-platform

Recibís PR + preview en Telegram → abrís preview en mobile → decidís merge.

### Equipo distribuido en Telegram

Notification con links permite que cualquiera del equipo abra el preview sin buscar.

### Para workflow PR-heavy

Si lanzás varios PRs por día, automation del notify ahorra fricción.

### Combo con [Vercel Auto Deploy](/boveda/hook-vercel-auto-deploy)

Vercel auto-deploys. Este hook avisa con la preview lista.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás GitHub CLI ni Vercel:** el hook necesita ambos.
- **Para repos sin previews:** el hook funciona pero menos valor.
- **Si tu canal Telegram tiene mucho ruido:** sumar PR notifications puede saturar.
- **Solo 69 installs:** valida implementación en sandbox.

</Callout>
