## What it is

It's the official plugin marketplace for Claude Code, maintained directly by Anthropic. The 15k stars are no accident: it's the place where the company itself publishes its reference plugins and where it curates third-party integrations that passed its quality filter.

If you just installed Claude Code and want to know where to start without having to evaluate community marketplaces, this is the short answer: install this one first, see what's there, and later maybe add more specific stuff.

119 plugins in total. The useful mental split is: **Anthropic-signed plugins** (~16+) and **verified third-party integrations** (~100+, marked as "external" or "community-managed").

## How to install it

```bash
# 1) Marketplace
/plugin marketplace add anthropics/claude-plugins-official

# 2) Start with the official Anthropic ones + a couple of integrations
/plugin install claude-code-setup@anthropics-claude-plugins-official
/plugin install commit-commands@anthropics-claude-plugins-official
/plugin install pr-review-toolkit@anthropics-claude-plugins-official
/plugin install hookify@anthropics-claude-plugins-official
```

Or `/plugin` → Discover.

For Claude:

```
Install the marketplace anthropics/claude-plugins-official. List only the
Anthropic-signed plugins. Then recommend which third-party integrations
suit me based on my stack [your stack].
```

## What you can do with this

### Base Claude Code setup in 10 minutes

`claude-code-setup` scans your repo and recommends hooks/skills/MCPs specific to that project. `hookify` lets you create the hooks from markdown without writing code. `pr-review-toolkit` covers the output.

### Production-grade frontend without a generic aesthetic

`frontend-design` is specifically designed so that the code it generates doesn't look "made by AI." It's the go-to plugin if you've spent 6 months fighting ugly auto-generated UIs.

### Keep `CLAUDE.md` alive

`claude-md-management` audits your `CLAUDE.md`, captures session learnings, and adds them in. Project memory stops aging on its own.

### Connect real tools without writing MCPs

Asana, Notion, Linear, Slack, GitHub, Sentry, etc. — all verified by Anthropic. This avoids the risk of installing a community MCP that leaks tokens.

### Try Stagehand for browser automation

[Stagehand](https://www.aitmpl.com/plugins/playwright-skill) is one of the most promising approaches to browser automation with LLMs. It's in the marketplace, ready to try.

### Official LSPs (better than guesswork)

12 LSPs signed by Anthropic. Equivalent to [Claude Code LSPs](/boveda/plugin-claude-code-lsps) but with Anthropic's guarantee behind them.

## Real benefits

- **Maintained by the source** — Anthropic releases plugins here first.
- **Serious curation** — third parties pass filters before appearing listed.
- **119 plugins** — covers the most important stuff without chaotic overlap.
- **Frequent updates** — a new feature ships in Claude Code, a plugin that leverages it ships too.
- **It's the ecosystem "default"** — if a guide tells you "install X plugin," it's probably here.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **There's no "install everything":** there are 119, several overlap (`code-review` Anthropic vs CodeRabbit vs autofix-bot). Choose.
- **Some third-party plugins have their own sub-marketplace** (`atomic-agents`, `superpowers`) — installing this one opens another rabbit hole.
- **It doesn't have the more experimental community plugins** — for those you go to `cc-marketplace`, `buildwithclaude`, or `jeremylongshore/claude-code-plugins-plus-skills`.
- **If you want Docker MCPs:** the official marketplace has specific ones (chrome-devtools, playwright). For the full catalog of 199+, [buildwithclaude](/boveda/plugin-buildwithclaude) has `mcp-servers-docker`.

</Callout>
