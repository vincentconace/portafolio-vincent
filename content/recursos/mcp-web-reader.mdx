## Qué es

Web Reader es una implementación de Z.AI para lectura web. Z.AI es una empresa china que provee modelos (GLM) y servicios AI varios. Este MCP usa su infraestructura para hacer reading web optimizado para que LLMs lo procesen.

Es uno de los varios MCPs Z.AI (también están [zai-mcp-server](/boveda/mcp-zai-mcp-server), [web-search-prime](/boveda/mcp-web-search-prime), [zread](/boveda/mcp-zread)). Si tu uso involucra modelos Z.AI o querés alternativa a Western providers, este stack funciona.

## Qué podés hacer con esto

### Lectura web optimizada para LLM

URL → contenido limpio. Mismo concepto que [Firecrawl](/boveda/mcp-firecrawl) y similares, pero usando infraestructura Z.AI.

### Combinación con otros MCPs Z.AI

Si usás varios de Z.AI ([Vision](/boveda/mcp-zai-mcp-server), [Search](/boveda/mcp-web-search-prime), [zread](/boveda/mcp-zread)), tener uno consolidado bajo un mismo provider puede simplificar billing y auth.

### Casos de uso similares a Firecrawl

- Convertir URLs a markdown
- Extraction de contenido
- Anti-hallucination en research
- Read recurrente de páginas

## Pre-requisitos

- **Cuenta Z.AI** activa
- **API key**
- **Conexión a internet**

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si ya usás [Firecrawl](/boveda/mcp-firecrawl) o [Jina AI](/boveda/mcp-jina-ai):** hay overlap. Probá uno antes de adoptar otro.
- **Sin cuenta Z.AI:** no funciona. Si tu jurisdicción tiene restricciones para servicios chinos, validá compliance.
- **148 installs es medio:** menos comunidad probando que Firecrawl (371) o Web Fetch (3,346).
- **Data sensible:** el contenido pasa por servidores Z.AI. Para data altamente regulada, validá su política.

</Callout>
