## What it is

CC Marketplace is Anand Tyagi's original marketplace for Claude Code. It has no single theme: it gathers whatever the community contributes ŌĆö git workflows, code review, PR creation, debugging, Michael Galpert's sub-agents (the famous "studio" agents: rapid-prototyper, ui-designer, growth-hacker, etc.), planning with Galen Ward's `plan` and `explore`, prompt optimization with `lyra`, and more.

In other words: if instead of a themed marketplace ([alirezarezvani](/boveda/5-claude-skills-necesarias) for eng/marketing, [HashiCorp](https://www.aitmpl.com/plugins/agent-skills) for Terraform) you want a drawer full of assorted things contributed by different authors, this is that drawer.

It covers 119 plugins. Most are 1 command or 1 agent each ŌĆö they're small pieces, not monolithic toolkits.

## How to install it

```bash
# 1) Add the marketplace
/plugin marketplace add ananddtyagi/cc-marketplace

# 2) Install the plugins you want (one by one)
/plugin install plan@ananddtyagi-cc-marketplace
/plugin install create-pr@ananddtyagi-cc-marketplace
/plugin install lyra@ananddtyagi-cc-marketplace
```

## What you can do with this

### Build your end-to-end PR flow

`plan` ŌåÆ `explore` ŌåÆ implementation ŌåÆ `code-review` ŌåÆ `create-pr` ŌåÆ `fix-pr` (if comments come in). It's the Explore-Plan-Execute pattern but with curated plugins for each step. A good replacement for "I'll just do it by hand".

### An agent stack for building products

If you work on something agency- or small-product-like, Michael Galpert's agents give you a virtual team: `rapid-prototyper` to get started, `ui-designer` to design, `mobile-app-builder` to package, `growth-hacker` for distribution, `app-store-optimizer` to ship. It's the full agile line.

### Working better with GitHub Issues

`analyze-issue` reads the issue and builds a spec. `fix-github-issue` implements it using the spec. A decent shortcut when your backlog is noisy and you want Claude to do the grunt work of reading and understanding.

### Cross-tool: combine with Codex

The marketplace also includes plugins that play well with Codex (same philosophy as [Claude + Codex as a team](/boveda/claude-codex-equipo)). If you already pay for Codex, this combo extracts more value per dollar.

## Real benefits

- **Author diversity** ŌĆö different voices, different styles. If one doesn't win you over, there are alternatives.
- **Small, specific plugins** ŌĆö you install what you use and discard the rest.
- **Covers day-to-day flows** (commit, PR, review, debug) without trying to be a framework.
- **Not tied to a single domain** ŌĆö works just as well for a solo dev as for a startup team.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you already use another giant marketplace** like `alirezarezvani/claude-skills` or `jeremylongshore/claude-code-plugins-plus-skills`, there's overlap (debug, code-review, PR helpers). Pick one as your main.
- **High heterogeneity = variable quality.** Plugins from well-known authors (Galen Ward, Michael Galpert, Anand) are solid. Others are experiments. Try one before adopting them seriously.
- **Some plugins are redundant with native primitives** ŌĆö `plan` vs Claude Code's native `/plan` mode; `double-check` vs hooks. Check what you already have before duplicating.
- **If you only want 2-3 specific commands**, you can copy them locally without installing the whole marketplace.

</Callout>
