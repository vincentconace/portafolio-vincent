## What it is

Configures how Claude Code handles AWS credentials to use Claude via Bedrock (instead of Anthropic's direct API).

Two key settings:
- `awsAuthRefresh` — command to refresh auth (e.g. `aws sso login --profile myprofile`)
- `awsCredentialExport` — custom script to export credentials (`/bin/generate_aws_grant.sh`)

Useful when:
- Your org uses AWS SSO with ephemeral tokens
- Vault or STS with automatic rotation
- Multi-account with profiles

## How to install it

```bash
npx claude-code-templates@latest --setting global/aws-credentials
```

For Claude:

```
I installed the global/aws-credentials setting. Set up awsAuthRefresh with
my SSO profile. Verify that I can invoke Claude via Bedrock with
the credentials renewed automatically.
```

## What you can do with this

### Automatic SSO

SSO token expires → the setting triggers `aws sso login` → keep using Claude.

### Multi-profile

If you have several AWS profiles (dev, staging, prod), you configure the right one based on context.

### Custom Vault

If your org has its own vault, `awsCredentialExport` connects to it.

### Combo with [API Key Helper](/boveda/setting-api-key-helper)

API Key Helper for generic tokens. AWS Credentials for Bedrock specifically. They can coexist.

## When it's NOT a good fit

<Callout type="warning" title="Before enabling">

- **If you use Claude directly (not Bedrock):** it doesn't apply.
- **Without an AWS account / SSO:** there's no point.
- **Auth scripts with bugs:** if the refresh fails, Claude loses access to Bedrock. Test thoroughly.
- **Security:** the scripts can see credentials. Make sure they live in secure paths with the right permissions.
- **For cross-account Bedrock:** verify that your setup supports the pattern.

</Callout>
