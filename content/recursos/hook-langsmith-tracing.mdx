## Qué es

[LangSmith](https://www.langchain.com/langsmith) es la plataforma de observability para LLM apps de LangChain. Trazas, evals, debugging — está pensado para aplicaciones AI seriamente productivas.

Este hook manda automáticamente traces de las conversaciones Claude Code a LangSmith cuando una sesión termina. Hook event: `Stop`.

Pre-requisitos: `jq` instalado + API key de LangSmith.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook monitoring/langsmith-tracing
```

Pre-requisitos:
- `brew install jq` (macOS) o `sudo apt-get install jq` (Linux)
- LangSmith account + API key

Para Claude:

```
Instalé el hook monitoring/langsmith-tracing. Configurá
LANGSMITH_API_KEY. Hacé una sesión corta y verificá que aparezca el
trace en mi project de LangSmith.
```

## Qué podés hacer con esto

### Observability de uso de Claude Code

LangSmith te muestra qué hacés con Claude: tools usados, latency, tokens. Útil para optimizar workflows.

### Eval de productividad

¿Cuántas sesiones por día? ¿Tareas típicas? ¿Cuáles tardan más? LangSmith te da los charts.

### Compare workflows

Si probás dos approaches (ej. con vs sin un plugin), LangSmith te ayuda a comparar.

### Para teams con LangSmith ya adoptado

Si tu org usa LangSmith para otras LLM apps, sumar Claude Code centraliza la observability.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin cuenta LangSmith:** este hook es LangSmith-only.
- **Sin jq instalado:** el hook falla.
- **Para uso casual de Claude Code:** observability es overkill si solo usás unas horas semana.
- **Privacy:** los traces incluyen conversaciones. Para data sensible, mirá tu policy.
- **Costos de LangSmith:** plan free tiene cap. Para uso volumétrico, vas a pagar.

</Callout>
