## What it is

Performance budgets are limits you define: "the JS bundle can't exceed 200KB gzipped", "LCP can't be greater than 2.5s". If your app breaks them, there's a regression.

This hook automatically monitors:
- **Bundle size** after every build
- **Core Web Vitals** metrics (LCP, FID, CLS)

If you break a budget, it blocks the deploy and shows you what grew.

Hook event: `PostToolUse`.

## How to install it

```bash
npx claude-code-templates@latest --hook performance/performance-budget-guard
```

For Claude:

```
I installed the performance/performance-budget-guard hook. Set up the
initial budgets: max bundle 200KB gzip, max LCP 2.5s, max CLS
0.1.
```

## What you can do with this

### Catch bundle regressions

Claude adds a heavy lib by mistake → bundle grows 50KB → the hook blocks the deploy and tells you which dependency it was.

### Keep Core Web Vitals in check

If your app lives on SEO, you can't tolerate an LCP regression. The hook protects it.

### Combo with [Deployment Health Monitor](/boveda/hook-deployment-health-monitor)

Performance Budget Guard is pre-deploy. Deployment Health Monitor is post-deploy. Together they cover both windows.

### For teams with a UX SLO

If you have a formal performance agreement, the hook is automatic enforcement of the SLO.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your app has no perf SLO:** budgets without a baseline are arbitrary.
- **For internal / admin apps:** Core Web Vitals matter less.
- **For pure backends:** the hook is for frontend perf — it doesn't apply.
- **If you block deploys with this:** make sure you have an emergency override.

</Callout>
