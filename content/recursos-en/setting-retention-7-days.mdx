## What it is

`cleanupPeriodDays: 7` — Claude Code deletes chat transcripts older than 7 days automatically.

For strict privacy or orgs with data minimization policies.

## Summary table

| Field | Value |
| --- | --- |
| aitmpl category | `cleanup` |

| Setting | `cleanupPeriodDays: 7` |

## How to install it

```bash
npx claude-code-templates@latest --setting cleanup/retention-7-days
```

For Claude:

```
I installed the cleanup/retention-7-days setting. Verify that transcripts
>7 days will be deleted on the next cleanup cycle.
```

## What you can do with this

### Strict privacy

Recent sessions only. Old history goes away.

### Data minimization compliance

For GDPR/CCPA, keep only the necessary data. 7 days is aggressive but compliant.

### For projects with NDAs

If you work under an NDA, keeping transcripts minimal reduces risk.

### Opposite of [Retention 90 Days](/boveda/setting-retention-90-days)

90 days = extended history. 7 days = strict privacy. Choose based on your case.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you value extensive history:** old transcripts can be useful as a reference.
- **If your workflow is long projects:** transcripts older than 7 days may hold relevant context.
- **Combined with auditing:** if your org requires long audit logs, 7 days is too little.

</Callout>
