## Qué es

[Railway](https://railway.app) es uno de los hostings modernos más usados para apps con services + databases: deploy en minutos, support para varios runtimes, networking automático entre services. La opción Heroku-like sin las limitaciones de Heroku.

Este MCP enchufa Railway a Claude Code. Operás projects, services, deployments y environment vars sin abrir el dashboard.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/railway
```

Pre-requisito: cuenta Railway + Personal API token desde [railway.app/account/tokens](https://railway.app/account/tokens).

Para Claude:

```
Instalé el MCP devtools/railway. Listame mis projects y para el project
X, mostrame los services con su status y URL.
```

## Qué podés hacer con esto

### Triage de deploys

"Listame los services con deploys fallidos en las últimas 24 horas. Para cada uno, leé los logs y dame root cause".

### Gestión de env vars

"En el service `api-prod`, agregame estas env vars: `STRIPE_KEY=sk_...`, `SENTRY_DSN=https://...`. Reinicia el service después de aplicar".

### Promoción entre environments

"Comparame las env vars entre `staging` y `production`. Identificá las que faltan en prod. Para las que claramente deben promoverse (no secrets de staging), copialas".

### Rollback rápido

"En el service X, mostrame los últimos 10 deploys con timestamp y status. Rollback al penúltimo (el actual está roto)".

### Setup de service nuevo

"Creame un service nuevo `worker` con: source desde `~/my-repo` branch `main`, runtime Node.js, env vars X/Y/Z, autodeploy on push".

### Database management

Railway maneja Postgres, Redis, MongoDB, MySQL como services. El MCP puede operar (crear DB, conectar a otra service, etc.).

### Combo con [All Deploy](/boveda/all-deploy)

All Deploy puede usar Railway como target automáticamente. El MCP da control manual cuando lo necesitás.

## Pre-requisitos

- **Cuenta Railway** activa
- **API token** con permisos sobre los projects relevantes
- **Project ya creado** (o el MCP puede crearlo)

## Cuándo NO conviene

<Callout type="danger" title="Antes de habilitar operaciones en producción">

- **Operaciones destructivas:** delete service, delete deployment, restart prod sin warning. Hooks de aprobación.
- **Env vars sensibles:** cuando manejás secrets via MCP, terminan visibles en la sesión. Validá tu sec policy.
- **Costos:** Railway cobra por compute / storage. Aprovisionar muchos services genera bills. Mirá tu plan.
- **Si usás otro hosting** (Vercel, Render, Fly, AWS), este MCP no aplica.
- **Token amplio:** el token deja a Claude operar todo lo que tu account ve. Restringí scope si Railway lo permite.

</Callout>
