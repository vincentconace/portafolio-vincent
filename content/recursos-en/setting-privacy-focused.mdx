## What it is

An all-in-one privacy bundle. It disables 5 things with 5 env vars:
- `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC: 1` — no non-essential traffic
- `DISABLE_TELEMETRY: 1` — no usage analytics
- `DISABLE_ERROR_REPORTING: 1` — no error reports to Anthropic
- `DISABLE_BUG_COMMAND: 1` — no bug reporting CLI
- `DISABLE_AUTOUPDATER: 1` — no automatic updates

It replaces having to install [Disable Telemetry](/boveda/setting-disable-telemetry) separately and configure the rest by hand.

## What you can do with this

### Maximum privacy in one command

Instead of configuring 5 vars manually, a single install.

### For regulated data

You work with healthcare/finance/legal: this setting is the baseline.

### For airgapped or offline-priority

Auto-updater off + non-essential traffic off = more predictable behavior offline.

### Combo with [Deny Sensitive Files](/boveda/setting-deny-sensitive-files)

Privacy Focused for the network. Deny Sensitive for the filesystem. They cover both angles.

### Combo with [Custom Telemetry](/boveda/setting-custom-telemetry)

If you do want to send telemetry but to your own endpoint (not Anthropic), combine them.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **DISABLE_AUTOUPDATER:** with no auto updates, your Claude Code goes stale. You'll have to update it manually.
- **DISABLE_BUG_COMMAND:** you can't report bugs via CLI. If you find a bug, it's manual.
- **If you want to help with feature improvements:** telemetry helps. It's a trade-off.
- **If your org does NOT require strict privacy:** unnecessary overhead.

</Callout>
