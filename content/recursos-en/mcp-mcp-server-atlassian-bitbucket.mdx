## What it is

For teams that live in Atlassian (Jira + Confluence + Bitbucket) instead of GitHub or GitLab, this MCP gives access to Bitbucket Cloud. Same model as the [GitHub MCP](/boveda/mcp-github-integration), but pointed at the Atlassian stack.

Implemented in Node/TypeScript, it connects to the Bitbucket Cloud REST API so that Claude (or other LLMs like Cursor AI) can operate safely on repos, PRs and branches.

## How to install it

```bash
npx claude-code-templates@latest --mcp devtools/mcp-server-atlassian-bitbucket
```

Prerequisites: a Bitbucket Cloud account + an App Password with the correct scopes (repository:read/write, pullrequest:read/write).

For Claude:

```
I installed the MCP devtools/mcp-server-atlassian-bitbucket. Set up the
credentials (workspace, app password) and list my repos in workspace X
with their active branches.
```

## What you can do with it

### PR triage

"List the open PRs across the repos in workspace X. For each one, give me days since it was opened, the author, and whether it has pending comments." Handy for morning triage.

### Assisted code review

"Open PR #234 in repo Y. Read the diff and give me a review file by file: bugs, edge cases. If it's OK, leave an approval comment with a summary."

### Cross-repo search

"Across all my repos, find files that use the legacy function `<X>` and put together a migration plan." Useful when a company has 50+ repos.

### Full PR workflow

"Create branch `feature/X` from main, commit the current change, open a PR with title Y and description Z, assign reviewers W." The whole flow.

### Release / tag management

"Cut a v1.2.0 tag with an autogenerated changelog from the last 30 commits, grouped by type."

### Pipeline inspection

If you use Bitbucket Pipelines, the MCP can query build status (depends on the implementation — check the config).

## Prerequisites

- **Active Bitbucket Cloud account**
- **App Password** with the correct scopes (Settings → App Passwords)
- **Workspace identifier**
- **Access to the repos** you're going to query

## When it's NOT a good fit

<Callout type="danger" title="Before enabling writes on important repos">

- **Destructive operations:** force-push, delete branch, close PR. ALWAYS use approval hooks on production repos.
- **Broad app password scopes:** the password inherits whatever your user can do. Restrict scopes to the minimum.
- **Bitbucket Server (on-prem):** this MCP is Bitbucket Cloud-only. For Server there are other implementations, or none.
- **Auditing:** everything Claude does is tied to your user. Logs record it.
- **Rate limits:** the API has quotas. Bulk operations can temporarily block you.

</Callout>
