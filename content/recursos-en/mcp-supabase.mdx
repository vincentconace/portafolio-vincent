## What it is

Supabase is the most widely adopted BaaS in the open-source ecosystem: managed Postgres + Auth + Storage + Realtime + Edge Functions, all with solid SDKs. If your stack is Next/React/Vue with a serverless backend, there's a good chance you're on Supabase.

This MCP connects your Claude Code to Supabase. Its 2,815 installs make it the second most adopted database MCP, right behind [PostgreSQL Integration](/boveda/mcp-postgresql-integration).

It covers:
- **Postgres queries** and schema management
- **Auth** (users, roles, sessions)
- **Storage** (buckets, files)
- **Realtime** (channels, subscriptions)
- **Edge Functions** (deploy, logs)
- **RPC** (exposed Postgres functions)

## How to install

```bash
npx claude-code-templates@latest --mcp database/supabase
```

Prerequisites: a Supabase project + URL + keys (anon for public reads, service_role for admin).

For Claude:

```
I installed the database/supabase MCP. Configure SUPABASE_URL +
SUPABASE_SERVICE_KEY. List the tables in the public schema, the storage
buckets and the deployed edge functions.
```

## What you can do with this

### Serious Postgres on Supabase

Like the [PostgreSQL Integration MCP](/boveda/mcp-postgresql-integration) but with awareness of the extensions Supabase enables by default (`pgvector`, `postgis`, `pgcrypto`, etc.).

### Assisted Auth ops

"List the users who signed up in the last 7 days, grouped by provider (email vs OAuth). Identify users who haven't confirmed their email."

### Storage management

"In the `avatars` bucket, list files without a valid owner_id (orphans). Suggest a cleanup."

### Edge Functions

"Show me the logs for the `webhook-handler` edge function from the last 24h. Identify recurring errors."

### Realtime channels

"List active channels with their subscribers. If any has stuck subscriptions, give me the broker info."

### RPC functions

"List the Postgres functions exposed as RPC. For each one, give me its signature and permissions."

### Combo with [PagoKit Agent](/boveda/agente-pagokit)

PagoKit can build out your payments setup on Supabase. This MCP lets you operate it afterward.

## Prerequisites

- **An active Supabase project** (the Free tier is enough to get started)
- **Service role key** (for admin ops) and/or anon key (public reads)
- **Network access** to your project

## When it's NOT a good fit

<Callout type="danger" title="CRITICAL — the service_role key has full power">

- **The service role key bypasses RLS:** you can read/write everything. If you share that key in poorly protected Claude sessions, it's a serious attack vector.
- **Destructive operations:** `delete user`, `drop bucket`, `delete table`. ALWAYS use approval hooks.
- **For Auth with real users:** a prompt that alters production users' passwords is catastrophic. Restrict the scope.
- **If your stack isn't Supabase:** it doesn't apply. For generic Postgres, use [PostgreSQL Integration](/boveda/mcp-postgresql-integration).
- **Costs:** Supabase has tiers. Bulk operations (storage egress, heavy queries) can generate a bill.

</Callout>
