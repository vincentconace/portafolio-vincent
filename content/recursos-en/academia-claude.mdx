## What is Anthropic Learn

The official academy from Anthropic — the company that makes Claude. **Free** courses, some with an **official certificate**. URL: `anthropic.com/learn` (courses hosted on `anthropic.skilljar.com`).

Designed for all levels. **You don't need to know how to code** to start the basics.

| Feature | Detail |
|----------------|---------|
| Cost | Free |
| Certificates | Yes (on select courses) |
| Credit card | Not required |
| Levels | Beginner / Intermediate / Advanced |
| Languages | Mostly English (some Spanish subtitles) |

---

## The 2 key courses to start with

### Claude 101

> Foundational course. Learn what Claude is, how it works, and how to use it day to day.

URL: `anthropic.skilljar.com/claude-101`

Duration: ~2-3h. **Worth it even if you already use Claude**. There are nuances you're probably not taking advantage of (extended thinking, tool use, prompt structure).

### Claude Code in Action

> Hands-on Claude Code course. Use it as a development copilot to build, debug, and automate.

URL: `anthropic.skilljar.com/claude-code-in-action`

Duration: ~4-6h. **Worth it if you use Claude Code heavily**. It covers flows the official docs don't detail as much.

## Paths by level

### Beginner — "I've never used AI seriously"

```
1. Claude 101
2. AI Fluency
3. Claude for Personal
```

Total time: ~10h spread across 2-3 weeks. By the end:
- You understand how an LLM works under the hood
- You can write decent prompts
- You have use cases for your daily life

### Intermediate — "I use Claude but I want more"

```
1. Claude for Work
2. Skills
3. Claude Code in Action
```

Time: ~15h. By the end:
- You know how to bring Claude into a professional workflow
- You understand the skills system (not just generic plugins)
- Your Claude Code usage becomes 2-3x more efficient

### Advanced — "I want to build serious things"

```
1. Build with Claude
2. APIs & SDKs
3. MCP
4. Agents
```

Time: ~25h + practice. By the end:
- You can build apps with the API
- You understand when to use a prompt vs. a tool vs. an agent
- You know how to design multi-step agents with MCPs

## What's good and what's not

### Pros

- **Official from Anthropic** — correct, up-to-date info
- **Free, no gimmicks** — no hidden upsell
- **Serious certificates** — they count on a resume/LinkedIn
- **Real practice** — hands-on exercises, not just theory
- **Updated** — kept current with releases (Sonnet, Opus, new features)

### Honest cons

- Mostly in **English** — subtitles yes, dubbing no
- Some courses assume a **specific stack** (Python for APIs)
- **No substitute for experience** — completing everything won't make you a senior
- Less active community than paid courses (Udemy/Coursera have livelier Q&A)

## The path I recommend (in order)

If you had to make **a single balanced pass**:

```
Week 1-2: Claude 101 (free, 2-3h)
Week 3: Prompt Engineering course (free, 4h)
Week 4-5: Claude Code in Action (free, 4-6h)
Week 6: MCP course (free, 3h)
Week 7-8: Agents course + real practice (free, 5h + practice)
```

Total: ~20h over 2 months. You come out with a solid baseline to work professionally with Claude.

## Common mistakes when taking the courses

### 1. Jumping from one to the next without practicing

The temptation: "I'll do all 10 first, then practice." **Mistake**. Do 1, practice for a week, do 2, practice. Retention without practice is ~10%.

### 2. Skipping Claude 101 "because I already use Claude"

It's surprising how much 101 covers that **you didn't know**. Even if you've used Claude for months, the refresher on fundamentals is worth it.

### 3. Doing courses instead of projects

Doing 8 courses without shipping a project = waste. **2 courses + 1 real project** > 6 courses with no practice.

### 4. Ignoring the exercises

The courses come with hands-on work. If you skip it "to finish faster," **you lose 70% of the value**.

### 5. Not downloading the certificate

If the course offers one, **download it**. It's a signal of real initiative on a resume/LinkedIn. It's not a university degree, but it helps.
