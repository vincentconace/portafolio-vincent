## What it is

After every Edit to a Python file (`.py`), the hook runs `black` automatically. Hook event: `PostToolUse`.

`black` is "the uncompromising Python code formatter": opinionated, non-configurable. Your code ends up PEP-8 + black style, no discussion.

1,036 installs make it the top in the `post-tool` category — one of the most adopted hooks in the catalog.

## How to install it

```bash
npx claude-code-templates@latest --hook post-tool/format-python-files
```

Prerequisite: `black` installed (`pip install black` or `pipx install black`).

```

```

## What you can do with this

### Automatic PEP-8

It doesn't matter how Claude indents or places spaces. After the edit, black reformats it according to convention.

### Noise-free diffs

No formatting noise in `git diff`. You only see semantic changes.

### Combo with [Lint On Save](/boveda/hook-lint-on-save)

Lint runs after format. Catch issues that format doesn't cover (unused imports, complexity, etc.).

### For shared projects

If your team uses black as the standard, this hook guarantees Claude respects it without you having to remember to ask.

### For autonomous sessions

Long `/goal` or `/loop` runs: the code stays well formatted without your intervention.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **Without black installed:** the hook fails.
- **If your team uses another formatter** (autopep8, yapf, ruff): this hook is black-only.
- **For projects with a different line length:** black default is 88. If you need another, configure it in `pyproject.toml`.
- **For non-Python files:** this hook is .py-only. For JS use [Format JavaScript Files](/boveda/hook-format-javascript-files).

</Callout>
