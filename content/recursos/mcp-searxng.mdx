## Qué es

[SearXNG](https://searxng.org) es un metasearch engine open source: hacés una búsqueda, SearXNG la dispara contra varios engines (Google, Bing, DuckDuckGo, Wikipedia, Reddit, GitHub, etc.) y te devuelve resultados unificados — sin tracking, sin perfilarte, sin cookies.

Este MCP enchufa SearXNG a Claude Code. Le pedís a Claude que busque algo y consulta tu instancia de SearXNG (self-hosted o pública) en vez de un engine específico.

La ventaja: no le mandás tus queries a Google ni a OpenAI; van a tu SearXNG. Privacy-first.

## Qué podés hacer con esto

### Búsqueda sin tracking

"Buscame `<query>` sin que mi search history quede registrada en Google ni en otros engines". Cero perfilamiento.

### Multi-engine en un query

SearXNG consulta varios engines a la vez y combina resultados. Útil cuando un solo engine no tiene cobertura completa.

### Filtros y categorías

SearXNG permite filtrar por categoría (general, imágenes, videos, news, science, files). El MCP debería respetar esos filtros.

### Self-hosted para teams

Si tu org tiene una instancia SearXNG self-hosted, este MCP le da uso AI. Búsqueda interna privacy-first.

### Combo con [Jina AI](/boveda/mcp-jina-ai)

SearXNG para search; Jina para grounding/fact-checking. Privacy-first todo el flow.

## Pre-requisitos

- **URL de SearXNG** (self-host o instancia pública)
- **Para self-host:** Docker o Python + las deps
- **Para uso productivo:** considerá self-host (las públicas pueden ser bloqueadas por engines)

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin SearXNG corriendo:** el MCP necesita una instancia. Setear self-host es overhead si solo querés búsquedas casuales.
- **Para search con APIs estructuradas:** SearXNG devuelve resultados estilo browser. Para data API (Google Search JSON, Bing API), otros MCPs.
- **Solo 17 installs:** comunidad probando reducida. Verificá tools disponibles en el JSON.
- **Latencia:** consultar varios engines suma tiempo. Para queries instantáneas, single-engine puede ser más rápido.
- **Engines pueden bloquear:** instancias públicas de SearXNG suelen ser detectadas y bloqueadas. Self-host con rotation es la opción seria.

</Callout>
