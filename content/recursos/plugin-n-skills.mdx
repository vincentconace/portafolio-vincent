## Qué es

N-Skills es un marketplace pequeño y curado de Numman Ali. Solo 5 plugins, pero todos pasaron su filtro personal. La filosofía es la opuesta a los marketplaces gigantes de 100+: menos es más, cada plugin tiene un propósito claro y vale la pena su slot.

Funciona cross-tool: Claude Code, Codex y openskills. Si rotás entre herramientas, N-Skills te sigue.

Los 5 plugins:
- `zai-cli` — Z.AI: vision, search, reader y exploración de GitHub vía MCP
- `gastown` — orquestador multi-agent con vocabulario propio (convoys, polecats, rigs, slinging work)
- `dev-browser` — browser automation con persistent page state (de Sawyer Hood)
- `orchestration` — coordinación multi-agent para tareas que se descomponen en paralelo
- `open-source-maintainer` — mantenimiento end-to-end de repos open-source

## Cómo se instala

```bash
# 1) Marketplace
/plugin marketplace add numman-ali/n-skills

# 2) Plugins según uso
/plugin install dev-browser@numman-ali-n-skills
/plugin install orchestration@numman-ali-n-skills
/plugin install open-source-maintainer@numman-ali-n-skills
```

## Qué podés hacer con esto

### Browser automation con estado persistente

`dev-browser` mantiene el estado entre llamadas. Logueás una vez, hacés varias acciones — el browser no se reinicia entre cada una. Diferencia clave con [Playwright Skill](/boveda/plugin-playwright-skill) que es model-invoked sin estado mantenido.

### Mantener un repo open-source sin volverte loco

`open-source-maintainer` cubre los cuatro dolores típicos: triage de issues nuevos, review de PRs entrantes, análisis de actividad de contributors, generación de reportes de salud del repo. Si mantenés un OSS solo, vale el slot.

### Orquestación multi-agent seria

`orchestration` está diseñado específicamente para tareas que se descomponen en subtareas paralelas. Cuando una sola sesión de Claude no alcanza porque hay 8 cosas a hacer al mismo tiempo, este plugin las coordina.

### Vocabulario propio con Gastown

`gastown` es interesante: el autor armó un orquestador con palabras propias ("convoys", "polecats", "rigs", "slinging work"). El uso es: vos hablás en ese vocabulario y Claude entiende qué orquestar. Suena gimmicky pero ayuda cuando querés activar comportamientos específicos sin escribir prompts largos.

### Z.AI como tooling adicional

`zai-cli` te da acceso a Z.AI (vision, search, reader) vía MCP. Útil si querés capacidades que el stack Claude/Codex no cubre.

## Beneficios reales

- **Curación rigurosa** — solo 5 plugins, todos con propósito claro.
- **Cross-tool** — los plugins andan en Claude Code, Codex y openskills.
- **Dev-browser con estado persistente** — único en el catálogo.
- **`open-source-maintainer`** — pocos plugins se enfocan en mantener OSS.
- **No te abruma** con 200 opciones.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si querés un marketplace que cubra todo:** N-Skills son 5 plugins focales. Para variedad, mirá [buildwithclaude](/boveda/plugin-buildwithclaude) o [Claude Plugins Official](/boveda/plugin-claude-plugins-official).
- **`gastown` requiere aprender vocabulario propio:** si no querés invertir en eso, te queda como ornamento.
- **`zai-cli` necesita cuenta Z.AI:** sin eso, el plugin no sirve.
- **Para browser automation sin estado mantenido:** [Playwright Skill](/boveda/plugin-playwright-skill) puede ser más simple.

</Callout>
