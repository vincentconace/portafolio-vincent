## Qué es

Statusline + alert system para Vercel. Cuando detecta deployment failures o build issues:
- **Statusline** muestra error state
- **Desktop notification** automatic
- **Tracking** persistido para análisis

Anti "el deploy falló hace 30 min y recién me entero".

## Qué podés hacer con esto

### Catch failures inmediato

Mientras codeás, deploy falla → desktop notification → ves a fixearlo.

### Para sesiones largas con auto-deploys

Si tenés auto-deploy + sesión productiva, este setting catch los problems sin que rompas focus.

### Combo con [Vercel Deployment Monitor](/boveda/setting-vercel-deployment-monitor)

Deployment Monitor para status normal. Error Alert para anomalies. Pero son alternativos (un statusline).

### Combo con [Deployment Health Monitor Hook](/boveda/hook-deployment-health-monitor)

Setting cubre statusline. Hook cubre execution-time. Doble red.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin Vercel:** no aplica.
- **Si desactivaste notifications del OS:** el alert no llega.
- **Para deploys infrequent:** less util.

</Callout>
