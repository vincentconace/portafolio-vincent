## What it is

[Microsoft Clarity](https://clarity.microsoft.com) is probably the best free surprise in the analytics stack: heatmaps, session recordings (videos of how real users use your app), rage click and dead click insights, all for free. It competes directly with Hotjar / FullStory, which charge per session.

This MCP connects it to Claude Code. You ask Claude to query Clarity and analyze behavior patterns without opening the dashboard.

## How to install

```bash
npx claude-code-templates@latest --mcp devtools/microsoft-clarity
```

Prerequisites:
- Account at [clarity.microsoft.com](https://clarity.microsoft.com) (free)
- Project ID + API token

For Claude:

```
I installed the devtools/microsoft-clarity MCP. Configure the credentials and
show me the heatmaps for the home page from the last week. Identify the
3 elements with the most rage clicks.
```

## What you can do with it

### Rage click detection

"In the last 7 days, which elements generated the most rage clicks (frustrated users clicking repeatedly)? They're probably the ones breaking the UX." A super actionable insight.

### Heatmaps by page

"For the pricing page, give me the scroll heatmap: how far down do visitors get? Is there a cliff where they drop off?"

### Filtered session recordings

"Show me 5 sessions of users who reached checkout but abandoned before paying. For each one, identify when they dropped off and why."

### Dead clicks

"Detect visual elements that look clickable but don't respond (dead clicks). List the top 10." Catch low-hanging UX bugs.

### Sync with cross-source analysis

Combined with [Google Workspace](/boveda/mcp-google-workspace) or data in sheets, you can cross-reference Clarity behavior with user cohorts or marketing segmentation.

### Priority table

"For the 10 most frequent rage clicks, assign an estimated fix effort and estimated impact. Suggest which one to tackle first."

## Prerequisites

- **Microsoft Clarity account** (free, no paid tier)
- **Clarity tracking installed** on your site (a JS snippet)
- **API token / project ID**

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you don't have Clarity installed on your site:** without the tracking, there's no data. That's the first step.
- **GDPR compliance:** Clarity has its own tracking policy. Validate consent on your site if your audience includes the EU.
- **For complete web analytics** (funnels, attribution): Clarity is session replay + heatmaps. For that, [Google Analytics 4](/boveda/mcp-google-workspace) or [Plausible](https://plausible.io) are complementary.
- **For mobile apps:** Clarity has an iOS/Android SDK but it's less mature. Check whether your case applies.
- **Privacy:** session recordings see everything. Configure masking for sensitive fields (password, credit card) — Clarity allows it.

</Callout>
