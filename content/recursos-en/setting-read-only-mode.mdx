## What it is

A setting that restricts Claude Code entirely to read-only operations:
- **Allow:** 4 ops (Read, Grep, Glob, listings)
- **Deny:** 5 ops (Write, Edit, Bash, MultiEdit, etc.)

Result: Claude can analyze but can't modify anything.

Ideal for:
- **Code review** of other people's repos
- **Auditing** codebases
- **Exploring** untrusted repos (cloned without knowing what's inside)
- **Analysis** with no risk of side effects

## How to install it

```bash
npx claude-code-templates@latest --setting permissions/read-only-mode
```

For Claude:

```
I installed the permissions/read-only-mode setting. Confirm that you can't
edit files or run commands. Read only.
```

## What you can do with this

### Safe code review

You clone a third-party repo and want to analyze it without risk: read-only protects you.

### For sandbox / shared envs

On shared machines or temporary sandboxes, read-only prevents side effects.

### For security audits

You want to explore a suspicious repo without Claude accidentally running something.

### Anti vs [Development Mode](/boveda/setting-development-mode)

Dev Mode is a broad allow + small deny. Read Only is the opposite: broad deny.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you want Claude to implement things:** read-only blocks writes. You're left with analysis only.
- **For a productive workflow:** [Development Mode](/boveda/setting-development-mode) or granular allow.
- **If you need to run tests:** read-only blocks Bash too.

</Callout>
