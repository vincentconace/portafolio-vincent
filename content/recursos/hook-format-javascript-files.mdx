## Qué es

Después de que Claude edita un archivo `.js`, `.jsx`, `.ts`, `.tsx`, este hook corre prettier sobre él automáticamente. Hook event: `PostToolUse`.

Resultado: el código siempre queda formateado según tu `.prettierrc`. No más diffs ruidosos por whitespace o comma trailing.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook post-tool/format-javascript-files
```

Pre-requisito: `prettier` instalado (`npm i -D prettier` o global).

Para Claude:

```
Instalé el hook post-tool/format-javascript-files. Probá: editá un
archivo TS con formato inconsistente — debería quedar formateado
después del edit.
```

## Qué podés hacer con esto

### Format consistente automático

Tu `.prettierrc` se aplica siempre. No tenés que pedirle a Claude que respete formato — el hook lo hace después.

### Diffs limpios

Las modificaciones que Claude haga van a quedar formateadas. Tus diffs (`git diff --staged`) quedan focal en cambios reales, no en whitespace.

### Combo con [Lint On Save](/boveda/hook-lint-on-save)

Format primero, lint después. El lint detecta lo que el format no puede arreglar.

### Combo con [Auto Git Add](/boveda/hook-auto-git-add)

Auto Git Add stagea las modificaciones. Format en este hook también afecta al stage. Combinados, tu workflow es: edit → format → stage automático.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin prettier instalado:** el hook falla silently.
- **Sin .prettierrc en el repo:** prettier usa defaults que pueden no coincidir con tu equipo.
- **Si tu equipo no usa prettier:** alineá antes de imponer.
- **Para archivos no-JS/TS:** este hook es específico. Para Python usá [Format Python Files](/boveda/hook-format-python-files).

</Callout>
