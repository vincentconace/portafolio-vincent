## Qué es

Después de cada uso de Edit tool, el hook escribe una entrada a `.claude/edit-log.txt` del proyecto con timestamp + path del archivo modificado. Hook event: `PostToolUse`.

El log es **project-local** (no global) — queda en `.claude/` dentro del proyecto. Si trabajás múltiples proyectos, cada uno tiene su trail.

## Qué podés hacer con esto

### Tracking cross-sesión por proyecto

Cada proyecto tiene su `edit-log.txt`. Útil cuando rotás entre varios y querés ver actividad reciente sin abrir git log.

### Versionar el audit (o no)

Podés decidir si commiteás el log (para que el equipo lo vea) o lo ignorás en `.gitignore`.

### Comparado con [Change Tracker](/boveda/hook-change-tracker)

Change Tracker es más popular (1,127 installs). Ambos hacen tracking simple. Cualquiera sirve — elegí según preferencia de path/formato.
