## What it is

For teams doing spec-driven development: before editing production code, there should be a `.spec.md` documenting the change. This hook checks that a recent one exists (default: last 14 days) and warns if there isn't.

Hook event: `PreToolUse`.

It fits the philosophy of [Spec Kit](/boveda/spec-kit) and similar workflows.

## How to install it

```bash
npx claude-code-templates@latest --hook quality-gates/plan-gate
```

For Claude:

```
I installed the quality-gates/plan-gate hook. Show me the existing
.spec.md files and which ones fall within the last 14 days. If I edit
prod code without a recent spec, it should warn.
```

## What you can do with it

### Enforce spec-first development

Any change to prod requires an approved spec. The hook enforces it without you having to remember.

### Combo with [Spec Kit](/boveda/spec-kit)

Spec Kit generates and maintains specs. Plan Gate enforces that they exist. A natural combo.

### For enterprise teams

Compliance and traceability: every change to prod has its spec documented and auditable.

### For autonomous sessions

Claude with `/goal` won't touch prod without a spec. A safety net for autonomous workflows.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your team doesn't do spec-driven development:** the hook adds friction.
- **For personal / experimental repos:** specs are overhead.
- **If your definition of "prod code" isn't clear:** the hook can be ambiguous.
- **Without Spec Kit or another source of specs:** you'll be creating .spec.md by hand every time.

</Callout>
