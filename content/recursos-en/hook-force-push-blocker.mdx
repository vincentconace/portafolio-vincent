## What it is

A hook that detects any `git push --force`, `-f`, or `--force-with-lease` and blocks execution. It covers the common variations. Hook event: `PreToolUse` on Bash.

It addresses a concrete risk: Claude resolves a conflict with a force push, you overwrite remote history, and you lose other people's commits. Catastrophic if it happens on main or a shared branch.


## What you can do with it

### Anti-catastrophe on shared branches

When you work on shared branches, force-push is dangerous. The hook prevents the accident.

### For repos with branch protection

Even if your GitHub/GitLab has branch protection, the hook acts locally before reaching the server.

### For teams new to git

Devs just learning git may reach for force-push out of desperation. The hook protects that moment.

### Combo with [Conventional Commits](/boveda/hook-conventional-commits)

Conventional Commits enforces format. Force Push Blocker enforces safety. Combined, your git workflow ends up decent by default.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you work on disposable personal branches:** force-push is legitimate. The hook adds friction.
- **If your workflow includes interactive rebases on feature branches:** after the rebase you need force-push. Override required.
- **Only 50 installs:** validate the implementation before adopting it everywhere.

</Callout>
