## What it is

Another post-edit testing hook. The difference from [Run Tests After Changes](/boveda/hook-run-tests-after-changes): it intelligently detects which tests are relevant based on extensions and file structure, instead of running the whole suite.

Hook event: `PostToolUse`.

## How to install it

```bash
npx claude-code-templates@latest --hook testing/test-runner
```

For Claude:

```
I installed the testing/test-runner hook. Edit `src/auth/login.ts` —
it should run `tests/auth/login.test.ts` and its dependencies, not the
whole suite.
```

## What you can do with it

### Relevant tests only

Instead of running 1,000 tests for a small edit, it runs the 5 related ones.

### Faster feedback

Partial suite = seconds vs minutes.

### Combo with [TDD Gate](/boveda/hook-tdd-gate)

TDD Gate enforces that tests exist. Test Runner runs them. A complete TDD workflow.

### Combo with [Lint On Save](/boveda/hook-lint-on-save)

Lint + relevant tests = a fast quality pipeline.

## When to pick which

| Hook | When |
| --- | --- |
| **Test Runner (this one)** | You want smart tests (relevant only) |
| [Run Tests After Changes](/boveda/hook-run-tests-after-changes) | You want the full suite (more widely adopted, 1,017 installs) |
| Both | Don't install them together |

## When it's not a good fit

<Callout type="warning" title="Before installing">

- **If your file structure has no clear test/source convention:** the hook may not detect well.
- **For long autonomous sessions:** a periodic full suite may be more reliable.
- **If your suite is fast:** running everything is low overhead.

</Callout>
