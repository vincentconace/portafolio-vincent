## Qué es

Web Fetch es el MCP base de la categoría `web`: **3,346 installs**, el más adoptado. Su scope es directo: fetching de contenido web, extraction básica, acceso a APIs externas.

Comparado con alternativas:
- [Firecrawl](/boveda/mcp-firecrawl) — markdown LLM-ready, paga
- [Jina AI](/boveda/mcp-jina-ai) — grounding + fact-checking
- **Web Fetch (este)** — simple, sin costo, sin features fancy

Es el "fetch básico que funciona". Para muchos casos, alcanza.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp web/web-fetch
```

Para Claude:

```
Instalé el MCP web/web-fetch. Fetchá esta URL [URL] y devolveme el
contenido. Si es HTML, extraé el texto principal.
```

## Qué podés hacer con esto

### Fetch de URLs

"Traeme el contenido de `<URL>` y mostrame el body en texto plano". Operación más básica.

### Consumir APIs externas

"Hacé GET a `https://api.github.com/repos/owner/repo` y dame name, stars, last update". JSON parsing simple.

### Lectura puntual de docs

"Andá a esta página de docs de Next.js y respondeme: ¿qué es Server Actions?". Anti-hallucination casero.

### Scraping casual

Para pages sin anti-bot, alcanza. Para anti-bot serio, mejor [Bright Data](/boveda/mcp-brightdata) o [Browserbase](/boveda/mcp-mcp-server-browserbase).

### Combo con cualquier flujo

Por ser tan básico, encaja en muchos workflows como conector inicial.

## Pre-requisitos

- **Node.js + Claude Code**
- **Conexión a internet**

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para sitios con JS pesado (SPA):** Web Fetch trae HTML estático. Para SPA necesitás [Playwright](/boveda/mcp-playwright-mcp-server) o [browser-use](/boveda/mcp-browser-use-mcp-server).
- **Para sitios con anti-bot:** mejor herramientas especializadas.
- **Para output markdown limpio:** [Firecrawl](/boveda/mcp-firecrawl) es más prolijo.
- **Para fact-checking / grounding:** [Jina AI](/boveda/mcp-jina-ai).

</Callout>
