## Qué es

`enabledMcpjsonServers: ["memory", "github", "filesystem"]` — solo los servers listados se activan. Los demás definidos en `.mcp.json` quedan disabled.

Allowlist approach. Más restrictivo que [Enable All Project Servers](/boveda/setting-enable-all-project-servers) (que activa todo).

## Tabla resumen

| Campo | Valor |
| --- | --- |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting mcp/enable-specific-servers
```

Editá la lista con los MCPs reales:

```json
{
  "enabledMcpjsonServers": [
    "memory",
    "github",
    "filesystem",
    "context7"
  ]
}
```

Para Claude:

```
Instalé el setting mcp/enable-specific-servers. Listame los MCPs
del .mcp.json y dame el array de los que conviene enable para mi
uso. Los demás quedan disabled.
```

## Qué podés hacer con esto

### Control granular enterprise

Tu org tiene lista approved de MCPs. Este setting enforce que solo esos corran.

### Pequeño set vs catálogo amplio

`.mcp.json` puede tener 20 MCPs definidos. Si solo necesitás 5, listalos.

### Combo con [Disable Risky Servers](/boveda/setting-disable-risky-servers)

Enable Specific + Disable Risky = cinturón y tirantes. Cualquiera de los dos bloquea.

### Para sesiones autónomas

`/goal` largos: si Claude inventa usar un MCP no autorizado, no funciona.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu workflow legítimo varía:** allowlist puede ser restrictivo cuando cambias de tareas.
- **Para devs que experimentan con MCPs:** allowlist requiere actualizar setting cada vez.
- **Si no usás .mcp.json:** el setting no tiene caso.

</Callout>
