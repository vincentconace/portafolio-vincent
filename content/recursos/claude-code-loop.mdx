## Qué hace `/loop`

`/loop` es un comando dentro de Claude Code que ejecuta una tarea **en intervalos fijos** sin que vos estés mirando. Sintaxis básica:

```
/loop 5m /comando-o-tarea
```

Eso significa: cada 5 minutos, ejecutá `/comando-o-tarea`. Sigue corriendo hasta que vos lo pares.

Variantes:

```
/loop 1h "/check-prs"
/loop daily 09:00 "/brief"
/loop weekly mon 17:00 "/weekly-report"
```

Lo que cambia frente a un cron tradicional: **Claude maneja el output** de cada iteración. Si una iteración falla, te avisa. Si una iteración detecta algo crítico, escala. Si hay nada que hacer, lo dice y no te molesta.

---

## Cuándo `/loop` gana

Vs. otras opciones de automatización:

| Opción | Cuándo gana |
|--------|-------------|
| **`/loop`** | Tareas inteligentes que requieren razonamiento + acción contextual |
| **Cron tradicional** | Comandos puros que no necesitan LLM |
| **GitHub Actions** | Lógica atada al ciclo de PR/push |
| **Hooks SessionStart** | Trigger por evento, no por tiempo |
| **Managed Agents** | 24/7 sin tu máquina prendida |

`/loop` brilla cuando:
- Tu compu está prendida casi todo el día (no necesitás servidor)
- La tarea requiere contexto y razonamiento (no solo comando)
- Querés que las decisiones se tomen con criterio, no script duro

## 12 patrones útiles

### Monitoreo

**1. Watch de un PR específico**
```
/loop 10m "Revisá el PR #234. Si pasa CI, mostrame el green check.
            Si falla, decime el error específico."
```

**2. Monitor de logs de prod**
```
/loop 5m "Mirá los logs de prod del último intervalo (npm run logs:tail).
          Si hay >5 errores 5xx, alertame. Sino, silencio."
```

**3. Health check de servicios**
```
/loop 1m "Curlea https://api.miapp.com/health. Si responde 200, no digas
          nada. Si responde otra cosa, alertame."
```

### Trabajo recurrente

**4. Brief diario al arrancar**
```
/loop daily 08:30 "/brief"
```

(Tu skill `/brief` ya armada, ver [9 prompts para Cowork](/boveda/nueve-prompts-cowork))

**5. Reporte semanal de equipo**
```
/loop weekly fri 17:00 "/weekly-report"
```

**6. Limpieza de inbox**
```
/loop daily 09:00 "/triage-inbox"
```

### Investigación / scraping

**7. Mentions de marca**
```
/loop 2h "Buscá menciones de 'miempresa' en HN/Twitter/Reddit del
          intervalo. Si hay nuevas, resumime tono y links."
```

**8. Cambios en sitio del competidor**
```
/loop 6h "Scrapeá el sitio de competidorX. Compará vs lo de antes
          (guardás snapshot). Si hay cambios significativos en
          features/pricing, avisame."
```

### Development workflow

**9. Tests en watch mode con feedback**
```
/loop 30s "Si hay cambios en src/ desde el último check, corré los
          tests afectados y reportame solo si fallaron."
```

**10. Auto-rebase de feature branch contra main**
```
/loop 4h "Si estoy en una feature branch y main avanzó, rebaseame
          mostrándome conflictos primero. Si no hay conflictos, hacelo."
```

### Productividad personal

**11. Recordatorio inteligente de break**
```
/loop 45m "Recordame tomar un break si llevo más de 45 min sin parar.
          Si estoy en flow (commits recientes), no me interrumpas."
```

**12. Review de uso de tokens**
```
/loop daily 22:00 "Mostrame el gasto de tokens del día y compará vs el
                   promedio de la última semana. Si subió >40%, decime
                   qué tarea fue la culpable."
```

## Setup

### Modo básico — desde Claude Code

Dentro de una sesión:

```
/loop 10m "tu prompt o comando"
```

El loop arranca. **Si cerrás la sesión, el loop sigue corriendo** (en background, no necesitás tener la ventana abierta).

Para listar loops activos:

```
/loop list
```

Para parar uno:

```
/loop stop <id>
```

### Modo agendado — desde Claude Desktop

La app desktop tiene un **programador visual**:

1. Settings → Scheduled Tasks
2. New Task → seleccionás "Claude Code Loop"
3. UI para definir intervalo + comando + condiciones
4. Save

Más amigable si no querés tipear el intervalo.

### Modo remoto — desde teléfono

Combinado con [Claude Canales (Telegram)](/boveda/claude-canales-tg-discord):

```
[En Telegram]
Vos: /loop 30m "check si llegó email de Acme"
Bot: Loop activado. Te aviso si entra.

[30 min después]
Bot: Email de Acme llegó. Asunto: "Re: propuesta". Te lo resumo:
     [resumen]
```

Controlar tu Claude desde el celular como si fuera un asistente que sigue instrucciones standing.

## Anti-patrones

### 1. `/loop 1s "..."`

Loops muy cortos queman tokens. **Mínimo recomendado: 1 minuto.** Para algo que necesite menos, usá un script externo con curl.

### 2. Loops que no terminan nunca

Si un loop chequea algo que nunca cambia, va a correr eternamente sin propósito. **Definí siempre la condición de salida.**

```
❌ /loop 5m "watcha el PR"
✅ /loop 5m "watchá el PR #234 hasta que esté merged"
```

### 3. Acciones destructivas en loop

NUNCA `/loop ... "deletea archivos viejos"`. Una equivocación en el prompt → daño multiplicado por las iteraciones.

Para acciones destructivas: scripts con confirmación humana, no loops.

### 4. Demasiados loops simultáneos

Cada loop consume contexto + tokens. **Mantené máximo 3-5 loops activos**. Más allá empezás a notar lentitud y costo.

### 5. Loops sin output filtering

Si tu loop te avisa de TODO lo que ve, te volvés loco. Decile **qué te interesa avisar**:

```
✅ "Si hay >5 errores, alertame. Sino, silencio."
✅ "Solo decime cuando cambie el status."
```

## Combinación con hooks

`/loop` y [hooks](/boveda/hooks-automatizacion) se complementan:

- **Hooks**: trigger por **evento** (después de Edit, al arrancar sesión)
- **Loop**: trigger por **tiempo** (cada N minutos)

Para tareas que tienen ambos componentes, los combinás:

```
- Hook PostToolUse: corre tests cuando editás algo
- Loop daily 18:00: revisá si quedaron tests fallando del día
```

Cada uno cubre su ángulo.

---

## El costo

Cada iteración de un loop **gasta tokens**. Considerá:

| Loop | Tokens aprox / iteración | Costo / día |
|------|--------------------------|--------------|
| Health check simple | 200-500 | menos de $0.10 |
| Brief diario | 2000-5000 | ~$0.05 |
| Monitor de logs | 1000-3000 | ~$0.30 |
| Scraping + análisis | 5000-15000 | ~$1-3 |

Loops de razonamiento ligero corriendo todo el día → unos pocos dólares mensual. Loops pesados → calculá antes para no sorprenderte.

---

## Cuándo no usar `/loop`

- **Compu apagada seguido**: el loop muere cuando cerrás Claude / la compu. Para 24/7 real, [Managed Agents](/boveda/agentes-managed) o servidor propio
- **Latencia crítica (menos de 1 min)**: `/loop` no garantiza intervalo exacto. Para SLA estricto, scripts dedicados
- **Tareas que NO requieren contexto**: si es solo "correr este comando", cron es más barato
