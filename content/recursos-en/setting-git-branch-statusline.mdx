## What it is

**#3 among statuslines (1,545 installs).** A functional statusline that shows:
- **Model** name in use
- **Directory** of the project
- **Git branch** currently active
- **Change indicators** (count of uncommitted changes)

No gimmicks. Useful info for devs with an intense git workflow.

## What you can do with it

### Constant git context

Know which branch you're on and whether you have uncommitted changes without having to run `git status` every time.

### Anti "oops, I'm on main"

When you see "main" in the statusline, stop editing before you make a mess.

### Combo with [Validate Branch Name](/boveda/hook-validate-branch-name) or [Prevent Direct Push](/boveda/hook-prevent-direct-push)

Statusline + git protection hooks = a safe, visible git workflow.

### Combo with [Git Flow Settings](/boveda/setting-git-flow-settings)

Git Flow Settings covers permissions + hooks + statusline for a complete Git Flow. Comparable to this one but more opinionated.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you don't use git:** it adds nothing.
- **If you want more info** ([Context Monitor](/boveda/setting-context-monitor) also covers context + cost).
- **If your terminal is very narrow:** the statusline may not fit.

</Callout>
