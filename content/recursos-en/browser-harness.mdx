## What it solves

Classic scraping (curl + HTML parsing) **fails on many modern sites**:

- LinkedIn requires login → curl can't do it
- Amazon has aggressive anti-bot protection → it blocks you
- Facebook Ads Library is a heavy-JS SPA → the initial HTML has no data
- Mercado Libre changes its HTML often → your parser breaks

**Browser Harness** solves this differently: instead of simulating traffic, it **uses your real Chrome with your logged-in sessions**. Claude operates your browser the way you would — clicks, scrolling, navigation.

**Bonus**: every successful task **gets saved as a per-site skill**. The first time, Claude "learns" how to pull prices from Amazon; the second time it goes straight to it.

## How it works

### Technical setup

Chrome runs with the **debugging port open** (`--remote-debugging-port=9222`). That lets an external process (Claude) control it via the Chrome DevTools Protocol.

Important: **it's YOUR Chrome with YOUR session**. If you're logged into LinkedIn, Claude operates as you.

### The typical flow

```
[You]: Get me the top 10 headphone prices on Amazon, sorted
       by best sellers.

[Claude]:
  First time with Amazon. I'm going to "learn" how to do it:
  - Open Amazon
  - Navigate to Electronics > Headphones
  - Apply the "Most popular" filter
  - Scrape the first 10
  - Save the procedure as the skill `amazon-products-search.md`

  [Shows you the progress step by step]

  Done. Result:
  1. Sony WH-1000XM5 — $399
  2. Apple AirPods Pro 2 — $249
  ...

  Skill saved. Next time you ask for something similar on Amazon,
  I'll go straight to it.
```

### The second time

```
[You]: Now get me the top 10 mechanical keyboards on Amazon.

[Claude]:
  [uses the amazon-products-search skill it learned]
  [goes straight to the flow]
  [brings you the 10 in 30 seconds]
```

## Anti-patterns and warnings

### 1. Sites with login = detection risk

LinkedIn / Facebook **prohibit scraping in their terms**. Even if your real browser makes the connection, **repetitive patterns** give you away:

- Lots of fast requests → they ban you
- Non-human pattern (you don't scroll, you don't move the mouse) → flag

**Mitigation**:
- **Low volume** (dozens of queries per day, not thousands)
- **Sleep between actions** (make it look human)
- **Use a separate account** for scraping, not your personal one

### 2. DON'T log in with your sensitive personal account

If you get blocked / suspended scraping LinkedIn, you lose that account. **Use a secondary account** for scraping.

### 3. Terms of service

Every site has its own terms. **Read them first** before building a serious pipeline. Some explicitly prohibit automation, others allow it with conditions.

### 4. NO email harvesting

As with any scraping, **pulling emails for mass spam** is problematic legally and technically (CAN-SPAM, GDPR). Browser Harness doesn't exempt you from those laws.

### 5. Generated skills need validation

The skill Claude learns the first time **may not be optimal**. Review it before trusting it 100%:
- Are the selectors robust or too specific?
- Does it handle the "no results found" case?
- What happens if the site changes its layout?

## Installation

### Step 1 — Paste the prompt

```
> Install Browser Harness for me following the install.md from the
  official repo.
```

Claude:
1. Clones the repo
2. Installs deps
3. Configures the Chrome hooks
4. Explains how to launch Chrome with the debugging port

### Step 2 — Launch Chrome with the debugging port

Mac:
```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222
```

Close all Chrome instances first. This session is the one Claude controls.

### Step 3 — Test it

```
> Browser Harness, open google.com and search for "test".
```

If it works, you're connected.

## The most important warning

When Browser Harness operates your Chrome:

- **It can see what's open** in other tabs (potentially)
- **It can act as you** on sites where you're logged in (bank, email, etc.)

**Serious mitigation**:
- Use a **dedicated Chrome profile** just for this
- In that profile, do **NOT** be logged into your bank / critical systems
- Use your main Chrome for normal life, and Chrome with the debugging port for Claude

This separation reduces the risk of a random command touching something critical.
