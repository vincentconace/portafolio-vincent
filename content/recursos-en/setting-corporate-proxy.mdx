## What it is

For enterprises that route all outbound internet traffic through a corporate proxy. Without this setting, Claude Code can't reach the Anthropic API.

It configures:
- `HTTP_PROXY` = `http://proxy.company.com:8080`
- `HTTPS_PROXY` = `https://proxy.company.com:8080`

(The real values are the ones for your org.)

```bash

```

## What you can do with it

### A working Claude Code in the enterprise

Without this, Claude Code doesn't work on a corporate network because it can't reach the API.

### Egress compliance

All outbound traffic goes through the proxy with logging/inspection. The setting enables Claude while respecting the corporate flow.

### Combo with [Custom Headers](/boveda/setting-custom-headers)

If your proxy requires auth headers, combine it with Custom Headers.

### Combo with [Bedrock Configuration](/boveda/setting-bedrock-configuration)

Some enterprises prefer Bedrock (not Anthropic direct) + proxy. Complementary settings.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your network doesn't require a proxy:** doesn't apply.
- **If your proxy requires non-standard auth:** check whether Claude Code supports it.
- **Performance:** a proxy adds latency. On a slow corporate network, Claude Code will be slower.
- **Proxy URL changes:** if the proxy URL changes (rotation), you have to update it.

</Callout>
