## What it is

Superpowers Marketplace is the personal curation of **obra** (Jesse Vincent, a well-known figure in the Anthropic ecosystem). It's small (9 plugins) but each one is carefully thought out and maintained. The style is "what I use every day." Since the original Superpowers (also in this marketplace) is included in Anthropic's [Claude Plugins Official](/boveda/plugin-claude-plugins-official), this marketplace is where you'll find the more recent versions and the experimental siblings.

The 9 plugins:
- `superpowers` — the core: TDD + debugging + collaboration patterns
- `superpowers-chrome` — direct Chrome DevTools Protocol (BETA)
- `episodic-memory` — semantic memory across sessions
- `elements-of-style` — clear writing based on Strunk Jr. (1918)
- `superpowers-lab` — experimental: tmux automation, MCP discovery, Slack, headless Windows VM
- `superpowers-developing-for-claude-code` — for building plugins/skills/MCPs
- `superpowers-dev` — DEV branch (don't install alongside other versions)
- `claude-session-driver` — control other Claude Code sessions as workers via tmux
- `double-shot-latte` — eliminates "Would you like me to continue?"

## What you can do with this

### Automatic TDD red/green

`superpowers` brings the full TDD red/green pattern: you write the expected behavior, Claude generates the failing test (red), then implements until it passes (green). It's the author's opinionated version.

### Brainstorming + systematic debugging

The plugin proposes specific patterns: when you start something new, guided brainstorming; when something breaks, debugging with hypotheses + verification, not "let's try stuff."

### Chrome direct with no middlemen

`superpowers-chrome` gives you direct CDP (Chrome DevTools Protocol). Lower level than Playwright. For tasks where you need fine-grained browser control without Playwright's abstractions. BETA: lightly tested, use with judgment.

### Semantic memory across sessions

`episodic-memory` indexes your past conversations with semantic search. When you ask for "what we discussed about X last week," it retrieves the relevant context. Compare it with [Claude Mem](/boveda/plugin-claude-mem) — they're different approaches to the same problem.

### Writing that respects the classics

`elements-of-style` applies Strunk Jr.'s rules when Claude writes. Result: more concise docs, READMEs, and copy. If "inflated AI language" bothers you, this plugin is an ally.

### Claude workers controlled via tmux

`claude-session-driver` lets you launch other Claude Code sessions as workers that you control from a master session. Multi-agent with real sessions, not simulated agents.

### Goodbye to "Would you like me to continue?"

`double-shot-latte` eliminates the interruptions where Claude asks whether to keep going. The plugin itself evaluates whether continuing makes sense and decides. For long autonomous sessions, it removes friction.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **`superpowers-chrome` is marked BETA** — use it with judgment in serious code.
- **`superpowers-dev` requires uninstalling other versions** first. Version conflicts.
- **`double-shot-latte` can lead to very long runs**: if Claude always decides to continue, you can burn through tokens without realizing it. Set a cap.
- **`claude-session-driver` requires tmux** installed and working. It doesn't work well on native Windows.
- **Overlap between `episodic-memory` vs `claude-mem` vs native memory:** first evaluate what native memory already covers before layering on one or both.

</Callout>
