## What it is

When Claude edits a source file, this hook detects your project's build tool (npm, pnpm, yarn, vite, tsc, etc.) and automatically runs the corresponding build command. Hook event: `PostToolUse`.

Handy for fast feedback during assisted development: you edit → the build runs → you see whether you broke something without having to ask for it.

## How to install it

```bash
npx claude-code-templates@latest --hook automation/build-on-change
```

For Claude:

```
I installed the automation/build-on-change hook. Test it: edit a TS file
and verify that it triggers the build. If it fails, show me the errors.
```

## What you can do with this

### Continuous error feedback

Every edit triggers a build. If Claude introduces a type error or syntax error, you see it immediately.

### Combo with type checking

For strict TS projects, the build runs `tsc --noEmit`. Catch type issues on the spot, not in CI.

### Watch without watch

For projects without a watch mode set up, this hook simulates one: edit → automatic build.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **Slow builds:** if your build takes 30+ seconds, every edit stalls the session. Consider whether you should edit several files before building.
- **For bulk editing:** Claude edits 20 files in one pass = 20 builds. You probably want manual debouncing.
- **For non-build languages:** interpreted Python, Bash, etc. — the hook adds nothing.

</Callout>
