## What it is

The "github-official" variant of the GitHub MCP in the aitmpl catalog. The inline description is terse ("GitHub") and the concrete details live in the [JSON config](https://github.com/davila7/claude-code-templates/blob/main/cli-tool/components/mcps/devtools/github-official.json).

The natural question: how does it differ from the more popular [GitHub Integration](/boveda/mcp-github-integration) (2,867 installs vs 84)? The main difference is which MCP server implementation it uses under the hood — the catalog lists both as separate entries.

For practical use: if you're starting from scratch, [GitHub Integration](/boveda/mcp-github-integration) has more adoption and almost certainly more community maintenance. This "official" variant is here in case you have a preference for a specific implementation that the JSON reveals.

## How to install it

```bash
npx claude-code-templates@latest --mcp devtools/github-official
```

Prerequisites: a GitHub Personal Access Token.

## What you can do with it

The same as with [GitHub Integration](/boveda/mcp-github-integration): read repos, manage issues and PRs, releases, search. The difference, again, lies in what exactly each MCP exposes and how it implements it.

For concrete use cases, check the examples in [GitHub Integration MCP](/boveda/mcp-github-integration) — they apply almost 1:1.

## Prerequisites

- **GitHub account** + Personal Access Token with the right scopes.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you already have `github-integration` installed:** don't install both. Names and tools will collide.
- **Without clear info on the difference:** the description on aitmpl is generic. If your case is standard, [GitHub Integration](/boveda/mcp-github-integration) (more widely adopted) is the safe option.
- **Only 84 installs:** far fewer people in the community testing it than the integration version. Issues may be less visible.

</Callout>
