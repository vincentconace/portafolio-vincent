## La combinación

Replicar el diseño de cualquier página web requiere 2 piezas:

| Pieza | Rol |
|-------|-----|
| **UI UX Pro Max Skill** | Conocimiento — 161 reglas de diseño por industria |
| **El prompt de replicación** | Instrucción — analizá URL y replicá todo |

Resultado: pegás URL → Claude analiza layout/colores/fonts/animations → reconstruye en tu proyecto con **tu contenido**.

---

## Pre-requisitos

- **Claude Code** instalado ([instalar Claude Code](/boveda/instalar-claude-code))
- **Proyecto frontend** (Next.js, React, etc.)

---

## Instalar UI UX Pro Max

```bash
claude install-skill https://github.com/nextlevelbuilder/ui-ux-pro-max-skill
```

### Qué incluye

| Componente | Cantidad |
|------------|----------|
| Estilos UI | 67 (Glassmorphism, Neumorphism, Brutalism, AI-Native, más) |
| Paletas de colores | 161 (por categoría de producto) |
| Combos de fuentes | 57 (con Google Fonts) |
| Reglas de razonamiento | 161 en 8 dominios (tech, finanzas, salud, e-commerce, más) |

## El prompt de replicación

```
Esta es la landing page [URL].

Replicá:
- Layout y espaciado
- Tipografía
- Patrones de animación
- Colores y paleta

Usá el skill UI UX Pro Max y Tailwind CSS para construir un
sitio completamente responsive.

Las plantillas debe poder aplicarlas a otras páginas/landings,
para que mi landing se vea como este sitio pero con mi
contenido y copy.

Si no tenés imágenes o gráficos, usá NanoBanana para generarlos.
```


## Anti-patrones

### 1. Replicar sin permisos legales

Aunque el código que generás es 100% tuyo, **clonar diseños protegidos de marcas grandes** es zona gris. Para uso personal/experimentación, OK. Para producto comercial, **inspirate, no copies literal**.

### 2. Esperar pixel-perfect

El skill replica **patrones y estética**, no copia exacta. Si necesitás pixel-perfect, usá [Clonador](/boveda/clonador-de-paginas) o trabajalo a mano.

### 3. No usar tu propio brand

Replicar y dejar **colores y fuentes del original** = tu brand se ve genérica. Cambialo después por tu paleta.

### 4. Replicar sin entender qué replicás

El skill genera código. **Leelo**. Si no entendés algo, pedile a Claude que explique. Es **aprendizaje**, no solo copy-paste.

### 5. Ignorar performance del original

A veces el site original es **bonito pero pesado**. Si replicás todo incluido el peso, **vas a tener mismo problema**. Optimizá lo que aplique.

## Combinaciones potentes

### Con Cult UI

```
> Replicá estructura del site con prompt de Replicación.
> Después reemplazá componentes genéricos con [Cult UI](/boveda/cult-ui).
```

### Con humanizer

```
> El copy que Claude genera para llenar la réplica → pasalo por
  /humanizalo. Que no suene IA.
```

### Con Brand Guidelines

```
> Carga mi brand pack.
> Replicá estructura del site pero aplicá MI paleta y MI tipografía,
  no las del original.
```
