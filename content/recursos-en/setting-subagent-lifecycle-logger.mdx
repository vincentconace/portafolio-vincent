## What it is

SubagentStart + SubagentStop hooks that log every subagent invocation with a timestamp. Every time Claude spawns or finishes a subagent, a JSON entry is recorded.

For multi-agent workflows where you need to audit:
- Which subagents ran
- When they started / finished
- How long they took
- For what purpose

## How to install

```bash
npx claude-code-templates@latest --setting hooks/subagent-lifecycle-logger
```

For Claude:

```
I installed the hooks/subagent-lifecycle-logger setting. Launch a task
that uses subagents and show me the generated JSON log.
```

## What you can do with it

### Audit multi-agent flows

If your workflow launches 8 subagents for a goal, the log shows you what each one did.

### Performance analysis

Identify slow subagents by analyzing the timestamps.

### Compliance trail

For orgs that require documentation of AI agent invocations.

### Combo with [Context Timeline Hook](/boveda/hook-context-timeline)

Subagent Logger for structured JSON. Context Timeline for a visual view.

### For autonomous sessions

Long `/goal` runs that launch agents: the log is essential for the post-mortem.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you don't use subagents:** the log stays empty.
- **Only 7 installs:** very niche.
- **If you want to log ALL tool calls:** [Command Logger Hook](/boveda/hook-command-logger) is broader.

</Callout>
