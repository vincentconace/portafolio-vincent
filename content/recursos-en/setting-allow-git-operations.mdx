## What it is

By default, Claude Code asks before running commands. This setting pre-authorizes common git ops:
- `git status`
- `git diff`
- `git add`
- `git commit`
- `git push`
- ...and more (7 ops total)

Destructive operations still require confirmation. A smooth git workflow without "allow / cancel" every time.


## What you can do with it

### Uninterrupted git workflow

You edit, Claude commits, Claude pushes — no "allow" on every step.

### For autonomous sessions

A `/goal` that includes git ops: the setting keeps Claude from getting stuck asking for confirmation.

### Combo with [Conventional Commits Hook](/boveda/hook-conventional-commits)

The setting authorizes commits. The hook validates the format. An automated but safe git workflow.

### Combo with [Prevent Direct Push Hook](/boveda/hook-prevent-direct-push)

The setting authorizes generic push. The hook blocks pushes to protected branches. Defense in depth.

## When it's NOT a good fit

<Callout type="warning" title="Before enabling">

- **For critical repos:** consider whether you always want confirmation.
- **Without security hooks:** auto-push without [Prevent Direct Push](/boveda/hook-prevent-direct-push) can be dangerous.
- **If your workflow is review-each-step:** this setting skips those steps.
- **Git tokens with broad scope:** the setting + broad tokens = potential catastrophe.

</Callout>
