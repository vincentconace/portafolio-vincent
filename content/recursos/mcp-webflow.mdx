## Qué es

[Webflow](https://webflow.com) es un site builder visual no-code muy usado para marketing sites, landing pages y portfolios. Su API permite gestión programática de CMS, collections, items, publishing.

Este MCP conecta esa API a Claude Code. Útil cuando tu sitio Webflow es la fuente de truth de contenido y querés que Claude opere sin abrir el designer.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/webflow
```

Pre-requisito: cuenta Webflow + API token desde [webflow.com/dashboard/account/integrations](https://webflow.com/dashboard/account/integrations).

Para Claude:

```
Instalé el MCP devtools/webflow. Configurá el API token. Listame mis
sites. Para el site X, mostrame collections y un sample de items por
cada una.
```

## Qué podés hacer con esto

### CMS automation

"Para mi site Y, agregame estos 20 items en la collection `Blog Posts` desde mi CSV". Bulk import sin abrir designer.

### Sync con código

"De los PRs que cerré con label `release-notes`, generame un item en la collection `Changelog` con título, fecha y resumen del cambio".

### Update masivo

"En la collection `Products`, actualizame el campo `price` multiplicando por 1.15 (incremento del 15%) para los items en la categoría X".

### Publishing programático

"Después de actualizar items, publica los cambios al dominio live". Sin click manual.

### Audit de contenido

"Listame items en la collection `Blog Posts` sin imagen destacada o sin meta description. Para SEO matters".

### Sync entre [Notion](/boveda/mcp-notion) y Webflow

Pipeline tipo "redacto en Notion, publico en Webflow". El MCP lo facilita.

## Pre-requisitos

- **Cuenta Webflow** activa
- **API token** con scopes correctos
- **Sites y collections** ya estructurados

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Webflow:** para WordPress, Ghost, Sanity, otros MCPs.
- **Operaciones destructivas:** delete items / collections / sites. Hooks de aprobación.
- **API rate limits:** Webflow tiene cuotas. Bulk ops pueden bloquearte.
- **Si tu uso es solo design:** el MCP es para CMS y API. Para tocar visualmente, usá el designer.
- **Para sitios chicos:** la UI de Webflow es buena. El MCP brilla cuando manejás muchos items o sites.

</Callout>
