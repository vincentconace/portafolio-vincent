## Qué es

Otra implementación del patrón "auto-stage". Después de cada file modification, corre `git add` automático. Hook event: `PostToolUse`.

Diferencia con [Auto Git Add](/boveda/hook-auto-git-add) (870 installs, en `git-workflow` category): mismo concepto, distinta implementación. El más adoptado es Auto Git Add.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `post-tool` |
| Installs | 420 |
| Hook event | `PostToolUse` |
| Acción | `git add` post-edit |

## Cómo se instala

```bash
npx claude-code-templates@latest --hook post-tool/git-add-changes
```

Para Claude:

```
Instalé el hook post-tool/git-add-changes. Después de un edit, corre
`git status` para verificar que quedó staged.
```

## Qué podés hacer con esto

Mismas operaciones que [Auto Git Add](/boveda/hook-auto-git-add):
- Workflow git limpio sin overhead
- Diff fácil de ver (`git diff --staged`)
- Combinable con auto-format y conventional commits

## Cuándo elegir cuál

| Caso | Recomendación |
| --- | --- |
| Sin preferencia | [Auto Git Add](/boveda/hook-auto-git-add) (más adoptado) |
| Categorización por `post-tool` matters | Este |
| No los instales ambos | Solo uno |

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si querés review manual antes de stage:** ambos hooks stagean automático.
- **Si ya tenés [Auto Git Add](/boveda/hook-auto-git-add):** no instales este encima — duplicación.
- **Para edits experimentales:** archivos staged son commit-ready. Cuidado con WIP.

</Callout>
