## Qué es

Postman es la app universal para testear y documentar APIs: collections, environments, tests, mocking, monitors. Este MCP conecta tu cuenta a Claude Code.

Capabilities:
- **Buscar collections** y requests
- **Ejecutar requests** con environments
- **Correr tests**
- **Operar workspaces**
- **Generar requests** desde specs (OpenAPI / GraphQL)

Útil cuando tu equipo mantiene specs en Postman como source of truth y querés que Claude opere ahí mismo.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/postman
```

Pre-requisito: cuenta Postman + API key desde [postman.com/settings](https://www.postman.com/settings).

Para Claude:

```
Instalé el MCP devtools/postman. Listame mis collections en el workspace
`Backend APIs`. Ejecutame el request `Health Check` en environment
`production` y mostrame la response.
```

## Qué podés hacer con esto

### Audit de collections

"Listame todas las collections en el workspace X. Identificá las que no se editaron en los últimos 90 días (candidatas a archivado)".

### Ejecución de tests

"Corré toda la collection `Smoke Tests` en environment `staging`. Si algún test falla, mostrame el request, response y assertion que falló".

### Generación de requests desde OpenAPI

"De este OpenAPI spec, generame una Postman collection con un request por cada endpoint, agrupados por tag".

### Sync con código

"Para cada endpoint que cambié en mi backend, actualizá el request correspondiente en la collection X — params, body, headers".

### Documentation as code

"Para los 20 endpoints en mi collection, generá docs en markdown con: description, request example, response example, error codes". Output para README o site.

### Combinación con [GitHub](/boveda/mcp-github-integration)

PR que cambia endpoint → MCP de Postman actualiza la collection → MCP de GitHub commitea los cambios al repo de docs. Pipeline.

## Pre-requisitos

- **Cuenta Postman** activa
- **API key** desde Postman Settings → API Keys
- **Workspaces y collections** ya estructurados

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Postman:** este MCP es Postman-only. Para Insomnia / Bruno / Hoppscotch hay otros tools.
- **Para tests E2E serios:** Postman es para API testing. Para E2E mejor [Playwright](/boveda/mcp-playwright-mcp-server) o herramientas dedicadas.
- **Operaciones destructivas:** delete collections / workspaces. Hooks de aprobación.
- **API key con scope amplio:** restringí si tu plan lo permite.
- **Si tu equipo no actualiza collections:** el MCP brilla si las collections son fuente de verdad. Si son stale, no aporta.

</Callout>
