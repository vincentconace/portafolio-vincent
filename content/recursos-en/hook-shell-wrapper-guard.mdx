## What it is

The basic [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker) matches direct patterns: `rm -rf`, `dd if=`. But `bash -c 'rm -rf /'` slips past the simple match.

This hook covers that gap: it detects destructive commands inside wrappers:
- `sh -c "..."`
- `bash -c "..."`
- `python3 -c "..."`
- `node -e "..."`
- `perl -e "..."`
- `ruby -e "..."`

It inspects the embedded code looking for destructive commands. Hook event: `PreToolUse`.

## What you can do with it

### Close the basic blocker's gap

If all you have is [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker), a command routed through a wrapper gets through. This hook covers it.

### For autonomous sessions

AI models sometimes "invent" creative ways to get things done — including destructive commands via wrappers. Defense in depth.

### Natural combo

A triple safety net:
1. [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker) — direct patterns
2. **Shell Wrapper Guard (this one)** — wrappers
3. [AI Bash Guard](/boveda/hook-ai-bash-guard) — AI-driven, catches variations

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your legitimate workflow relies heavily on wrappers:** sysadmins, devops. Overriding may be frequent.
- **Without [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker) installed:** the complement stands alone. Install both together.
- **Only 76 installs:** validate the implementation.

</Callout>
