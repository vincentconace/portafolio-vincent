## What it is

**#3 overall in the catalog (2,076 installs).** The universal formatter pipeline.

After every edit, it detects the language and applies the right formatter:
- **JS/TS** → Prettier
- **Python** → Black
- **Go** → gofmt
- **Rust** → rustfmt
- **And more language-specific ones**

It replaces the individual hooks ([Format JavaScript Files](/boveda/hook-format-javascript-files), [Format Python Files](/boveda/hook-format-python-files)) with a single one that covers them all.

Hook event: `PostToolUse`.

## How to install it

```bash
npx claude-code-templates@latest --hook development-tools/smart-formatting
```

Prerequisite: the formatters for your languages installed.

For Claude:

```
I installed the development-tools/smart-formatting hook. List which
formatters it detects as available on my system (prettier, black,
gofmt, etc.).
```

## What you can do with it

### Universal format pipeline

No matter the language, formatting gets applied. You don't have to install a hook per language.

### For polyglot repos

If your repo has JS + Python + Go (a common case), a single hook covers them all.

### Combo with [Lint On Save](/boveda/hook-lint-on-save)

Format first, lint after. A clean cross-language pipeline.

### For autonomous sessions

Long `/goal`: code always ends up formatted, without you having to remember to ask for it.

### Replaces several hooks

If you have [Format JS](/boveda/hook-format-javascript-files) and [Format Python](/boveda/hook-format-python-files) separately, this one replaces them with a single hook.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **No formatters installed:** the hook fails silently for languages without a tool.
- **If your team has a custom formatter:** it may not be in the supported list.
- **For very niche languages:** check compatibility.
- **If you already have [Format JS](/boveda/hook-format-javascript-files) or [Format Python](/boveda/hook-format-python-files):** don't install both.

</Callout>
