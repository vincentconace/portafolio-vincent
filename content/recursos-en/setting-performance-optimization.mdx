## What it is

**4,902 installs — the absolute #2 among settings (after Context Monitor 10.8k).** Nearly 5x more adopted than #3.

Optimizes Claude Code performance with 3 env vars:
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS: 8000` — caps output per response
- `DISABLE_NON_ESSENTIAL_MODEL_CALLS: 1` — avoids non-critical model calls
- `DISABLE_COST_WARNINGS: 1` — silences cost warnings

Result: faster responses, fewer tokens (= less $), less UI noise.

```bash

```

## What you can do with it

### Reduce cost

Capping output = fewer tokens generated = less $.

### Faster responses

The output cap also cuts response time.

### Cleaner UI

Without constant cost warnings, there's less noise.

### For teams on a tight budget

Hard caps prevent a dev from accidentally overshooting.

### Combo with [Bash Timeouts](/boveda/setting-bash-timeouts) and [MCP Timeouts](/boveda/setting-mcp-timeouts)

Performance Opt + Bash Timeouts + MCP Timeouts = every limit dialed in. A tuning bundle.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your workflow needs long responses:** 8000 tokens is restrictive. Raise it if needed.
- **If you want to see cost warnings:** disabling silences them, but some warnings are useful.
- **DISABLE_NON_ESSENTIAL_MODEL_CALLS:** some features may depend on "non-essential" calls. Check the impact.

</Callout>
