## Qué es

[Microsoft Clarity](https://clarity.microsoft.com) es probablemente la mejor sorpresa gratis del stack analytics: heatmaps, session recordings (videos de cómo usuarios reales usan tu app), insights de rage clicks y dead clicks, todo sin pagar. Compite directo con Hotjar / FullStory que cobran por sesiones.

Este MCP lo conecta a Claude Code. Le pedís a Claude que consulte Clarity y analice patterns de comportamiento sin abrir el dashboard.


- Project ID + API token

Para Claude:

```
Instalé el MCP devtools/microsoft-clarity. Configurá las credenciales y
mostrame los heatmaps de la última semana para la home page. Identificá
los 3 elementos con más rage clicks.
```

## Qué podés hacer con esto

### Detección de rage clicks

"En las últimas 7 días, ¿qué elementos generaron más rage clicks (usuarios haciendo click frustrados)? Probablemente sean los que rompen UX". Insight súper accionable.

### Heatmaps por página

"Para la página de pricing, dame el heatmap de scrolls: ¿hasta dónde llegan los visitors? ¿hay un cliff donde abandonan?".

### Session recordings filtrados

"Mostrame 5 sessions de usuarios que llegaron al checkout pero abandonaron antes de pagar. Para cada uno, identificá en qué momento abandonó y por qué".

### Dead clicks

"Detectá elementos visuales que parecen clickables pero no responden (dead clicks). Listame los top 10". Catch low-hanging UX bugs.

### Sync con análisis cross-source

Combinado con [Google Workspace](/boveda/mcp-google-workspace) o data en sheets, podés cruzar comportamiento de Clarity con cohort de usuario o segmentación de marketing.

### Tabla de prioridades

"Para los 10 rage clicks más frecuentes, asigná effort estimado de fix y impact estimado. Sugiéreme cuál atacar primero".

## Pre-requisitos

- **Cuenta Microsoft Clarity** (gratis, sin tier paid)

- **API token / project ID**

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no tenés Clarity instalado en tu sitio:** sin el tracking, no hay data. Es lo primero.
- **Compliance GDPR:** Clarity tiene su política de tracking. Validá consent en tu sitio si tu audiencia incluye EU.
- **Para web analytics completos** (funnels, attribution): Clarity es session replay + heatmaps. Para eso, [Google Analytics 4](/boveda/mcp-google-workspace) o [Plausible](https://plausible.io) complementarios.
- **Para apps móviles:** Clarity tiene SDK iOS/Android pero menos maduro. Verificá si tu caso aplica.
- **Privacy:** session recordings ven todo. Configurá masking para campos sensibles (password, credit card) — Clarity lo permite.

</Callout>
