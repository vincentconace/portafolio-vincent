## What it is

This is **the** Playwright MCP to use by default. Its 4,419 installs make it the most adopted in the entire `browser_automation` category, ahead of [browser-use](/boveda/mcp-browser-use-mcp-server) (1,636), [Chrome DevTools](/boveda/mcp-chrome-devtools) (2,685), and other Playwright variants.

It's `executeautomation`'s Playwright implementation as an MCP. It covers everything you need:
- Navigation
- Element interaction (click, type, fill forms)
- Screenshots and PDFs
- Arbitrary JS execution
- DOM inspection
- Cookie and session handling

If your team is getting started with browser automation from Claude Code and you don't want to compare variants, install this one.

## How to install it

```bash
npx claude-code-templates@latest --mcp browser_automation/playwright-mcp-server
```

For Claude:

```
I installed the browser_automation/playwright-mcp-server MCP. Open Chromium
in headless mode, navigate to example.com, wait for full load, and give me
back the title, the final URL, and a screenshot.
```

## What you can do with this

### Assisted E2E tests

"Go to `localhost:3000/signup` and test the 3 cases: valid email + OK password (should pass), invalid email (should show an error), short password (should show an error). Validate the specific messages and show me screenshots."

### Cross-browser sanity

"Go to `my-app.com` in Chromium, Firefox, and WebKit. Take a screenshot of the home page in each one and compare whether there are any visible differences."

### Clean scraping (when there's no serious anti-bot)

"Go to this page, extract the 10 products with price, rating, and link, and return them to me in JSON."

### Post-deploy validation

"Go to production and validate: the home page loads, the checkout link works, the signup form submits. Give me back a smoke test report."

### Assisted debugging

"On localhost:3000, open the DevTools console and bring me the errors that appear when clicking `Checkout`. Then navigate and describe which flow is broken."

### Login flows

"Log in with email X and password Y, navigate to the dashboard, and bring me the first 5 visible cards." It keeps the session.

## Prerequisites

- **Playwright installs browsers** automatically the first time
- **Disk space** for Chromium + Firefox + WebKit (several GB)
- **If your system is Linux:** some OS dependencies may be required

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **Sites with aggressive anti-bot:** use [Browserbase](/boveda/mcp-mcp-server-browserbase) or [Bright Data](/boveda/mcp-brightdata).
- **Maintainable E2E tests for CI:** better to use Playwright Test directly, not via MCP.
- **Resources:** Chrome eats RAM. Parallel sessions can saturate it.
- **Sites with your own login that you want to keep:** [Browser Harness](/boveda/browser-harness) for your real session.

</Callout>
