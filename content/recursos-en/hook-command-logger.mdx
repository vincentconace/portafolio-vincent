## What it is

Before each tool use (Bash, Edit, Write, Read, etc.), this hook appends an entry to the log: timestamp + tool name + basic args. Hook event: `PreToolUse`.

Useful for understanding Claude's behavior: which tools does it use? In what order? Are there repetitive patterns you could automate?

```bash
npx claude-code-templates@latest --hook development-tools/command-logger
```

## What you can do with it

### Tool audit

Know exactly which tools Claude used during the session. For compliance or to understand behavior.

### Debugging when something goes wrong

If Claude behaved oddly, you look at the log and reconstruct what it did step by step.

### Spot patterns to optimize

If you see Claude always uses the same sequence Read → Edit → Bash → Read, you can build a skill that does it in a single step.

### Combo with [Change Tracker](/boveda/hook-change-tracker)

Command Logger records which tools were used (including reads). Change Tracker records only the mutations. Combine them for a complete audit.

## When it's NOT worth it

<Callout type="warning" title="Before you install">

- **For very intensive sessions:** lots of tool calls = big log. Log rotation recommended.
- **If tool arg contents are sensitive:** some args may include sensitive paths or snippets. Validate what gets recorded.
- **If you already use [Context Timeline](/boveda/hook-context-timeline):** there's overlap (Timeline also shows tool usage).

</Callout>
