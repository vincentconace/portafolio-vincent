## Qué resuelve

Si usás [Claude Design](/boveda/claude-design) intensamente, conocés 2 problemas:

1. **Quema tokens rápido** — cada iteración de diseño consume cuota
2. **A veces se siente genérico** — el output default suena a "shadcn UI estándar"

**Open Design** (de `nexu-io`) es la alternativa **local-first, open-source, Apache-2.0**:

- Vive en TU compu (no en la nube de Anthropic)
- Detecta automáticamente tu agente (Claude Code, Codex, Cursor, Gemini, Qwen)
- Lo usa como **motor** sin consumir tokens "extra" — los que ya gastás con tu agente
- **Conectado a modelo local** (Llama, Qwen), **cero costo por uso**

## Cómo funciona

### El daemon

Open Design corre un **daemon local** (proceso en background) que:

1. **Detecta automáticamente** qué CLI tenés activo (Claude Code, Codex, Cursor, etc.)
2. **Inserta** sus capacidades como skills disponibles
3. **Usa el CLI como motor de LLM** — no hace llamadas API propias
4. **Genera artefactos** en `~/open-design/output/`

Resultado: cuando trabajás con Claude Code, **Open Design es un "addon"** que extiende tus capacidades de diseño sin costos extras.

### Local-first

Toda tu data:
- Brand templates
- Outputs generados
- Tu config personal

**Vive en tu compu**. Nada se sube. Privacy total.

## La pregunta clave — ¿gana sobre Claude Design oficial?

| Aspecto | Claude Design oficial | Open Design |
|---------|------------------------|-------------|
| Costo por uso | Tokens de tu plan | **Cero** (si usás modelo local) o "tokens de tu plan" si usás Claude |
| Capacidades | 3-4 (landings, slides, apps clickeables) | 19 |
| Brand templates pre-cargados | 0 | 71 |
| Customización | Limitada | Total (open-source) |
| Privacidad | Pasa por Anthropic | Local |
| UX | Pulida, integrada en claude.ai | Más DIY |
| Soporte / Docs | Oficial | Comunidad |
| Setup | Cero (ya está en tu plan) | 15-30 min |

**Open Design gana en flexibilidad + costo**.
**Claude Design gana en pulidez + integración nativa**.

Si producís diseño operativo intensivo, vale el setup de Open Design. Si es uso ocasional, Claude Design alcanza.

## El truco del costo cero con modelo local

Para **realmente** tener costo cero, conectá Open Design a modelo local con [Ollama](/boveda/cinco-repos-ai-engineer):

```bash
ollama pull qwen3.5-coder
ollama pull llama3.3
```

En config de Open Design:

```json
{
  "engine": "ollama",
  "model": "qwen3.5-coder"
}
```

Ahora cada diseño:
- **No consume tokens de Claude**
- **No consume créditos de nada**
- Corre en tu GPU/CPU local
- Latencia un poco mayor que Claude (5-15s en lugar de 2-3s)

**Trade-off**: calidad un poco menor que Claude/GPT en designs complejos. Para 80% de casos operativos, **suficiente**.

## Casos donde brilla

### 1. Agencia produciendo muchos diseños

Si producís 50+ piezas por semana, el costo de tokens con Claude Design oficial te lima. Open Design + Ollama = **costo marginal cero**.

### 2. Diseñador freelance experimentando

Probar 20 variantes de un landing para presentar a cliente, sin quemar suscripción.

### 3. Educación / aprendizaje

Estudiantes de diseño que no pueden pagar Claude Pro pueden usar Open Design + modelo local gratis.

### 4. Compliance / privacidad

Cliente cuyos diseños no pueden pasar por terceros (NDAs, gobierno, military) → Open Design local cumple.

### 5. Iteración rápida con brand fijo

Si tu brand es "Linear-style" y vas a producir 100 piezas en ese estilo, **cargás el template una vez** y siempre sale consistente.

## Lo honesto — limitaciones

### 1. Setup más complejo

Claude Design es "abrí claude.ai/design y listo". Open Design requiere 15-30 min de setup.

### 2. Comunidad chica

Si encontrás bug o tenés duda, **menos respuestas** que en producto Anthropic oficial.

### 3. Modelos locales tienen techo de calidad

Para diseños donde realmente necesitás "el mejor modelo posible", **Claude/GPT siguen ganando**. Ollama es 80-90% pero no 100%.

### 4. Brand templates pueden estar outdated

Los 71 estilos son snapshot del momento. Si Linear rediseña su website, **el template puede quedar desactualizado**. La comunidad actualiza pero hay lag.

---

## Anti-patrones

### 1. Pedirle "estilo Linear" para producto que NO es Linear

Si tu producto es e-commerce de joyería, **estilo Linear no aplica**. Linear es UI muy específica para SaaS B2B técnico. Usá el template que matchea tu mercado.

### 2. Querer reemplazar Figma

Open Design genera artifacts. **No es editor de diseño profesional**. Para revisión iterativa con un equipo, Figma sigue ganando.

### 3. No customizar para tu marca

Aunque uses "estilo Linear", **tu marca es tuya**. Customizá la paleta, fuentes, tono. Sino sale "Linear-clone" que parece startup random.

### 4. Modelo local sin GPU decente

En CPU sola, los modelos de 20B+ parámetros son **lentísimos** (1+ min por generación). Necesitás GPU decente (8GB+ VRAM) para uso fluido.
