## The model

You don't need money to get started. Just:

1. **Time**
2. **Willingness to learn**
3. **The right tools** — all with a free tier

The flow:

```
1. You build on the free tier
2. You show a working demo to the client
3. The client pays
4. Their payments cover upgrades if you need to scale
```

**Zero upfront investment**. Just your time.

## The step-by-step flow

### 1. Learn

Practice with the tools until you feel comfortable. Do 2-3 personal projects first.

### 2. Build a demo

Build a real project with the stack — something **you can show**. Not "I'm going to build" — finished and working.

### 3. Present it to the client

Show the demo. **A real demo is worth more than a thousand proposals**.

### 4. Scale with revenue

When the client pays, that income covers upgrades if you need them. Vercel Pro, Supabase Pro, Stripe in real production, and so on.

## A real project from start to finish

### Case — landing page for a local client

```
1. Client: "I need a website for my consultancy"
2. You design it in Figma (free)
3. You build it with Claude Code + Next.js
4. DB in Supabase for the contact form (free)
5. Auth with Clerk if they need login (free)
6. Deploy on Vercel (free)
7. Domain + DNS in Cloudflare (DNS free, domain separate)
8. You charge the client with Stripe
9. With that payment you cover upgrades if you scale
```

Your tooling cost during the project: **$0**.

## Where to start

If you've never sold a digital service, **start with landing pages**. They're the easiest thing to sell because:

- Everyone needs them
- Short sales cycle
- Simple technical setup
- Paid per project, not per hour

Then scale up to:
- Apps with auth and a DB
- E-commerce
- Automations
- A small SaaS

## Honest take on the model

### It works if

- You have time to invest in learning (~3-6 months)
- You're willing to do 2-3 free projects at first to build a portfolio
- You know how to sell (or you learn)
- You understand that "free" means a **limited tier** — you pay as you scale

### It does NOT work if

- You're looking for immediate income this week
- You don't have the patience to learn
- You expect clients without any outreach effort
- You don't want to touch code (even with AI, you still have to understand it)

---

## Anti-patterns

### 1. Skipping the "real demo"

Selling without a demo = selling hot air. **Build the demo first**, even a generic one.

### 2. Charging by the hour from day 1

Charge **per project** at first. Hourly billing punishes your speed with AI.

### 3. Paying for pro plans of everything before you have a client

Don't upgrade anything until you **have a client who justifies** the upgrade.

### 4. An incomplete stack

If you pick Supabase, **master it**. Don't jump to Firebase mid-project. Every stack has its quirks.

### 5. Offering "everything"

At first, **one service done well** > 10 services done badly. Specialize first.
