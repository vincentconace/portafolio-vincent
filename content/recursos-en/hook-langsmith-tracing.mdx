## What it is

[LangSmith](https://www.langchain.com/langsmith) is LangChain's observability platform for LLM apps. Traces, evals, debugging — it's built for seriously production-grade AI applications.

This hook automatically sends traces of your Claude Code conversations to LangSmith when a session ends. Hook event: `Stop`.

Prerequisites: `jq` installed + a LangSmith API key.

## How to install

```bash
npx claude-code-templates@latest --hook monitoring/langsmith-tracing
```

Prerequisites:
- `brew install jq` (macOS) or `sudo apt-get install jq` (Linux)
- LangSmith account + API key

For Claude:

```
I installed the monitoring/langsmith-tracing hook. Set up
LANGSMITH_API_KEY. Run a short session and verify that the
trace shows up in my LangSmith project.
```

## What you can do with it

### Observability of your Claude Code usage

LangSmith shows you what you do with Claude: tools used, latency, tokens. Handy for optimizing workflows.

### Productivity eval

How many sessions per day? What are your typical tasks? Which ones take the longest? LangSmith gives you the charts.

### Compare workflows

If you try two approaches (e.g. with vs. without a plugin), LangSmith helps you compare them.

### For teams that have already adopted LangSmith

If your org uses LangSmith for other LLM apps, adding Claude Code centralizes your observability.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **No LangSmith account:** this hook is LangSmith-only.
- **No jq installed:** the hook fails.
- **For casual Claude Code use:** observability is overkill if you only use it a few hours a week.
- **Privacy:** traces include conversations. For sensitive data, check your policy.
- **LangSmith costs:** the free plan has a cap. For heavy usage, you'll pay.

</Callout>
