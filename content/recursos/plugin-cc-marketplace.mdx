## Qué es

CC Marketplace es el marketplace original de Anand Tyagi para Claude Code. No tiene una temática única: junta lo que aporta la comunidad — git workflows, code review, PR creation, debugging, sub-agents de Michael Galpert (los famosos "studio" agents: rapid-prototyper, ui-designer, growth-hacker, etc.), planificación con `plan` y `explore` de Galen Ward, prompt optimization con `lyra`, y más.

Es decir: si en lugar de un marketplace temático ([alirezarezvani](/boveda/5-claude-skills-necesarias) para eng/marketing, [HashiCorp](https://www.aitmpl.com/plugins/agent-skills) para Terraform) querés un cajón con cosas variadas que aportan distintos autores, este es ese cajón.

Cubre 119 plugins. La mayoría son de 1 comando o 1 agent cada uno — son piezas chicas, no toolkits monolíticos.

## Cómo se instala

```bash
# 1) Agregar el marketplace
/plugin marketplace add ananddtyagi/cc-marketplace

# 2) Instalar los plugins que querés (uno por uno)
/plugin install plan@ananddtyagi-cc-marketplace
/plugin install create-pr@ananddtyagi-cc-marketplace
/plugin install lyra@ananddtyagi-cc-marketplace
```

O `/plugin` → Discover.

Para curar con Claude:

```
Instalé el marketplace ananddtyagi/cc-marketplace. Soy [tu rol] trabajando en
[tu proyecto]. De los 119 plugins disponibles, recomendame los 8-10 más
útiles para mi caso, agrupados por categoría.
```

## Qué podés hacer con esto

### Armar tu flujo de PR end-to-end

`plan` → `explore` → implementación → `code-review` → `create-pr` → `fix-pr` (si llegan comentarios). Es el patrón Explore-Plan-Execute pero con plugins curados para cada paso. Buen reemplazo para "voy a hacerlo a mano".

### Stack de agents para construir productos

Si trabajás en algo tipo agencia o producto chico, los agents de Michael Galpert dan un equipo virtual: `rapid-prototyper` para arrancar, `ui-designer` para diseñar, `mobile-app-builder` para empaquetar, `growth-hacker` para distribución, `app-store-optimizer` para publicar. Es la línea ágil completa.

### Trabajar mejor con GitHub Issues

`analyze-issue` te lee el issue y arma una spec. `fix-github-issue` lo implementa con la spec. Atajo decente cuando tu backlog es ruidoso y querés que Claude haga el grunt work de leer y entender.

### Cross-tool: combinar con Codex

El marketplace también incluye plugins que se llevan bien con Codex (igual filosofía que [Claude + Codex en equipo](/boveda/claude-codex-equipo)). Si ya tenés Codex pago, este combo extrae más valor por dólar.

## Beneficios reales

- **Diversidad de autores** — distintas voces, distintos estilos. Si una no te convence, hay alternativas.
- **Plugins chicos y específicos** — instalás lo que usás, descartás el resto.
- **Cubre flujos de día a día** (commit, PR, review, debug) sin pretender ser un framework.
- **No está atado a un dominio** — sirve tanto a un dev solo como a un equipo de startup.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si ya usás otro marketplace gigante** como `alirezarezvani/claude-skills` o `jeremylongshore/claude-code-plugins-plus-skills`, hay solapamiento (debug, code-review, PR helpers). Elegí uno como principal.
- **Heterogeneidad alta = calidad variable.** Los plugins de autores conocidos (Galen Ward, Michael Galpert, Anand) son sólidos. Otros son experimentos. Probá uno antes de adoptarlos en serio.
- **Algunos plugins son redundantes con primitivos nativos** — `plan` vs el `/plan` mode nativo de Claude Code; `double-check` vs hooks. Mirá qué ya tenés antes de duplicar.
- **Si solo querés 2-3 commands específicos**, podés copiarlos en local sin instalar el marketplace entero.

</Callout>
