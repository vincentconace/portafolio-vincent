## Qué es

Sistema para **conectar Meta Marketing API con Claude Code** y operar campañas más autónomamente con **guardrails de seguridad**.

Tagline: *"Este anuncio es ilimitado"*.

| Característica | Detalle |
|----------------|---------|
| Cómo opera | MCP + Claude Code |
| Input | Lenguaje natural |
| Output | Operaciones válidas sobre Meta Ads |
| Estado default | **PAUSED** |
| Confirmaciones | Explícitas para cada cambio |

---

## Qué hace

### 1. Convierte lenguaje natural → operaciones

```
> Creá campaña para "Black Friday 2026". Objetivo: ventas.
  Budget diario: $200. Audiencia: clientes existentes + lookalike 1%.
```

Claude lo traduce a llamadas válidas a la Marketing API.

### 2. Opera la jerarquía Campaign → Ad Set → Ad

Crea, lee, optimiza campañas a los 3 niveles. No solo "crear ad" — entiende **estructura completa** de Meta.

### 3. Aplica guardrails

Para evitar errores caros:
- **PAUSED por default**
- Confirmaciones explícitas
- Límites de gasto
- Validación de policies

### 4. Logs y trazabilidad

Cada cambio queda con ID + razón. Si algo sale mal, **podés rastrear**.

## Setup

### Pre-requisitos

- **Claude Code** instalado y validado con `claude doctor`
- **App de Meta** registrada en developers.facebook.com
- **System User token** generado
- Cuenta de Meta Business

### Paso 1 — Credenciales

```bash
# .env (NO commitear)
META_ACCESS_TOKEN=tu_system_user_token
META_AD_ACCOUNT_ID=act_xxxxx
META_BUSINESS_ID=xxxxx
```

Agregá `.env` a `.gitignore`.

### Paso 2 — Configurar MCP

`.mcp.json`:

```json
{
  "mcpServers": {
    "meta-ads": {
      "command": "node",
      "args": ["./scripts/meta-ads-server.js"],
      "env": {
        "META_ACCESS_TOKEN": "${META_ACCESS_TOKEN}",
        "META_AD_ACCOUNT_ID": "${META_AD_ACCOUNT_ID}"
      }
    }
  }
}
```

### Paso 3 — Verificar

```
> /mcp
```

`meta-ads` debe aparecer como conectado.

## Reglas de seguridad CRÍTICAS

### 1. Siempre PAUSED por default

Cualquier campaña/ad set/ad creado **arranca en PAUSED**. Vos lo activás manual.

### 2. Confirmación explícita

Antes de cualquier write operation, Claude **muestra resumen y espera**:

```
> Voy a crear:
  - Campaña "Black Friday" (PAUSED)
  - Budget diario: $200
  - 2 ad sets: cliente existente + LAL 1%
  - 3 creatives por ad set

  Total gasto si lo activás: $200/día × 7 días = $1,400

  ¿Procedo? [s/n]
```

### 3. Límites de gasto hard-coded

```
MAX_DAILY_BUDGET_PER_AD: $100
MAX_TOTAL_DAILY: $500
```

Si proponés algo por encima, Claude **se rehúsa**.

### 4. Una API call a la vez

Para evitar throttling y patrones de bot:

```
sleep 2 entre cada call
```

Ver [Claude Meta Ads](/boveda/claude-meta-ads) para por qué esto es crítico.

## Anti-patrones

### 1. Confiar en auto-activar

NUNCA. Aunque parezca repetitivo, **vos activás campañas**.

### 2. Sin confirmation prompts

Si Claude detecta que `--no-confirm` está activo, **se rehúsa**. Es por algo.

### 3. Múltiples cambios en paralelo

Meta detecta y banea. **Secuencial siempre**.

### 4. Subir imágenes por API

NO. Ver [Claude Meta Ads](/boveda/claude-meta-ads) para por qué. Generá brief, subí desde Ads Manager.

### 5. Hardcode tokens

NUNCA en código. **Siempre `.env` + .gitignore**.
