## The winning workflow — no direct copy-paste

The typical mistake: copy a prompt from the repo → paste into ChatGPT → mediocre output.

The trick: **personalize with Claude first** before pasting.

```
Repo prompt →  Claude (personalizes it to your case) →  ChatGPT (generates)
```

### Why Claude in the middle

The repo prompt is generic ("product photo, white background, studio lighting"). Your product / brand has its own specs (palette, preferred angle, mood). Claude **adapts** the generic prompt to your context before sending it to GPT.

### Workflow example

#### Step 1 — grab the base prompt from the repo

Category: "Amazon-style product photography"

```
Photo of {product} on white background, professional studio lighting,
soft shadow, sharp focus, photorealistic, commercial photography style
```

#### Step 2 — Claude personalizes it

```
[In Claude]
> Personalize this prompt for my product: running shoes.
  
  My brand: minimalist, eco-friendly, audience runners 25-40.
  Product: unisex shoe, green + cream palette.
  I need 3 variants for an A/B test.

Claude:
  Variant 1 (Amazon-style standard):
  "Photo of minimalist eco-friendly running shoe, green and cream colorway,
  on pure white background, professional studio lighting from upper-left,
  soft shadow under shoe, 45-degree angle, sharp focus, photorealistic,
  commercial photography for e-commerce, 4K"

  Variant 2 (editorial):
  "[same shoe] photographed on textured natural canvas background, golden
  hour lighting, slightly elevated angle showing sole detail, magazine
  editorial style, depth of field, premium feel"

  Variant 3 (action/lifestyle):
  "[same shoe] photographed on running trail at dawn, slight motion blur
  suggesting movement, soft natural lighting, environmental context,
  authentic lifestyle photography, Nike Sandtrap meets Allbirds style"
```

3 variants with their own personality from a generic prompt.

#### Step 3 — Paste into ChatGPT

Each variant is generated separately in GPT Image 2. You compare outputs. Iterate on the one that gets closest.

## Anti-patterns

### 1. Direct copy-paste without personalizing

The repo prompt is a **base**. Without Claude personalizing it, you end up with an image "that looks like a stock photo brand".

### 2. Trying 1 prompt and giving up

GPT Image 2 (like any image gen) **requires iteration**. The first output of a new prompt is rarely the final one. Iterate 3-5 times.

### 3. Not saving what works

When you find a prompt that gives you perfect output, **save it**. For recurring use, keep your own `mis-prompts/` folder with the winners.

### 4. Ignoring the difference between models

A prompt that works in GPT Image 2 may not work the same in Midjourney or Flux. **The prompt is model-specific**. For Claude / Anthropic: they have different tags, better to ask Claude to adapt it.

## The Claude component that adds value

The most important part of the workflow isn't the repo (4400 prompts is a lot but you can find them elsewhere). **It's Claude personalizing them**.

Without Claude:
- You apply the generic prompt → generic output

With Claude:
- You give it context about your brand, audience, constraints
- It generates 3-5 variants thinking about your specific case
- You compare and pick the best base
- You iterate more easily because you have a strong starting point

**80% of the value is in this intermediate step**, not in the repo or in GPT.
