## What it is

Claude Octopus is the closest thing to an LLM "switchboard" inside Claude Code that you'll find. The author (nyldn) built it so you're not tied to a single model: it integrates 8 different providers and lets you decide which one to use for each step of the workflow.

The 8 providers it covers:
- Claude (official)
- OpenAI Codex
- Google Gemini
- Perplexity
- OpenRouter (gateway to several)
- GitHub Copilot
- Qwen (Alibaba)
- Ollama (local)

On top of that, it ships 47 commands and 50 skills, all designed around Double Diamond workflows (Discover → Define → Develop → Deliver). It's not just "hit another model" — it's an orchestration layer with complete flows: brainstorm, PRD, parallel agents, debate, sentinel, factory, and more.

If your AI usage already spans several tools and you want to unify them in a single environment, this is the plugin.

## What you can do with this

### Model debate over your spec

`skill-debate` launches several providers in parallel, each with a different role (proposer, challenger, judge), and returns a verdict. More robust than asking a single model to "critique this."

### Research with Perplexity + synthesis with Claude

`skill-deep-research` uses Perplexity to find up-to-date info and then hands the findings to Claude to synthesize. Useful when Claude doesn't have recent info on something.

### Cross code review

`skill-code-review` + `skill-staged-review` with two different models = more coverage. If Codex and Claude agree, high confidence. If they disagree, you take a look.

### Complete Double Diamond workflows

Discover → Define → Develop → Deliver with `flow-*` commands. It's a design framework applied to development: it structures the problem before jumping to solutions.

### Costs under control

`skill-cost-projections` tells you how much running the workflow will cost before you launch it. Crucial when you're moving work across 8 providers.

## Real benefits

- **The most complete multi-LLM option** in the catalog (8 providers).
- **It doesn't lock you into one model** — if Claude goes down or raises prices tomorrow, you have active alternatives.
- **47 commands + 50 skills** cover practically the entire dev cycle.
- **Structured Double Diamond workflows**, not ad-hoc.
- **Built-in cost projections** — the only serious way to avoid burning money with multi-LLM.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you only use Claude:** most of Octopus's value is in orchestrating several models. If your setup is Claude-only, Anthropic's official plugins already cover you with less overhead.
- **Costs:** running 8 providers in parallel can multiply your spend. Check `skill-cost-projections` before enabling everything.
- **Initial setup:** you'll need API keys or configuration for each provider you activate. It's not fully plug-and-play.
- **Steep learning curve:** 50 skills is a lot. Start with the `flow-*` commands and a couple of skills, not everything at once.

</Callout>
