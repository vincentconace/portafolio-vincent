## What it is

Unlike the [AI Bash Guard](/boveda/hook-ai-bash-guard) (which uses a Claude sub-agent to evaluate), this hook uses a **multi-level deterministic blocklist** to block catastrophic commands. Patterns:

- `rm -rf /` and obvious variations
- `dd if=...` writing directly to disk
- `mkfs.*` that formats filesystems
- `:(){ :|:& };:` (fork bomb)
- Operations on critical system paths

Hook event: `PreToolUse` on Bash. Instant blocking with zero token cost (it doesn't use an LLM).

Its 820 installs make it the second most adopted security hook.

## How to install it

```bash
npx claude-code-templates@latest --hook security/dangerous-command-blocker
```

For Claude:

```
I installed the security/dangerous-command-blocker hook. Test it: ask
Claude to run `rm -rf /tmp/test` — it should be blocked. Then try
`ls /tmp/test` — that should go through.
```

## What you can do with this

### Basic defense in depth

Like painting every Claude Code session. The cost is zero (it consumes no tokens), and the benefit can be huge (it stops you from wiping your home directory).

### Combo with [AI Bash Guard](/boveda/hook-ai-bash-guard)

Deterministic (this one) + AI-driven (Guard) = a double safety net. The deterministic one catches the obvious cases; the AI one covers creative variations.

### For autonomous sessions

If you leave Claude running with `/goal`, this hook is basically mandatory. Without it, a wrong decision by the model is catastrophic.

### Zero cost, high benefit

It consumes no tokens. There's no reason not to install it unless your workflow legitimately needs `rm -rf` etc. (rare).

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your legitimate workflow uses `dd`, `mkfs`, etc.:** genuine sysadmins may need them. The hook blocks without distinguishing intent.
- **For ephemeral container shells:** if Claude operates in a container that gets discarded, destructive commands are less risky. Even so, the hook does no harm.
- **If you expect a lot of false positives:** look at the patterns first.

</Callout>
