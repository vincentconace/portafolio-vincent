## The problem

When you hand a PDF, Word, or Excel file to Claude **directly**, it has to read the whole file. That eats up **a ton** of your account without you noticing.

For example:
- 50-page PDF → ~30,000 tokens on input alone
- Word with tables → extra tokens for structure
- Excel with formulas → even worse

**MarkItDown** is an open source translator from Microsoft. It converts any file into **clean Markdown** that Claude reads **lightning fast** while **spending almost nothing**.

Repo: `microsoft/markitdown`. 100k+ stars. Open source.

## The real savings

For a typical 30-page PDF:

| Approach | Input tokens | Quality |
|----------|--------------|---------|
| **PDF directly to Claude** | ~20,000 | Good but pricey |
| **MarkItDown → Markdown → Claude** | ~5,000 | Same or better |

**Savings: ~75%** in input tokens. For heavy use, it transforms your quota.

Bonus: Claude **understands it better** because the format is native (Markdown vs a parsed PDF).

## Installation — paste it to Claude

You don't need to know how to code. Copy-paste:

```
Install Microsoft's MarkItDown tool from
https://github.com/microsoft/markitdown on my computer.

Then use it every time I ask you to convert a file (PDF,
Word, Excel, PowerPoint, image, audio, or video) to Markdown
before reading it.

Let me know when you're done installing it.
```

Claude:
1. Goes to `github.com/microsoft/markitdown`
2. Downloads it
3. Installs it with pip
4. Confirms when it's ready

**You don't type anything technical**. You just approve permissions to run commands (it's only the installation).

Works in Claude Code, Codex, Cursor, Claude Cowork — any compatible agent.

## How to use it

Once it's installed, instead of dragging the file in, you tell it: **"use MarkItDown to convert this file"**.

Claude translates it into clean text first and then reads it. That extra step saves you **a lot of tokens**.

### Convert a PDF and summarize

```
> Use MarkItDown to convert /Users/me/Desktop/contract.pdf
  to Markdown.
  Then read it and summarize it for me in simple English with
  the most important points.
```

### Convert Excel and analyze

```
> Convert /Users/me/Desktop/report.xlsx with MarkItDown and
  tell me the 5 most important things you see in the table,
  in simple words.
```

### YouTube transcription

```
> Use MarkItDown with this YouTube link
  https://www.youtube.com/watch?v=XXXX
  Pull the transcript and give me a summary in short bullets.
```

### Word to Markdown

```
> Convert /docs/proposal.docx to Markdown with MarkItDown.
  Save it as /docs/proposal.md.
  Keep the tables and formatting.
```

### Audio to text

```
> Here's a recording /audio/meeting-2026-05-28.mp3.
  Use MarkItDown to transcribe it.
  Then give me:
  1. Executive summary
  2. Decisions made
  3. Action items with owners and deadlines
```

### Image with text (OCR)

```
> Here's a screenshot /screenshots/whiteboard.png with
  handwritten notes from a meeting.
  Use MarkItDown to extract the text (OCR).
  Then structure it for me in outline format.
```

## Anti-patterns

### 1. Forgetting to invoke MarkItDown

If you just say "read this PDF," Claude reads it directly (spending everything). **Tell it explicitly** to "use MarkItDown".

Workaround: add it to CLAUDE.md:

```markdown
## File rule
Whenever a PDF, Word, Excel, PowerPoint, image, audio, or video
file is mentioned, USE MarkItDown to convert it to Markdown
BEFORE processing it.
```

### 2. Converting trivial files

For a 1-page PDF, the MarkItDown overhead may not pay off. Use it for **medium to large** files.

### 3. Trusting image OCR 100%

OCR is good but **not perfect**. For critical text (legal, medical), validate manually.

### 4. Long audio without splitting it up

For 3+ hour audio, **split it first** into chunks. MarkItDown can take a while and consume a lot.

### 5. Scanned PDFs with bad OCR

If the PDF is a poorly scanned one, MarkItDown extracts broken text. For those: pre-process with better OCR (Tesseract, Adobe).
