## Qué es

Trace MCP se posiciona como una capa de code intelligence **framework-aware**: entiende no solo el lenguaje (TypeScript, Python, etc.) sino también el framework (Next.js, Django, Rails, etc.) y aplica esa awareness al análisis.

Las claims del autor:
- **60 framework integrations**
- **81 languages soportados**
- **Hasta 99% accuracy** en semantic navigation, impact analysis y refactoring

Es nuevo (7 installs solamente al momento del scrape), pero las claims son ambiciosas. Si funciona como promete, es la opción más completa del catálogo para code intelligence.

Compará con:
- [Serena](/boveda/mcp-serena) (689 installs) — semantic retrieval + editing
- [DeepGraph](/boveda/mcp-deepgraph-react) — análisis estructural framework-specific
- [Claude Code LSPs](/boveda/plugin-claude-code-lsps) — LSP individual por lenguaje

## Qué podés hacer con esto

### Impact analysis framework-aware

"Si cambio el shape de este Prisma model `User`, qué archivos del codebase se rompen? Cubre routers tRPC, components React que usan tipos derivados, schemas Zod".

### Refactoring seguro cross-framework

"Renombrar `apiClient` a `httpClient` en todo el codebase. Respetar nombres en docs, imports, JSDoc, y los nombres en `package.json` scripts si aparece".

### Semantic navigation

"Buscame todas las funciones que manejan auth across el stack. Identificá las que cruzan boundaries (frontend usa una función auth del backend via tRPC)".

### Detección de patterns

"Identificá componentes React que mezclan client + server state inadecuadamente". Trace puede detectar smells framework-specific.

### Comparado con [Serena](/boveda/mcp-serena)

Serena hace semantic retrieval + editing genérico. Trace agrega framework awareness. Para un proyecto monolítico con un framework dominante, Trace puede dar valor extra. Para multi-language genérico, Serena.

## Pre-requisitos

- **Proyecto con un stack identificable** (Trace funciona mejor cuando reconoce el framework)
- **Lenguajes y frameworks dentro de los 60+/81 soportados** — verificá la lista en el JSON o la doc oficial

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Solo 7 installs:** muy poca comunidad. Las claims son altas pero todavía no se probaron a escala. Probá en sandbox.
- **Para proyectos pequeños:** alcanza con [LSPs](/boveda/plugin-claude-code-lsps).
- **Stack muy nicho:** verificá que tu framework esté en la lista soportada.
- **"99% accuracy" es una claim del autor:** no hay benchmark público todavía. Tomá con grano de sal.

</Callout>
