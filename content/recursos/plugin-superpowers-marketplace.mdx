## Qué es

Superpowers Marketplace es la curación personal de **obra** (Jesse Vincent, una figura conocida del ecosistema Anthropic). Es chico (9 plugins) pero cada uno está pensado y mantenido con cuidado. El estilo es "lo que yo uso todos los días". Como el Superpowers original (también en este marketplace) está incluido en el [Claude Plugins Official](/boveda/plugin-claude-plugins-official) de Anthropic, este marketplace es donde encontrás versiones más recientes y los hermanos experimentales.

Los 9 plugins:
- `superpowers` — el core: TDD + debugging + collaboration patterns
- `superpowers-chrome` — Chrome DevTools Protocol directo (BETA)
- `episodic-memory` — memoria semántica entre sesiones
- `elements-of-style` — escritura clara basada en Strunk Jr. (1918)
- `superpowers-lab` — experimentales: tmux automation, MCP discovery, Slack, headless Windows VM
- `superpowers-developing-for-claude-code` — para construir plugins/skills/MCPs
- `superpowers-dev` — DEV branch (no instalar con otras versiones)
- `claude-session-driver` — controlar otras sesiones de Claude Code como workers vía tmux
- `double-shot-latte` — elimina "Would you like me to continue?"

## Cómo se instala

```bash
# 1) Marketplace
/plugin marketplace add obra/superpowers-marketplace

# 2) Core + 2-3 que te sirvan
/plugin install superpowers@obra-superpowers-marketplace
/plugin install episodic-memory@obra-superpowers-marketplace
/plugin install double-shot-latte@obra-superpowers-marketplace
```

## Qué podés hacer con esto

### TDD red/green automático

`superpowers` trae el patrón TDD red/green completo: escribís el comportamiento esperado, Claude genera el test que falla (red), después implementa hasta que pasa (green). Es la versión opinionada del autor.

### Brainstorming + debugging sistemático

El plugin propone patrones específicos: cuando arrancás algo nuevo, brainstorming guiado; cuando algo se rompe, debugging con hipótesis + verificación, no "probemos cosas".

### Chrome directo sin intermediarios

`superpowers-chrome` te da CDP (Chrome DevTools Protocol) directo. Más bajo nivel que Playwright. Para tareas donde necesitás control fino del browser sin las abstracciones de Playwright. BETA: testeado ligeramente, usalo con criterio.

### Memoria semántica entre sesiones

`episodic-memory` indexa tus conversaciones pasadas con semantic search. Cuando pedís "lo que hablamos sobre X la semana pasada", recupera el contexto relevante. Compará con [Claude Mem](/boveda/plugin-claude-mem) — son enfoques distintos del mismo problema.

### Escritura que respeta los clásicos

`elements-of-style` aplica las reglas de Strunk Jr. cuando Claude escribe. Resultado: docs, READMEs, copy más concisos. Si te molesta el "lenguaje IA inflado", este plugin es un aliado.

### Workers de Claude controlados vía tmux

`claude-session-driver` te deja lanzar otras sesiones de Claude Code como workers que vos controlás desde una sesión maestra. Multi-agent con sesiones de verdad, no agents simulados.

### Adiós a "Would you like me to continue?"

`double-shot-latte` elimina las interrupciones donde Claude pregunta si seguir. El propio plugin evalúa si tiene sentido continuar y decide. Para sesiones largas autónomas, libera fricción.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **`superpowers-chrome` está marcado BETA** — usalo con criterio en código serio.
- **`superpowers-dev` requiere desinstalar otras versiones** primero. Pelea de versiones.
- **`double-shot-latte` puede llevar a runs muy largos**: si Claude decide siempre continuar, podés gastar tokens sin darte cuenta. Configurá un cap.
- **`claude-session-driver` requiere tmux** instalado y funcional. No funciona en Windows nativo bien.
- **Solapamiento `episodic-memory` vs `claude-mem` vs memoria nativa:** evalúa primero qué te cubre la memoria nativa antes de capar con uno o ambos.

</Callout>
