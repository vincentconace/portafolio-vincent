## Qué es

Equivalente a [Discord Error Notifications](/boveda/hook-discord-error-notifications) para Slack. Avisa cuando:
- Long-running operations (Claude tarda demasiado)
- Tools toman significant time
- Notificaciones nativas de Claude Code

Hook events: `PreToolUse` + `PostToolUse` + `Notification`.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook automation/slack-error-notifications
```

Pre-requisito: webhook Slack.

Para Claude:

```
Instalé el hook automation/slack-error-notifications. Configurá
SLACK_WEBHOOK_URL. Mostrame los thresholds que disparan notification.
```

## Qué podés hacer con esto

### Detección de stuck sessions

Si Claude lleva 5+ min en una operación, problemamente algo va mal. El notification te avisa.

### Catch errors automatic

Cuando Claude dispara una notification interna, el hook la rebota a Slack.

### Para sesiones autónomas

`/goal` largos: el hook avisa si algo se queda colgado.

### Combo con Slack Notifications regulares

Si tenés también [Slack Notifications](/boveda/hook-slack-notifications) (success básico), separás success de errors en canales distintos.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si querés notifications de éxito:** este es solo errors. Usá los otros Slack hooks.
- **Sin webhook configurado:** no hace nada.
- **Si tu canal tiene mucho ruido:** evaluá thresholds primero.
- **Para uso casual:** menos crítico que para sesiones automáticas.

</Callout>
