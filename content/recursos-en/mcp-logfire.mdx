## What it is

[Logfire](https://logfire.pydantic.dev) is Pydantic's observability platform. Built for modern Python apps (FastAPI, PydanticAI), it uses OpenTelemetry as its foundation and adds magic auto-instrumentation features.

This MCP connects your Logfire to Claude Code. Query traces, metrics, logs, anomalies. Useful when your observability lives in Logfire and you want that context in your editor.

## How to install

```bash
npx claude-code-templates@latest --mcp devtools/logfire
```

Prerequisite: Logfire account + API token (write/read token depending on usage).

For Claude:

```
I installed the devtools/logfire MCP. Configure the credentials and show me
the slowest traces from the last day, grouped by route. For the
top 3, give me the dominant span.
```

## What you can do with this

### Latency diagnosis

"On the `/api/checkout` route, which span contributes most to p95 latency? Show me concrete examples." More direct than opening the dashboard.

### Anomaly detection

"Compare yesterday's latency vs last week. Are there regressions on any endpoint?" A time diff.

### Structured logs

Logfire integrates logs as spans. "Find me errors in the last hour that mention `database connection`." Not LogQL, Logfire's own.

### Trace exploration

"Take the trace with id X. Show me the full tree of spans with duration and tags." Useful for debugging specific requests.

### Integration with FastAPI / PydanticAI

If your stack is FastAPI with Logfire instrumented, endpoints show up as rich traces. The MCP lets you query over them.

### Combining with [Dynatrace](/boveda/mcp-dynatrace) / [Grafana](/boveda/mcp-grafana)

If you use Logfire for Python apps and another stack for infra, you can combine several observability MCPs. Claude correlates.

## Prerequisites

- **Logfire account** with a suitable plan
- **Correct API token**
- **Instrumented apps** with OTel/Logfire (the MCP queries what's already being reported)

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your observability is elsewhere:** Logfire-only. For New Relic, Datadog, Sentry, there are specific MCPs or nothing.
- **If your stack is non-Python:** Logfire supports multiple languages, but its sweet spot is Python. Validate your case.
- **97 installs is moderate:** a moderately-tested community. For critical production, verify that the tools you need are available.
- **For alerts:** Logfire has alerting, but the MCP is for querying. Routing to humans still happens via the Logfire UI or PagerDuty.

</Callout>
