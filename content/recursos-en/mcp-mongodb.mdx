## What it is

MongoDB is the most widely used document-oriented NoSQL database. Atlas is its managed cloud service. This MCP connects Claude Code to any MongoDB instance (local, self-hosted, Atlas).

Typical capabilities:
- **Queries** in natural language
- **Aggregations** (Mongo's most powerful feature)
- **Collection management** (create, drop, index)
- **Schema inspection** (Mongo isn't really schemaless — documents have shape)

319 installs make it the most adopted MongoDB MCP in the catalog.

## How to install it

```bash
npx claude-code-templates@latest --mcp devtools/mongodb
```

Prerequisites: a connection string to your MongoDB.

For Claude:

```
I installed the devtools/mongodb MCP. Set MONGODB_URI to [your URI].
List the databases and collections. For `users`, show me the typical
document shape (sampling 5 docs).
```

## What you can do with this

### Exploring a NoSQL schema

"For the `orders` collection, sample 100 docs and give me the inferred shape. Identify optional vs. required fields and the typical types."

### Queries in natural language

"Find me the 20 users with the most orders in the last month who haven't bought in the last 30 days." Claude translates it to `find()` + filters.

### Complex aggregations

"For each product category, calculate total Q1 revenue and the top 3 products. Output as a table." An aggregation pipeline gets generated.

### Index usage analysis

"List the slowest queries from the last day based on `db.currentOp` or the profiler. Suggest indexes that could help."

### Migration assistant

"For the `users-v1` collection, generate the transformation script to `users-v2`, adding a `tenant_id` field from a lookup in another collection."

### Atlas-specific

If you use Atlas, there are additional capabilities: cluster info, performance advisor recommendations, backup status. Check the JSON.

### Compared to [DBHub](/boveda/mcp-dbhub)

DBHub covers relational databases (Postgres, MySQL, etc.). The MongoDB MCP is NoSQL-specific. If your stack is hybrid (which is common), install both.

## Prerequisites

- **Connection string** valid for your MongoDB
- **Credentials** with the right permissions (read-only for exploration, write for mutations)
- **For Atlas:** API keys if you're going to use cluster management capabilities

## When it's NOT a good fit

<Callout type="danger" title="Before connecting Claude to a production DB">

- **Destructive operations:** `db.collection.drop()`, `deleteMany({})`, etc. Catastrophic if something goes wrong. ALWAYS use approval hooks for writes in prod.
- **Broad permissions:** a user with readWriteAnyDatabase has total power. Restrict the scope.
- **Queries that return huge collections:** you can exhaust memory or context. Limit with `.limit()`.
- **If your DB is relational:** [pg-aiguide](/boveda/plugin-pg-aiguide) or [DBHub](/boveda/mcp-dbhub) are the right choice.
- **Compliance:** queries over sensitive data can end up in history. Check your policies.

</Callout>
