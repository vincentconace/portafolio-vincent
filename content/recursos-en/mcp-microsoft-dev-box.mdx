## What it is

Microsoft Dev Box are cloud-hosted development environments (preconfigured Windows VMs) that many companies use instead of physical machines for their devs. The upside: a new developer is onboarded in minutes with a ready-to-go environment instead of spending 2 days setting up locally.

This MCP gives you natural language operation over Dev Boxes: create, suspend, configure, manage pools. Useful for Dev Box admins and for devs who want to operate their box without opening the Azure portal.

## How to install it

```bash
npx claude-code-templates@latest --mcp devtools/microsoft-dev-box
```

Prerequisites:
- Authenticated Azure CLI (`az login`)
- Subscription permissions for Dev Box
- Dev Box already configured in your org

For Claude:

```
I installed the devtools/microsoft-dev-box MCP. List my available
Dev Boxes with their pool, status and last used. If any has gone
more than 7 days without use, tell me whether it's worth suspending
it to save on cost.
```

## What you can do with this

### Managing personal Dev Boxes

"Suspend my main Dev Box (I won't use it over the weekend) and restore it for me Monday at 9 am". Trading-style operations to save money.

### Assisted onboarding

"Provision a new Dev Box for [name], with the `frontend-team` pool. Once it's ready, send me the RDP/URL so they can connect". Admin operation.

### Usage and cost audit

"List inactive Dev Boxes (no use for more than 14 days). Identify the estimated cost of each one and candidate to delete".

### Pool management

"For the `data-team` pool, what's the current utilization? Is it worth scaling the cap up or down?".

### Configurations

"Create a new configuration for Java devs: including IntelliJ, Maven, JDK 17 and Docker. Apply it to a new `java-team` pool".

### Combining with [AKS MCP](/boveda/mcp-azure-kubernetes-service)

If your Azure infra is Dev Box + AKS, the two MCPs give you cross-stack operation from Claude.

## Prerequisites

- **Azure account** with a plan suitable for Dev Box
- **Azure permissions** (Dev Box Project Admin or equivalents depending on the operation)
- **Pools and configurations** already defined (the MCP doesn't set up infra from scratch)

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you don't use Dev Box:** obviously. For devboxes on another cloud (GitHub Codespaces, Gitpod, Replit), different MCPs or none.
- **Destructive operations:** delete Dev Box, delete pool. Approval hooks.
- **Broad permissions:** Dev Box Admin has serious power. If your role allows deleting entire pools, Claude inherits that.
- **For casual devs:** the Azure portal may be more convenient. The MCP shines at admin tasks or when you combine it with automation.
- **Costs:** Dev Box isn't free. Operations that provision resources generate billing.

</Callout>
