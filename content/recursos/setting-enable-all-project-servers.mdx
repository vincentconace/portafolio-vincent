## Qué es

`enableAllProjectMcpServers: true` — todos los MCP servers definidos en `.mcp.json` del proyecto se activan sin prompt manual.

Útil para teams que definen MCPs como parte del repo (similar a `package.json` para libs): cuando alguien clona, los MCPs ya quedan activos.

## Qué podés hacer con esto

### Onboarding de team simplificado

Dev clona repo → MCPs activos sin pasos manuales.

### Reproducibility de environment

Lo que está en `.mcp.json` es la fuente de verdad. Todos los devs tienen el mismo setup MCP.

### Combo con [Disable Risky Servers](/boveda/setting-disable-risky-servers)

Enable All para el default + Disable Risky para excepciones específicas. Allowlist + blacklist.

### Para CI / sandboxes

CI que corre Claude Code en checked-out repo: este setting evita stuck por approval prompts.

## Cuándo NO conviene

<Callout type="danger" title="Antes de habilitar">

- **Confianza en .mcp.json:** auto-approve significa que si alguien agrega un MCP malicioso al `.mcp.json` del repo, se activa sin que vos lo veas. Reviá PRs que toquen ese file.
- **Para projects en untrusted networks:** no auto-aprobés MCPs en repos compartidos sin review.
- **Para uso conservador:** approval manual es más seguro.
- **Combo con review process:** si tu equipo no hace PR review serio, este setting puede ser vector.

</Callout>
