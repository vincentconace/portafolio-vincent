## Qué es

Notion es la plataforma elegida por muchos equipos para wikis, docs, databases híbridos y project management ligero. Este MCP **oficial** de Notion da acceso a su API para que Claude opere tu workspace.

Capabilities:
- **Buscar** páginas y databases
- **Crear / editar** docs
- **Manejar databases** (filtros, ordering, items)
- **Comments y mentions**
- **Permissions y workspace ops**

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp productivity/notion
```

Pre-requisitos: cuenta Notion + crear una "internal integration" en [notion.so/my-integrations](https://www.notion.so/my-integrations) + compartir las páginas relevantes con esa integration.

Para Claude:

```
Instalé el MCP productivity/notion. Configurá NOTION_TOKEN. Listame las
databases visibles en mi workspace. Para la database `Projects`,
mostrame los items con status `In Progress` que tienen due date esta
semana.
```

## Qué podés hacer con esto

### Búsqueda y referencia cruzada

"Buscame en mi workspace todas las notas que mencionen `OAuth 2.0`. Mostrame solo las modificadas en los últimos 30 días". Más rápido que la búsqueda nativa de Notion.

### Generación de docs desde código

"Para el endpoint `/api/users` que recién implementé, generame doc en Notion bajo la página `API Reference` con: descripción, params, response shape, ejemplos". Útil para docs vivos.

### Update de databases con context

"En la database `Roadmap`, agregame los items del backlog que cerré este sprint. Para cada uno, status=`Done`, due=hoy, asignee=yo, link al PR de GitHub".

### Sync entre Notion y código

"Para cada PR que tenga el label `feature`, creame una entrada en la database `Release Notes` con resumen del PR".

### Templates dinámicos

"Generame un nuevo proyecto en Notion siguiendo mi template `Sprint Plan`: 1 página principal, sub-páginas para goals/risks/timeline, database vinculada para tasks".

### Combo con [Google Workspace](/boveda/mcp-google-workspace)

Notion + Calendar + Gmail = workflow knowledge worker completo cuando Claude orquesta los tres.

## Pre-requisitos

- **Cuenta Notion** activa
- **Internal Integration token** (Notion Settings → My integrations)
- **Páginas compartidas con la integration** (Notion requiere esto explícitamente)
- **Permisos correctos** según workspace

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Notion:** obvio. Para Confluence, Slite, otros, otros MCPs.
- **Permisos no se heredan automático:** tenés que compartir cada page/database con la integration. Pasos manuales.
- **Operaciones destructivas:** delete page, archive masivo. Hooks de aprobación.
- **Para personal wikis chicos:** el MCP es overhead. Notion UI alcanza.
- **Datos sensibles:** el token deja a Claude ver lo compartido. Mirá qué exactamente le diste acceso.

</Callout>
