## What it is

The simplest statusline option: it only shows the model name and the current directory. No git info, no context bars, no gimmicks.

For devs who want maximum focus on the content, without distractions.

## Summary table

| Field | Value |

## How to install it

```bash
npx claude-code-templates@latest --setting statusline/minimal-statusline
```

For Claude:

```
I installed the statusline/minimal-statusline setting. Show me how it looks.
```

## What you can do with it

### Maximum focus

No extra info to distract you from the content.

### For users who already know their context

If you keep a mental track of which branch you're on and your context %, you don't need to see it all the time.

### Intentionally minimalist statusline

For devs who prefer a clean UI — orthogonal to the gimmicky or info-rich ones.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you want context % awareness:** use [Context Monitor](/boveda/setting-context-monitor).
- **If you want git awareness:** [Git Branch Statusline](/boveda/setting-git-branch-statusline).
- **If having NO info bothers you:** the other statuslines may be a better choice.

</Callout>
