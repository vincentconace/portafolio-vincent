## Qué es

**Antigravity Awesome Skills** es un repositorio con **978+ archivos markdown** de instrucciones. Cada archivo le enseña a tu agente de IA a hacer una tarea específica.

Pensalo así: cada skill es como **contratar un consultor experto bajo demanda**. Tu agente los lee automáticamente y los usa cuando son relevantes para lo que estás haciendo.

Repo: `sickn33/antigravity-awesome-skills`. ~32k estrellas.

## Instalación

Abrí terminal en la carpeta de tu proyecto y corré según tu herramienta:

### Claude Code

```bash
npx antigravity-awesome-skills --claude
```

### Cursor

```bash
npx antigravity-awesome-skills --cursor
```

### Gemini CLI

```bash
npx antigravity-awesome-skills --gemini
```

### Codex

```bash
npx antigravity-awesome-skills --codex
```

### Otros agentes compatibles

```bash
npx antigravity-awesome-skills
```

El instalador te deja **elegir qué categorías instalar**. **NO instales las 900+**. Elegí 2-3 categorías relevantes para tu proyecto.

## Cómo usar un skill

Una vez instalados, usás la sintaxis `@skill-name` para activar un skill en tu prompt:

### Ejemplos

```
@api-design diseñá la API para mi app de pedidos
```

```
@security-audit revisá este endpoint por vulnerabilidades
```

```
@pricing-strategy ayudame a definir precios para mi SaaS
```

```
@test-generation generá tests para este módulo
```

```
@refactor-suggestion mirá este código y sugerí refactor sin
romper funcionalidad
```

## Estrategia recomendada de adopción

### Día 1 — Instalá 2 categorías core

Las 2 que matchean **tu trabajo principal**. No más.

### Semana 1 — Probá 5-10 skills

Identificá cuáles realmente usás. La mayoría no aplican a vos.

### Semana 2 — Refinás

Quedate con los **20-30 skills** que más usás. Desinstalá los demás (ocupan contexto).

### Mes 2 — Categorías secundarias

Si tu workflow expande, sumá otra categoría.

### Mes 3+ — Cree los tuyos

Los skills de Antigravity son base. Para tu workflow específico, **crea tus propios** ([crear skills](/boveda/crea-claude-skills)).

## Lo bueno y lo no tan bueno

### Pros

- ✅ **Cantidad masiva** — 978+ es difícil de superar
- ✅ **Open source y mantenido** — community activa
- ✅ **Multi-agente** — funciona con todos los principales
- ✅ **Categorías bien organizadas** — fácil filtrar
- ✅ **Setup trivial** — `npx` one-liner

### Cons honestos

- ⚠️ **Calidad variable** — algunos skills son geniales, otros son básicos
- ⚠️ **Choice paralysis** — 978 opciones puede paralizar
- ⚠️ **Genericidad** — al ser para "todo el mundo", no aplica a tu nicho específico
- ⚠️ **No reemplaza skills custom** — tu workflow propio necesita tus propios skills

## Anti-patrones

### 1. Instalar las 9 categorías "por las dudas"

Cada skill ocupa contexto. Con 978 activos, **Claude se distrae** intentando matchear. Instalá 2-3 categorías relevantes.

### 2. No buscar antes de pedir

Antes de pedir algo, **listá los skills disponibles**:

```bash
npx antigravity-awesome-skills --list
```

A veces lo que necesitás ya existe. Pedirle a Claude sin invocar el skill correcto = no aprovechás Antigravity.

### 3. Olvidarse de remove

Si instalaste una categoría y no la usás, **removela**. Cada categoría aporta peso al contexto.

### 4. Crear tu skill cuando Antigravity ya lo tiene

Antes de armar tu propio `@deploy-vercel`, **chequeá** si Antigravity tiene `@vercel-deployment`. Si ya existe, ahorrate el setup.

### 5. Esperar magia con prompt vago

`@security-audit hacé tu cosa` → output mediocre.

`@security-audit revisá mi endpoint /api/orders por:
- SQL injection
- IDOR
- Auth bypass
- Rate limiting
Reportá con severity y fix sugerido` → output útil.

**El skill activa el conocimiento. El prompt activa el formato y profundidad**.

## Combinaciones potentes

### Con tus skills custom

```

├── (instalados de Antigravity)
│   ├── api-design.md
│   ├── pricing-strategy.md
│   └── security-audit.md
└── (tus custom)
    ├── nuestra-arquitectura.md
    ├── brand-tone.md
    └── client-onboarding.md
```

### Con [Sequential Thinking](/boveda/claude-fabrica-a-pro)

Sequential Thinking + skill apropiado = razonamiento estructurado **con expertise específica**.

## Cuándo NO conviene

| Caso | Por qué |
|------|---------|
| Querés calidad uniforme alta | Antigravity tiene varianza |
| Tu nicho es muy específico | Generic skills no encajan |
| Prefieres curaduría manual | Demasiada cantidad |
| Workflow ya tiene skills custom maduros | Overlap innecesario |

Si tenés tu propio set de skills probados, Antigravity es **complementario**, no central.
