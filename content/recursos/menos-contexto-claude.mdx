## El problema

Imaginate que tu escritorio tiene espacio limitado. Cada vez que Claude Code usa una tool (leer archivo, screenshot Playwright, revisar issues GitHub), el resultado entra a la conversación y ocupa espacio en ese escritorio. Ese espacio = **contexto**.

Los resultados son **enormes**:
- Un screenshot Playwright: **56 KB**
- Lista de issues GitHub: **59 KB**

Con pocas llamadas, tu escritorio se llena y:
- Claude **pierde el hilo**
- **Olvida decisiones**
- Sesión se reinicia → empezás de nuevo

---

## La solución — context-mode

MCP server open source que se instala como plugin. **3 pilares**:

### Pilar 1 — Sandbox (ejecución aislada)

Tu asistente lee reporte de 100 páginas y te da resumen de 1. Eso hace el sandbox: **comandos se ejecutan en entorno aislado** y solo el resultado procesado entra a tu conversación. **Los datos crudos nunca tocan tu contexto**.

Soporta 11 lenguajes: JavaScript, Python, Go, Rust, Ruby, PHP, Java, C#, C++, Haskell, Julia.

### Pilar 2 — Indexación inteligente

Cuando un resultado pesa **más de 5 KB**, lo indexa con **ranking BM25** (la tech de motores de búsqueda) y te devuelve solo las partes relevantes.

| Sin | Con |
|-----|-----|
| 315 KB de datos crudos | **5.4 KB** indexados |

**98% menos contexto**.

### Pilar 3 — Memoria de sesión

Cada archivo editado, operación git, tarea y decisión se guarda en SQLite. Cuando el contexto se compacta, **guarda foto del estado** (max 2 KB) con todo lo crítico.

Al retomar, restaura:
- Tareas pendientes
- Decisiones tomadas
- Archivos modificados
- Errores sin resolver

## Instalación — 2 comandos

```bash
# 1. Agregar marketplace
/plugin marketplace add mksglu/context-mode

# 2. Instalar plugin
/plugin install context-mode
```

Reiniciá Claude Code. El plugin configura MCP server + hooks automático.

## Verificar

```
> /plugin list
```

`context-mode` debe aparecer como Enabled.

Probá:

```
> Corré `ls -la` en el sandbox y mostrame solo los archivos
  modificados en últimas 24h.
```

Si responde con resultado **filtrado** (no todo el `ls`), está funcionando.

### 1. Debugging largo

Bug que requiere leer logs grandes + grep + investigación.

### 2. Refactor masivo

Tocar 30+ archivos.

Sin: cada archivo entero al contexto.

Screenshots y outputs pesados.

Necesitás entender estructura sin leer todo.

Querés que Claude retome donde dejaste ayer.

## Anti-patrones

### 1. Asumir que no perdés data

context-mode **filtra** lo que llega al contexto, pero la indexación es **buena, no perfecta**. Para data crítica completa, pedí Claude que **lea sin sandbox**.

### 2. Sandbox para todo

Tareas chicas que devuelven 200 bytes → sandbox agrega overhead. Útil para outputs **grandes**.

### 3. No revisar la memoria

context-mode guarda foto antes de compactar. **Revisá** qué guardó periódicamente. A veces guarda info que ya no aplica.

### 4. Confiar 100% en restore

Después de restore, **verificá** que Claude tiene contexto correcto. A veces falta una decisión clave.

### 5. Pelearse con sandbox cuando hace falta interactivo

Si necesitás **session interactiva** (REPL, debugger), sandbox no es la solución. Sandbox es para **outputs grandes**.
