## Qué es

Es **el** MCP de Playwright a usar por default. 4,419 installs lo hacen el más adoptado de toda la categoría `browser_automation`, por encima de [browser-use](/boveda/mcp-browser-use-mcp-server) (1,636), [Chrome DevTools](/boveda/mcp-chrome-devtools) (2,685), y otras variantes de Playwright.

Es la implementación de `executeautomation` de Playwright como MCP. Cubre todo lo que necesitás:
- Navegación
- Interacción con elementos (click, type, fill forms)
- Screenshots y PDFs
- Ejecución de JS arbitrario
- Inspección de DOM
- Manejo de cookies y sesiones

Si tu equipo arranca con browser automation desde Claude Code y no querés comparar variantes, instalá este.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp browser_automation/playwright-mcp-server
```

Para Claude:

```
Instalé el MCP browser_automation/playwright-mcp-server. Abrí Chromium

title, URL final y screenshot.
```

## Qué podés hacer con esto

### Tests E2E asistidos

"Andá a `localhost:3000/signup`, probá los 3 casos: email válido + password OK (debería pasar), email inválido (debería mostrar error), password corto (debería mostrar error). Validá los mensajes específicos y mostrame screenshots".

### Cross-browser sanity

"Andá a `mi-app.com` en Chromium, Firefox y WebKit. Tomame screenshot de la home en cada uno y compará si hay diferencias visibles".

### Scraping limpio (cuando no hay anti-bot serio)

"Andá a esta página, extraé los 10 productos con precio, rating, link y devolveme en JSON".

### Validación post-deploy

"Andá a producción y validá: home carga, link al checkout funciona, form de signup envía. Devolveme report de smoke test".

### Debugging asistido

"En localhost:3000, abrí DevTools console y traeme los errores que aparecen al hacer click en `Checkout`. Después navegá y describime cuál es el broken flow".

### Login flows

"Logueate con email X y password Y, navegá al dashboard y traeme las primeras 5 cards visibles". Mantiene sesión.

## Pre-requisitos

- **Playwright instala browsers** la primera vez automáticamente
- **Espacio en disco** para Chromium + Firefox + WebKit (varios GB)
- **Si tu sistema es Linux:** algunas deps de SO pueden ser necesarias

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sites con anti-bot agresivo:** usá [Browserbase](/boveda/mcp-mcp-server-browserbase) o [Bright Data](/boveda/mcp-brightdata).
- **Tests E2E mantenibles para CI:** mejor Playwright Test directamente, no via MCP.
- **Recursos:** Chrome consume RAM. Sesiones paralelas pueden saturar.
- **Sites con login propio que querés mantener:** [Browser Harness](/boveda/browser-harness) para tu sesión real.

</Callout>
