## What it is

A gimmicky themed statusline. It turns your debugging session into a circus performance: performers juggle bugs, and the audience cheers or reacts based on your progress.

It belongs to the "fun" statusline family in the catalog, alongside [Code Casino](/boveda/setting-code-casino-statusline), [Code Spaceship](/boveda/setting-code-spaceship-statusline), [Data Ocean](/boveda/setting-data-ocean-statusline).

176 installs: gimmicky but adopted by people who value the fun factor in long sessions.

## How to install it

```bash
npx claude-code-templates@latest --setting statusline/bug-circus-statusline
```

For Claude:

```
I installed the statusline/bug-circus-statusline setting. Show me live
how it looks while I run a couple of operations.
```

## What you can do with this

### Fun in long sessions

If you're going to be coding for 4 hours, a themed statusline can help the mood.

### For demos to non-technical people

Gimmicky statuslines are more visually engaging when showing Claude Code to people who don't know the editor.

### Customization

If you don't like Claude Code's boring default, gimmicky statuslines give it personality.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **For serious work that requires precise metrics:** [Context Monitor](/boveda/setting-context-monitor) is 100x more useful.
- **If it distracts you:** the constant animation can be annoying.
- **For small terminals:** the statusline takes up space.
- **For formal presentations:** unprofessional depending on the context.

</Callout>
