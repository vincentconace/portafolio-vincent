## What it is

Development-focused variant of the Neon statusline. It shows:
- **Connection status** (connected / disconnected)
- **Response time** (latency average)
- **Database activity** (queries in progress, etc.)

Unlike [Neon Context](/boveda/setting-neon-database-context) (static project info), this one is dynamic runtime info.

## What you can do with it

### Constant health check of Neon

Connection status tells you if the DB went down.

### Detecting latency issues

If response time goes up, you know something is wrong.

### For daily dev work

A stronger fit than [Neon Context](/boveda/setting-neon-database-context) when you run lots of dev queries.

### Combo with [Neon MCP](/boveda/mcp-neon)

Statusline for constant awareness. MCP for one-off ops (queries, branches).

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **No Neon:** doesn't apply.
- **If your app doesn't hit the DB often:** less useful than [Neon Resources](/boveda/setting-neon-database-resources) (which covers costs).
- **The statusline is a single slot:** pick between the 3 variants.

</Callout>
