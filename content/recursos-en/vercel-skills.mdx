## What skills.sh is

**skills.sh** is an **open directory** where you can search for skills available for Claude Code. It works like a skills store: you find what you need and install it with a single command.

URL: `skills.sh`

---

## The key skill — find-skills

**find-skills** is a special skill: it gives Claude Code the ability to **search for and install other skills for you**.

It's the **first thing you install**.

### Installation

```bash
npx skills add https://github.com/vercel-labs/skills --skill find-skills
```

### After that

You talk to Claude in plain English and it does the work.

---

## How to use it

### Search by topic

In Claude Code:

```
> Find frontend skills
```

```
> Find skills for testing
```

```
> What skills are there for databases
```

find-skills searches the directory and shows you options.

### Install what you find

```
> Install that skill
```

(referring to the one it showed you). It configures it automatically.

## Recommended workflow

### For Vercel ecosystem devs (Next.js, etc.)

```
1. Install find-skills
2. "Find skills for Next.js"
3. Install the relevant ones
4. Work as usual — Claude already has the skills
```

### To explore the ecosystem without overwhelming yourself

```
1. find-skills
2. "Show me the top 10 skills by category"
3. Install 2-3 based on immediate need
4. After 2 weeks, repeat — add whatever is justified
```

### For maintenance

Every 1-2 months:

```
> Audit my installed skills. Tell me which ones I haven't used
  in 30+ days. Suggest which to uninstall.
```

Apply the [global vs project skills](/boveda/skills-global-o-proyecto) rule.

## Anti-patterns

### 1. Skipping find-skills

If you're going to explore skills, **install find-skills first**. It saves you hours vs searching manually.

### 2. Installing everything you find

find-skills shows you options. **You choose** the relevant ones. Don't install 20 "just in case".

### 3. Not applying the global/project rule

When Claude offers to install a skill, **ask yourself**: do I use it in 3+ projects? If not, it goes to the project, not global.

### 4. Trusting the directory 100%

skills.sh has basic curation but no guarantee. **Read the README** before installing.

### 5. Forgetting that find-skills is discovery only

find-skills helps you **find** skills. **It doesn't use them for you**. After installing, you talk to the skills as usual.
