## Qué es

Statusline informativo no-gimmicky. Muestra:
- **Model** name
- **Directory** del proyecto
- **Node.js version** activa
- **Claude Code version** instalada

Perfect para **multi-project environments** donde rotás entre proyectos con stacks distintos (Node versions diferentes, etc.).

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/project-info-statusline
```

Para Claude:

```
Instalé el setting statusline/project-info-statusline. Mostrame
las 4 piezas de info actuales.
```

## Qué podés hacer con esto

### Multi-project con Node versions distintas

Si tu proyecto A usa Node 18 y B usa Node 20 (con `.nvmrc`), el statusline te confirma cuál tenés activa.

### Detection de Claude Code version

Para troubleshooting: la version te da context para buscar issues en docs.

### Más info que [Minimal Statusline](/boveda/setting-minimal-statusline), menos que [Context Monitor](/boveda/setting-context-monitor)

Punto medio. Info útil sin bars ni cost tracking.

### Combo con [Git Branch Statusline](/boveda/setting-git-branch-statusline)

Solo podés tener UN statusline. Compará: este (Node + Claude versions) vs Git Branch (branch + uncommitted). Elegí según prioridad.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si trabajás solo un proyecto siempre:** Node version es estática, no aporta.
- **Si querés awareness de context %:** [Context Monitor](/boveda/setting-context-monitor) es la opción.
- **Si tu terminal es muy estrecho:** 4 piezas pueden saturar.

</Callout>
