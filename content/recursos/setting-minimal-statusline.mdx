## Qué es

La opción más simple de statusline: solo muestra el model name y el current directory. Sin git info, sin context bars, sin gimmicks.

Para devs que querés focus máximo en el contenido sin distracciones.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `statusline` |
| Installs | 72 |
| Muestra | Solo model + directory |
| Foco | Distraction-free |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/minimal-statusline
```

Para Claude:

```
Instalé el setting statusline/minimal-statusline. Mostrame cómo se ve.
```

## Qué podés hacer con esto

### Focus máximo

Sin info extra que te distraiga del contenido.

### Para users que ya saben context

Si llevás trackeo mental de en qué branch estás y context %, no necesitás verlo todo el tiempo.

### Statusline minimalista intencional

Para devs que prefieren UI clean — ortogonal a los gimmicky o los info-rich.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si querés awareness de context %:** usá [Context Monitor](/boveda/setting-context-monitor).
- **Si querés awareness de git:** [Git Branch Statusline](/boveda/setting-git-branch-statusline).
- **Si te molesta NO tener info:** los demás statuslines pueden ser mejores.

</Callout>
