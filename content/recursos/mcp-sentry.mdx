## Qué es

Sentry es la plataforma de error tracking y APM más adoptada. Issues con stack traces, performance metrics, release tracking, alerts. Este MCP conecta tu cuenta a Claude Code con foco específico: **human-in-the-loop coding agents y dev workflows**.

Significado práctico: el MCP está pensado para que Claude (con vos supervisando) use Sentry como context para debugging asistido — leer issues, traces, contexto del error, y proponer fixes.

## Cómo se instala

```bash
npx claude-code-templates@latest --mcp devtools/sentry
```

Pre-requisito: cuenta Sentry + API token con scopes apropiados.

Para Claude:

```
Instalé el MCP devtools/sentry. Configurá las credenciales y listame
los issues abiertos del proyecto X con > 10 occurrences en las últimas
24h. Para los top 3, leé stack trace y dame root cause + suggested fix.
```

## Qué podés hacer con esto

### Triage de issues

"Listame issues abiertos del proyecto en producción agrupados por: critical / unresolved / regression. Para los critical, dame contexto y prioridad sugerida".

### Debugging asistido

"Issue X tiene 500 occurrences en las últimas 2 horas. Leé el stack trace, contexto del request, breadcrumbs y proponé qué cambió y dónde fixearlo. Si necesitás ver el código de la función X, pegalo".

### Análisis de release

"Después de release v2.1.0, ¿qué errores nuevos aparecieron que no existían en v2.0? Comparame issue rates pre/post-deploy".

### Correlación con deploys

"Cuando se desplegó X commit, ¿qué errores nuevos se introdujeron? Mostrame los issues que aparecieron en esa ventana".

### Performance issues

"Listame transactions con p95 > 2s en el último día. Para las top 3, dame el span dominante y sugerí optimization".

### Resolver issues por bloque

"Para todos los issues con label `legacy` que no se reprodujeron en los últimos 30 días, agrupalos y proponé marcarlos como resolved".

## Pre-requisitos

- **Cuenta Sentry** activa
- **API token** con scopes correctos
- **Acceso a los projects** que vas a consultar

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Operaciones destructivas:** delete issues / resolve sin context. Hooks de aprobación.
- **Token amplio:** un token con org admin deja a Claude ver todo. Restringí scope.
- **Solapamiento con alternativas:** [Logfire](/boveda/mcp-logfire), [Dynatrace](/boveda/mcp-dynatrace), [Grafana](/boveda/mcp-grafana). Mirá si tu observability ya está cubierta antes de sumar.
- **Si usás Sentry Self-Hosted on-prem:** verificá si este MCP lo soporta (algunos asumen Sentry.io).
- **Privacy de stack traces:** algunos stack traces tienen PII en variables. Tené cuidado al pegarlos a Claude.

</Callout>
