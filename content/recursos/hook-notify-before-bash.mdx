## Qué es

Antes de ejecutar cualquier Bash command, el hook muestra una notification simple (echo del comando que va a correr). No bloquea, solo informa.

Útil cuando querés tener "always-on" awareness de qué comandos corre Claude, sin la fricción de bloquear cada vez.

Hook event: `PreToolUse` (Bash).

## Cómo se instala

```bash
npx claude-code-templates@latest --hook pre-tool/notify-before-bash
```

Para Claude:

```
Instalé el hook pre-tool/notify-before-bash. Pedile a Claude que
ejecute un comando cualquiera — debería echoear el command antes
de correrlo.
```

## Qué podés hacer con esto

### Awareness pasivo

Para sesiones donde no querés bloqueo activo (que [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker) o [AI Bash Guard](/boveda/hook-ai-bash-guard) hacen) pero querés ver cada bash command.

### Para debugging

¿Por qué Claude está tardando? Ves los commands en vivo y entendés qué está haciendo.

### Para teaching / demos

Si mostrás Claude Code a alguien, este hook hace explícito qué tools usa.

### Combo con monitoring

Suma sobre [Command Logger](/boveda/hook-command-logger) (que persiste a archivo) la información en vivo.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si querés seguridad de verdad:** este hook informa, no bloquea. Usá [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker) o [AI Bash Guard](/boveda/hook-ai-bash-guard) si querés bloqueo.
- **Para uso productivo intenso:** muchos commands = muchas notifications = ruido.
- **Si ya logueás con [Command Logger](/boveda/hook-command-logger):** overlap.

</Callout>
