## What it is

Web Fetch is the base MCP of the `web` category: **3,346 installs**, the most adopted. Its scope is straightforward: web content fetching, basic extraction, access to external APIs.

Compared to alternatives:
- [Firecrawl](/boveda/mcp-firecrawl) — LLM-ready markdown, paid
- [Jina AI](/boveda/mcp-jina-ai) — grounding + fact-checking
- **Web Fetch (this one)** — simple, free, no fancy features

It's the "basic fetch that works". For many cases, that's enough.

## How to install it

```bash
npx claude-code-templates@latest --mcp web/web-fetch
```

For Claude:

```
I installed the web/web-fetch MCP. Fetch this URL [URL] and give me back the
content. If it's HTML, extract the main text.
```

## What you can do with it

### Fetching URLs

"Get me the content of `<URL>` and show me the body in plain text". The most basic operation.

### Consuming external APIs

"Do a GET to `https://api.github.com/repos/owner/repo` and give me name, stars, last update". Simple JSON parsing.

### One-off doc reading

"Go to this Next.js docs page and answer me: what is Server Actions?". Homemade anti-hallucination.

### Casual scraping

For pages without anti-bot, it's enough. For serious anti-bot, better to use [Bright Data](/boveda/mcp-brightdata) or [Browserbase](/boveda/mcp-mcp-server-browserbase).

### Combo with any flow

Because it's so basic, it fits into many workflows as an initial connector.

## Prerequisites

- **Node.js + Claude Code**
- **Internet connection**

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **For sites with heavy JS (SPA):** Web Fetch brings back static HTML. For SPAs you need [Playwright](/boveda/mcp-playwright-mcp-server) or [browser-use](/boveda/mcp-browser-use-mcp-server).
- **For sites with anti-bot:** better to use specialized tools.
- **For clean markdown output:** [Firecrawl](/boveda/mcp-firecrawl) is tidier.
- **For fact-checking / grounding:** [Jina AI](/boveda/mcp-jina-ai).

</Callout>
