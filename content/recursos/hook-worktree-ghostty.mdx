## Qué es

Para devs que usan [Ghostty](https://mitchellh.com/ghostty) (terminal emulator moderno de Mitchell Hashimoto) y trabajan con git worktrees, este hook automatiza el setup visual.

Al crear un worktree, abre layout 3-panel:
- **Izquierda:** Claude Code
- **Top-derecha:** lazygit
- **Bottom-derecha:** yazi (file manager TUI)

Al eliminar el worktree, cierra el layout.

Hook events: `WorktreeCreate` + `WorktreeRemove`.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook development-tools/worktree-ghostty
```

Pre-requisitos:
- Ghostty terminal
- lazygit instalado (`brew install lazygit`)
- yazi instalado (`brew install yazi`)

Para Claude:

```
Instalé el hook development-tools/worktree-ghostty. Verificá que tengo
todos los CLI tools. Creá un worktree de prueba y mostrame el layout
3-panel.
```

## Qué podés hacer con esto

### Setup productivity terminal-first

Para devs que viven en terminal, el 3-panel optimiza workflow: code (Claude) + git (lazygit) + file nav (yazi).

### Worktrees paralelos visuales

Si trabajás 3 features en paralelo en 3 worktrees, cada uno tiene su layout consistente.

### Para usuarios de [Agent View](/boveda/agent-view)

Combina bien con multi-session workflows donde cada worktree es una sesión Claude.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Ghostty:** este hook es específico de ese terminal. iTerm, Warp, otros — no aplica.
- **Si no usás worktrees:** el hook no se dispara nunca.
- **Sin lazygit / yazi:** dependencias faltantes = layout incompleto.
- **Solo 139 installs:** nicho, validá en sandbox antes de adoptarlo seriamente.
- **Para usuarios de IDE GUI:** Ghostty + TUI workflow no encaja con VSCode/JetBrains.

</Callout>
