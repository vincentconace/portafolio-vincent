## What it is

**#2 overall in the catalog (2,577 installs).** Only behind [Simple Notifications](/boveda/hook-simple-notifications) (6,810).

The hook intercepts `git commit` and generates the message automatically based on the changed files. Before committing, it validates the message.

Hook event: `PostToolUse`.

It's the #1 use case you want when you code with AI: having the commit message reflect the changes without you having to write it.

```

```

## What you can do with it

### Automatic descriptive commits

Instead of a generic "fix" or "update", the hook generates "fix: handle null email in registration form" based on the actual change.

### Combo with [Conventional Commits](/boveda/hook-conventional-commits)

Smart Commit generates. Conventional Commits validates the format. Double safety net.

### Combo with [Auto Git Add](/boveda/hook-auto-git-add)

Auto Git Add stages. Smart Commit commits with a descriptive message. Semi-automatic git workflow.

### For autonomous sessions

Long `/goal` runs: every auto commit gets a useful message. No thousands of "wip".

### Catching weak messages

If you write "fix", the hook can suggest something more descriptive based on the diff.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your team has a strict message convention:** auto-generate may not follow it. Combine it with [Conventional Commits](/boveda/hook-conventional-commits) if you need a specific format.
- **For commits that need a human reason:** auto-generate doesn't know the "why", only the "what".
- **If your workflow is committing very often:** the overhead of generating can be noticeable.

</Callout>
