## What it is

A setting that points Claude Code to use **Amazon Bedrock** as the model provider instead of Anthropic's direct API. Useful when:

- Your organization already has centralized AWS billing
- You need AWS-specific compliance (HIPAA BAA, FedRAMP, etc.)
- You need data residency in specific AWS regions
- You want unified auditing with the rest of your AWS services

It sets `CLAUDE_CODE_USE_BEDROCK=1` and `AWS_BEARER_TOKEN_BEDROCK` with your Bedrock API key.

## How to install it

```bash
npx claude-code-templates@latest --setting api/bedrock-configuration
```

Then you set the real token in your env.

For Claude:

```
I installed the api/bedrock-configuration setting. Show me which env vars
I need to set and the Bedrock endpoint URL for my region.
```

## What you can do with this

### Centralize billing in AWS

Your org already has an AWS contract. Bedrock bills there, not separately to Anthropic.

### Enterprise compliance

If you need a HIPAA BAA with AWS, Bedrock allows it. Anthropic's direct API may not apply.

### Specific data residency

Bedrock runs in specific AWS regions. If you need us-east-1 for compliance, you have control.

### Combo with [AWS Credentials Setting](/boveda/setting-aws-credentials)

AWS Credentials for SSO/credential refresh. Bedrock Configuration to point at the right provider. A natural enterprise combo.

## When it's NOT a good fit

<Callout type="warning" title="Before enabling">

- **If you don't have an AWS account or Bedrock enabled:** these are prerequisite steps.
- **For individual use:** Bedrock can be more expensive than Anthropic's direct API.
- **If your AWS region doesn't have Bedrock:** check availability.
- **Models may differ:** Bedrock doesn't always have the latest version of Claude. Validate what you need.

</Callout>
