## Qué es

`cleanupPeriodDays: 90` — Claude Code mantiene chat transcripts 90 días.

Para users que valoran histórico extendido. Útil para reference, auditoría leve o memoria entre sesiones largas.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `cleanup` |
| Installs | 346 |
| Setting | `cleanupPeriodDays: 90` |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting cleanup/retention-90-days
```

Para Claude:

```
Instalé el setting cleanup/retention-90-days. Confirmá retention y
mostrame cómo buscar transcripts viejos.
```

## Qué podés hacer con esto

### Histórico para reference

Volver a sesión de hace 60 días: tenés el transcript disponible.

### Para projects largos

Sprints de 3 meses: el transcript completo del sprint queda disponible.

### Audit lite

Para compliance suave, 90 días es razonable trail.

### Combo con [Memory Integration MCP](/boveda/mcp-memory-integration)

Retention guarda transcripts brutos. Memory MCP indexa y recupera relevante. Combo natural.

### Combo con [Claude Mem Plugin](/boveda/plugin-claude-mem)

Retention para data persistida. Claude Mem para context inteligente.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para data altamente sensible:** 90 días es mucho. [Retention 7 Days](/boveda/setting-retention-7-days).
- **Disk space:** 90 días de transcripts ocupan storage. Verificá tu disk.
- **GDPR strict:** 90 días puede no cumplir con data minimization principles.

</Callout>
