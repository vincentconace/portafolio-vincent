## Qué es

Override del modelo a `claude-3-5-haiku-20241022`. Más rápido y barato que Sonnet, menos capaz para tareas complejas.

Para tasks donde priority es speed o cost — ej. queries simples, lookups, edits chicos.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting model/use-haiku
```

Para Claude:

```
Instalé el setting model/use-haiku. Confirmá que estoy usando Haiku
ahora.
```

## Qué podés hacer con esto

### Tasks chicos / cost-conscious

Para queries que no necesitan capability completa de Sonnet.

### Faster iteration

Haiku responde más rápido. Para exploración rápida ayuda.

### Cost reduction significativo

Haiku es bastante más barato que Sonnet.

### Combo con [Performance Optimization](/boveda/setting-performance-optimization)

Performance Opt limita output. Use Haiku cambia model. Ambos reducen cost.

### Anti vs [Use Sonnet](/boveda/setting-use-sonnet)

Estos dos son opposites. Para complex code work, Sonnet.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para tasks complejos:** Haiku puede dar respuestas pobres. Sonnet/Opus mejor.
- **Para code generation serio:** Haiku tiene menor calidad de output.
- **Si tu workflow ya es lento por otros motivos:** speed gain puede no compensar quality loss.

</Callout>
