## Qué es

Browser MCP es uno de los MCPs base del catálogo browser_automation. Más simple que [browser-use](/boveda/mcp-browser-use-mcp-server) o [Chrome DevTools](/boveda/mcp-chrome-devtools), su rol es darle a aplicaciones AI capacidades web básicas: navegar URLs, llenar forms, interactuar con elementos.

Útil cuando necesitás algo más liviano que browser-use pero más amigable que Chrome DevTools Protocol directo. 582 installs lo posicionan como una opción reconocida.

## Qué podés hacer con esto

### Navegación básica

Abrir una URL, esperar a que cargue, leer el contenido. Tareas tipo "andá a `mi-app.com/dashboard` y traeme qué cards aparecen".

### Llenar forms simples

"Andá al form de contacto y enviá un mensaje con datos de prueba". Útil para smoke tests rápidos.

### Scraping ligero

Para páginas sin anti-bot serio, alcanza. Si la página tiene Cloudflare o captchas, mejor [Bright Data](/boveda/mcp-brightdata).

### Validación post-deploy

"Andá a la URL de prod y verificá que la home carga sin errores". Validación liviana de uptime + sanity check.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para automatizaciones complejas:** [browser-use](/boveda/mcp-browser-use-mcp-server) (1,636 installs) tiene más madurez. [Chrome DevTools](/boveda/mcp-chrome-devtools) (2,685 installs) da más control.
- **Para scraping serio:** [Apify](/boveda/mcp-apify) o [Bright Data](/boveda/mcp-brightdata).
- **Para tests E2E reproducibles:** [Playwright Skill](/boveda/plugin-playwright-skill) es mejor.
- **Si solo necesitás un fetch HTTP:** `curl` o `fetch` directo es más simple. El MCP es para cuando necesitás interacción real con la página.

</Callout>
