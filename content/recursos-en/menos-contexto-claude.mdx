## The problem

Picture your desk as having limited space. Every time Claude Code uses a tool (reading a file, a Playwright screenshot, checking GitHub issues), the result lands in the conversation and takes up space on that desk. That space = **context**.

The results are **huge**:
- A Playwright screenshot: **56 KB**
- A GitHub issues list: **59 KB**

With just a few calls, your desk fills up and:
- Claude **loses the thread**
- **Forgets decisions**
- The session resets → you start over

---

## The solution — context-mode

An open source MCP server that installs as a plugin. **3 pillars**:

### Pillar 1 — Sandbox (isolated execution)

Your assistant reads a 100-page report and hands you a 1-page summary. That's what the sandbox does: **commands run in an isolated environment** and only the processed result enters your conversation. **Raw data never touches your context**.

Supports 11 languages: JavaScript, Python, Go, Rust, Ruby, PHP, Java, C#, C++, Haskell, Julia.

### Pillar 2 — Smart indexing

When a result weighs **more than 5 KB**, it indexes it with **BM25 ranking** (the tech behind search engines) and returns only the relevant parts.

| Without | With |
|-----|-----|
| 315 KB of raw data | **5.4 KB** indexed |

**98% less context**.

### Pillar 3 — Session memory

Every edited file, git operation, task, and decision is saved to SQLite. When the context compacts, **it saves a snapshot of the state** (max 2 KB) with everything critical.

When you pick back up, it restores:
- Pending tasks
- Decisions made
- Modified files
- Unresolved errors

## Installation — 2 commands

```bash
# 1. Add marketplace
/plugin marketplace add mksglu/context-mode

# 2. Install plugin
/plugin install context-mode
```

Restart Claude Code. The plugin sets up the MCP server + hooks automatically.

## Verify

```
> /plugin list
```

`context-mode` should show up as Enabled.

Try it:

```
> Run `ls -la` in the sandbox and show me only the files
  modified in the last 24h.
```

If it responds with a **filtered** result (not the whole `ls`), it's working.

## Anti-patterns

### 1. Assuming you lose no data

context-mode **filters** what reaches the context, but the indexing is **good, not perfect**. For complete critical data, ask Claude to **read without the sandbox**.

### 2. Sandbox for everything

Small tasks that return 200 bytes → the sandbox adds overhead. Useful for **large** outputs.

### 3. Not reviewing the memory

context-mode saves a snapshot before compacting. **Review** what it saved periodically. Sometimes it saves info that no longer applies.

### 4. Trusting restore 100%

After a restore, **verify** that Claude has the right context. Sometimes a key decision is missing.

### 5. Fighting the sandbox when you need interactive

If you need an **interactive session** (REPL, debugger), the sandbox isn't the solution. The sandbox is for **large outputs**.
