## What it is

After every code change, the hook detects the project's test runner (Jest, Vitest, pytest, etc.) and runs quick tests. If something breaks, it tells you immediately.

Hook event: `PostToolUse`. Focus: "quick tests" — unit tests, not heavy E2E.

1,017 installs is high for a testing hook — it reflects how important it is to catch regressions on the fly.

## How to install it

```bash
npx claude-code-templates@latest --hook post-tool/run-tests-after-changes
```

For Claude:

```
I installed the post-tool/run-tests-after-changes hook. Verify that my
test runner (Vitest) is configured. Try editing a business logic
file — it should run the associated tests.
```

## What you can do with it

### Instant regression catch

If Claude breaks a test while editing, you see it immediately. Not after 30 edits.

### Test-first workflow

If you follow TDD, this hook validates that your green stays green during refactors.

### Combo with [TDD Gate](/boveda/hook-tdd-gate)

TDD Gate enforces that tests exist first. Run Tests After validates that they stay green.

### Combo with [Lint On Save](/boveda/hook-lint-on-save)

Lint + tests = double check on quality. No extra scaffolding.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **Slow tests:** if your suite takes 30+ seconds, the friction is noticeable.
- **No existing tests:** the hook doesn't invent tests.
- **For experimental edits:** green tests are useful for release, not for WIP.
- **If your CI already runs tests:** running locally too can be overhead.

</Callout>
