## Qué es

`AGENTS.md` es una convención emergente para definir comportamiento agent-specific de un proyecto — similar al rol que `CLAUDE.md` tiene pero más enfocado a setups multi-agent o agent SDK. Si tu proyecto tiene `AGENTS.md`, este hook lo carga automáticamente al inicio de cada sesión Claude Code.

Hook event: `SessionStart`. Solo se ejecuta si `AGENTS.md` existe en el proyecto. Si no, no hace nada (no rompe).

Es una de las automatizaciones más útiles para mantener consistency cross-sesión. 937 installs lo posicionan como uno de los hooks más adoptados del catálogo.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook automation/agents-md-loader
```

Para Claude:

```
Instalé el hook automation/agents-md-loader. Verificá que esté activo
en mis hooks de SessionStart. Para probarlo, creame un AGENTS.md de
ejemplo con instrucciones específicas y reiniciá una sesión.
```

## Qué podés hacer con esto

### Consistencia cross-sesión automática

Cada vez que abrís una sesión nueva en el proyecto, Claude lee AGENTS.md sin que vos lo pidas. Las preferencias y comportamientos quedan aplicados.

### Multi-agent setups

Si tu proyecto usa múltiples agents (orchestrator + workers), AGENTS.md puede definir el comportamiento esperado de cada uno. Este hook asegura que se aplique.

### Onboarding instantáneo

Un nuevo dev clona el repo, abre Claude Code → AGENTS.md se carga y Claude opera según las reglas del equipo desde la primera interacción.

### Combo con CLAUDE.md

CLAUDE.md es el general; AGENTS.md puede ser más específico para flows de agent SDK. Tener ambos cubre los dos niveles.

## Pre-requisitos

- **Proyecto con `AGENTS.md`** (creá uno si no tenés)
- **Claude Code** con hooks habilitados

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás AGENTS.md ni planeás:** el hook no hace nada útil. No lo instales por las dudas.
- **Si tu CLAUDE.md cubre todo:** no necesitás duplicar.
- **AGENTS.md muy largo:** carga tokens en cada sesión. Mantenelo conciso.
- **Si tu workflow es 100% one-shot:** session start hooks aportan menos.

</Callout>
