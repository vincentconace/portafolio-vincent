## Qué es

Setting que configura las env vars para que Claude Code mande traces a LangSmith:

- `TRACE_TO_LANGSMITH: true`
- `CC_LANGSMITH_API_KEY: <tu key>`
- `CC_LANGSMITH_PROJECT: claude-code-project`
- `CC_LANGSMITH_DEBUG: false`

Es la versión setting (más simple, solo env vars) del [LangSmith Tracing Hook](/boveda/hook-langsmith-tracing) (que requiere jq + script de stop).

## Cómo se instala

```bash
npx claude-code-templates@latest --setting telemetry/langsmith-tracing
```

Reemplazá `YOUR_API_KEY_HERE` con tu key real.

Para Claude:

```
Instalé el setting telemetry/langsmith-tracing. Configurá la API key.
Hacé una sesión corta y verificá que el trace llegó a mi LangSmith
project.
```

## Qué podés hacer con esto

### Observability sin scripting

A diferencia del hook (que requiere bash + jq), este setting solo activa env vars. Más simple.

### Para teams ya en LangSmith

Si tu equipo usa LangSmith para otras LLM apps, agregar Claude Code centraliza observability.

### Combo con [Custom Telemetry](/boveda/setting-custom-telemetry)

Custom Telemetry para OTel generic. LangSmith Tracing para LangSmith específicamente.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin cuenta LangSmith:** no funciona.
- **Para uso casual:** observability serio es overkill.
- **Privacy:** los traces incluyen conversaciones. Para data sensible, validá.
- **Costos LangSmith:** plan free tiene cap.

</Callout>
