## What it is

Before running any Bash command, the hook shows a simple notification (an echo of the command that's about to run). It doesn't block, it just informs.

Useful when you want "always-on" awareness of which commands Claude runs, without the friction of blocking every time.

Hook event: `PreToolUse` (Bash).

## How to install it

```bash
npx claude-code-templates@latest --hook pre-tool/notify-before-bash
```

For Claude:

```
I installed the pre-tool/notify-before-bash hook. Ask Claude to
run any command — it should echo the command before
running it.
```

## What you can do with this

### Passive awareness

For sessions where you don't want active blocking (which [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker) or [AI Bash Guard](/boveda/hook-ai-bash-guard) provide) but you do want to see every bash command.

### For debugging

Why is Claude taking so long? You see the commands live and understand what it's doing.

### For teaching / demos

If you're showing Claude Code to someone, this hook makes it explicit which tools it uses.

### Combo with monitoring

On top of [Command Logger](/boveda/hook-command-logger) (which persists to a file), it adds the live information.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you want real security:** this hook informs, it doesn't block. Use [Dangerous Command Blocker](/boveda/hook-dangerous-command-blocker) or [AI Bash Guard](/boveda/hook-ai-bash-guard) if you want blocking.
- **For heavy production use:** lots of commands = lots of notifications = noise.
- **If you already log with [Command Logger](/boveda/hook-command-logger):** overlap.

</Callout>
