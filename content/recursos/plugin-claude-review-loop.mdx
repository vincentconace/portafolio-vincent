## Qué es

Hamel Husain armó algo simple y muy útil: cuando Claude termina de implementar algo, en vez de declarar "listo", arma un loop donde Codex le hace code review. Si Codex encuentra issues, Claude itera. Si Codex aprueba, Claude declara done.

El patrón se llama "review loop". La diferencia con un code review manual es que vos no estás ahí — el loop corre solo hasta convergencia. La diferencia con que Claude se autorevise es que tener un modelo externo (Codex de OpenAI) baja el bias y cubre puntos ciegos.

Es plugin único, con `commands` y `hooks`. La instalación es directa y el efecto es palpable: bajan los "se ve que ya está pero falta algo" típicos de generación autónoma.

## Qué podés hacer con esto

### Code review automático antes de cerrar tarea

Implementás un feature, lanzás el review loop. Codex critica:
- Bugs visibles
- Edge cases no cubiertos
- Inconsistencias con el resto del código
- Cuestiones de claridad

Claude itera el código hasta que Codex aprueba (o hasta un cap razonable de iteraciones).

### Validación cruzada para PRs

Antes de pushear un PR, el loop te da una pasada extra. No reemplaza al reviewer humano, pero filtra los errores triviales que humanos pierden tiempo señalando.

### Catch de "Claude declaró done pero no estaba"

El típico problema de generación autónoma: Claude dice "listo y funcionando" cuando todavía falta. El loop con Codex como external validator detecta esto.

### Trabajo nocturno más confiable

Si dejás Claude corriendo de noche en una tarea larga (con [/goal](/boveda/goal-claude-code) o un orquestador), agregar review loop hace que el output de la mañana sea más confiable.

## Beneficios reales

- **Catch de bias propio** — Claude tiende a aprobar su propio código. Codex no.
- **Convergencia automática** — el loop corre hasta el done real, no el done declarado.
- **Cross-tool** — explota que tengas Codex pago. Es el patrón base de [Claude + Codex en equipo](/boveda/claude-codex-equipo).
- **Liviano** — un solo plugin, pocos hooks.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no tenés Codex CLI:** sin Codex el plugin no sirve. Hay que pagar el sub o usar la versión free limitada de OpenAI.
- **Si tu cuota de tokens es ajustada:** el loop puede iterar 3-5 veces. Multiplica el consumo de ambos modelos.
- **Para tareas triviales:** review loop sobre un fix de typo es ridículo. Usá criterio.
- **Si querés solo una opinión externa, sin iterar:** mirá [Adversarial Spec](/boveda/plugin-adversarial-spec) o el patrón `consult` de Agentsys — más one-shot.
- **Codex puede ser sobre-pedante:** algunas iteraciones agregan ruido más que valor. Tené un cap de loops max.

</Callout>
