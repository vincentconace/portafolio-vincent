## Qué es

Equivalente a Discord/Slack Error pero para Telegram. Avisa cuando:
- Long-running operations
- Tools toman tiempo significativo
- Notifications de Claude Code internas

Hook events: `PreToolUse` + `PostToolUse` + `Notification`.

Pre-requisito: bot Telegram + chat ID.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook automation/telegram-error-notifications
```

Pre-requisitos: bot via @BotFather + chat ID.

Para Claude:

```
Instalé el hook automation/telegram-error-notifications. Configurá
TELEGRAM_BOT_TOKEN y TELEGRAM_CHAT_ID. Mostrame thresholds.
```

## Qué podés hacer con esto

### Stuck session detection

Cuando Claude se cuelga 5+ min en una op, te avisa al celular.

### Errors al mobile

Si Claude dispara error notification interna, llega a Telegram. Catch desde cualquier lado.

### Para sesiones autónomas

`/goal` largos overnight: si algo se rompe, te llega.

### Combo con [Telegram Detailed](/boveda/hook-telegram-detailed-notifications)

Detailed para success + Error para problems = canal completo de tracking.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Sin Telegram:** usá Slack/Discord variants.
- **Para tareas que SIEMPRE son lentas:** false positives constantes.
- **Sin bot setup:** no funciona.
- **Si querés también success:** este es errors-only.

</Callout>
