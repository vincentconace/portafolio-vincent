## What it is

This is **the** Postgres MCP to use by default. 3,214 installs make it the outright #1 in the `database` category — well ahead of MySQL (1,081), MongoDB (319), Neon (208).

Capabilities:
- **Connection** to any Postgres (local, cloud, Atlas)
- **Advanced queries** with full SQL support
- **Schema management** (DDL ops)
- **Enterprise ops** (replication info, slow queries, perf)

For shops running serious Postgres, this is the foundation. Combine it with [pg-aiguide](/boveda/plugin-pg-aiguide) (which adds skills + docs) to get the whole stack.

## How to install it

```bash
npx claude-code-templates@latest --mcp database/postgresql-integration
```

Prerequisite: connection string + correct credentials.

For Claude:

```

```

## What you can do with it

### Deep schema exploration

"Map out all the tables, their columns with exact types, FKs, indexes and constraints. Flag tables with more than 50 columns (likely candidates for normalization)."

### Natural-language queries

"Top 10 customers by revenue in Q1, with order count and average ticket. Sort by revenue desc."

### Performance debugging

"Show me the slowest queries using `pg_stat_statements`. For the top 5, give me the execution plan and suggest indexes that would help." Serious performance analysis.

### Index usage analysis

"Identify indexes that weren't used in the last 30 days according to `pg_stat_user_indexes`. Candidates for removal."

### Migration assistant

"Generate the migration script to add `tenant_id uuid` to 12 tables. Include a backfill from the `org_slug` column with a lookup." With a preview of the SQL.

### Data audit

"Check the unique constraint on `users.email`. If there are duplicates (e.g., case-insensitive), show me."

### Bulk operations

"In `products`, update the prices by adding 15% for category X. Show me the COUNT before applying and ask for confirmation."

## Prerequisites

- **Valid connection string** (`postgresql://user:pass@host:port/db`)
- **Credentials** with appropriate permissions
- **Network access** to your Postgres

## When it's NOT a good fit

<Callout type="danger" title="Before connecting Claude to production Postgres">

- **Destructive operations:** `DROP TABLE`, `DELETE` without WHERE, `TRUNCATE`. ALWAYS use approval hooks in prod.
- **Broad permissions:** a user with superuser inherits everything. Restrict to SELECT-only for initial exploration.
- **Huge queries:** without a LIMIT, you can dump millions of rows into Claude. Always limit.
- **Schema changes (DDL) in prod:** `ALTER TABLE` on large tables can lock. Validate the strategy first.
- **Connection strings with secrets:** treat them as sensitive credentials, don't hard-code them.

</Callout>
