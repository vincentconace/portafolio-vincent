## What it is

It's one of the **three** Playwright variants in the browser_automation catalog:
- [playwright-mcp-server](/boveda/mcp-playwright-mcp-server) (4,419 installs — top) — `executeautomation`
- **playwright-mcp** (this one, 1,393 installs) — alternative variant
- [mcp-server-playwright](/boveda/mcp-mcp-server-playwright) (675 installs) — `automatalabs`

Three distinct implementations of the same concept (wrapping Playwright as an MCP). Compare the JSON to see which one suits you; for most cases, the most widely adopted (`playwright-mcp-server`) is the default option.

## What you can do with it

The same browser automation operations with Playwright as the other variants:

- Navigate URLs
- Click, type, scroll
- Take screenshots
- Run JS in the browser
- Inspect the DOM
- Ad-hoc E2E testing

For concrete use cases, check out [Playwright MCP Server](/boveda/mcp-playwright-mcp-server) (the most widely adopted variant) — most of them transfer over.

## When to choose which

| Case | Recommendation |
| --- | --- |
| You have no preference | [playwright-mcp-server](/boveda/mcp-playwright-mcp-server) (most adopted) |
| You want to try several and compare | Test one in a sandbox before committing |
| Critical enterprise stack | Look at the JSONs and the upstream implementation |

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you already have another Playwright MCP:** don't install them together (tool collision).
- **Smaller community vs the top one:** `playwright-mcp-server` has 3x more adoption. Higher chance that known bugs are already fixed.
- **For sites with anti-bot:** better go with [Browserbase](/boveda/mcp-mcp-server-browserbase) or [Bright Data](/boveda/mcp-brightdata).

</Callout>
