## Qué es

Si dejás a Claude Code corriendo una tarea larga (un refactor, un loop autónomo, un build) y querés saber cuándo termina sin tener que volver cada 5 minutos al editor, este plugin resuelve eso. Manda notificaciones a tu sistema operativo (nativas de Linux, macOS y Windows) o a un webhook configurable (ntfy, Slack, Telegram, lo que tenga URL).

6 tipos de notificación, click-to-focus (te lleva directo a Claude Code cuando hacés click), zero dependencies, instalación de 1 línea. El plugin analiza contexto para decidir cuándo es un buen momento para avisarte — no te spammea con cada cambio menor.

Si trabajás con [/goal](/boveda/goal-claude-code) o sesiones largas autónomas, es básicamente obligatorio.

### Los 6 tipos de notificación

(Detalles exactos en el repo; el patrón típico es algo como)

1. **Task completed** — la tarea terminó
2. **Task failed** — falló con error
3. **Needs input** — Claude necesita una decisión humana
4. **Long-running started** — empezó algo que va a durar
5. **Resource warning** — context cerca del límite, costo alto, etc.
6. **Custom** — lo que vos definas vía hook

(El plugin es zero-dependencies y multi-platform; los detalles concretos los ves en el README del repo.)

## Cómo se instala

```bash
/plugin marketplace add 777genius/claude-notifications-go
/plugin install claude-notifications-go@777genius-claude-notifications-go
```

Después configurás con:

```
/notifications-init
/notifications-settings
```

Para Claude:

```
Instalá claude-notifications-go del marketplace 777genius. Configurame
notificaciones nativas de macOS para "task completed" y "needs input",
y un webhook a mi Telegram para "task failed".
```

## Qué podés hacer con esto

### Loops autónomos largos

Combinás con [/goal](/boveda/goal-claude-code), `next-task` o [Claude Code Loop](/boveda/claude-code-loop). Lanzás la tarea, te vas a hacer otra cosa, y volvés cuando suena el ping.

### Saber cuándo Claude necesita una decisión

A veces Claude espera input ("¿continuo o cambio de enfoque?") y vos no estás mirando. La notificación "needs input" te avisa.

### Alertas a un canal de Slack para equipos

Si varios devs en tu equipo usan Claude Code para tareas largas, podés mandar notificaciones a un canal `#claude-runs` con quién está corriendo qué.

### Telegram para emergencias

Webhook a Telegram para "task failed" o "resource warning". Más visible que mirar la terminal cada rato.

### Avisos de contexto cerca del límite

Si tu sesión llega a 90% del context window, podés configurar una notificación que te avise antes de que rompa.

## Beneficios reales

- **Cero dependencias** — no instala stack extra, todo cross-platform.
- **Click-to-focus** — la notificación te lleva directo a Claude Code, no a buscar la ventana.
- **Webhooks** — escala a cualquier servicio (ntfy, Slack, Telegram, Discord, lo que tenga URL POST).
- **Análisis de contexto** — el plugin decide cuándo avisar, no te spammea.
- **Liviano** — 4 commands, no es un framework.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si solo hacés tareas cortas (menos de 2 min):** el ping llega cuando ya volviste a mirar. No aporta.
- **Si tu sistema operativo bloquea notificaciones por default:** primero arreglá los permisos a nivel SO.
- **Webhooks públicos:** un webhook de Slack con URL pública filtrado en internet puede ser un vector. Protegé las URLs.
- **Ruido:** si configurás todos los 6 tipos sin filtrar, te vas a saturar. Empezá con 2 (completed + needs input) y ajustá.

</Callout>
