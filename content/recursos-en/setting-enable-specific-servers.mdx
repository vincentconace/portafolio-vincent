## What it is

`enabledMcpjsonServers: ["memory", "github", "filesystem"]` — only the listed servers are activated. The rest defined in `.mcp.json` stay disabled.

Allowlist approach. More restrictive than [Enable All Project Servers](/boveda/setting-enable-all-project-servers) (which activates everything).

## How to install

```bash
npx claude-code-templates@latest --setting mcp/enable-specific-servers
```

Edit the list with your real MCPs:

```json
{
  "enabledMcpjsonServers": [
    "memory",
    "github",
    "filesystem",
    "context7"
  ]
}
```

For Claude:

```
I installed the mcp/enable-specific-servers setting. List the MCPs
from .mcp.json and give me the array of the ones worth enabling for my
use. The rest stay disabled.
```

## What you can do with this

### Granular enterprise control

Your org has an approved list of MCPs. This setting enforces that only those run.

### Small set vs broad catalog

`.mcp.json` may have 20 MCPs defined. If you only need 5, list them.

### Combo with [Disable Risky Servers](/boveda/setting-disable-risky-servers)

Enable Specific + Disable Risky = belt and suspenders. Either one blocks.

### For autonomous sessions

Long `/goal` runs: if Claude tries to use an unauthorized MCP, it won't work.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If your legitimate workflow varies:** an allowlist can be restrictive when you switch tasks.
- **For devs experimenting with MCPs:** an allowlist requires updating the setting every time.
- **If you don't use .mcp.json:** the setting is pointless.

</Callout>
