## Qué es

Claude Octopus es lo más cercano a un "switchboard" de LLMs dentro de Claude Code que vas a encontrar. El autor (nyldn) lo armó para no atarte a un solo modelo: integra 8 providers distintos y te deja decidir cuál usar para cada paso del workflow.

Los 8 providers que cubre:
- Claude (oficial)
- OpenAI Codex
- Google Gemini
- Perplexity
- OpenRouter (gateway a varios)
- GitHub Copilot
- Qwen (Alibaba)
- Ollama (local)

Encima, trae 47 commands y 50 skills, todos pensados en Double Diamond workflows (Discover → Define → Develop → Deliver). No es solo "pegale a otro modelo" — es una capa de orquestación con flujos completos: brainstorm, PRD, parallel agents, debate, sentinel, factory, etc.

Si tu uso de IA ya pasa por varias herramientas y querés unificarlas en un solo entorno, este es el plugin.

**Flujos Double Diamond:**
- `flow-discover`, `flow-define`, `flow-develop`, `flow-deliver`

## Qué podés hacer con esto

### Debate entre modelos sobre tu spec

`skill-debate` lanza varios providers en paralelo, cada uno con un rol distinto (proposer, challenger, judge) y devuelve un veredicto. Más robusto que pedirle a un solo modelo "criticá esto".

### Investigación con Perplexity + síntesis con Claude

`skill-deep-research` usa Perplexity para buscar info actualizada y después le pasa los hallazgos a Claude para sintetizar. Útil cuando Claude no tiene info reciente sobre algo.

### Code review cruzado

`skill-code-review` + `skill-staged-review` con dos modelos distintos = más cobertura. Si Codex y Claude están de acuerdo, alta confianza. Si discuten, mirás vos.

### Workflows Double Diamond completos

Discover → Define → Develop → Deliver con `flow-*` commands. Es framework de diseño aplicado a desarrollo: estructura el problema antes de saltar a soluciones.

### Costos controlados

`skill-cost-projections` te dice cuánto va a costar correr el workflow antes de lanzarlo. Crucial cuando estás moviendo trabajo entre 8 providers.

## Beneficios reales

- **El más completo en multi-LLM** del catálogo (8 providers).
- **No te ata a un modelo** — si mañana Claude se cae o sube precios, tenés alternativas activas.
- **47 commands + 50 skills** cubren prácticamente todo el ciclo dev.
- **Workflows Double Diamond** estructurados, no ad-hoc.
- **Cost projections** built-in — única forma seria de no quemar dinero con multi-LLM.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si solo usás Claude:** la mayor parte del valor de Octopus está en orquestar varios. Si tu setup es solo Claude, los plugins oficiales de Anthropic ya te cubren con menos overhead.
- **Costos:** correr 8 providers en paralelo puede multiplicar el gasto. Mirá `skill-cost-projections` antes de habilitar todo.
- **Setup inicial:** vas a necesitar API keys o configuración para cada provider que actives. No es plug-and-play total.
- **Sobre-curva de aprendizaje:** 50 skills es mucho. Empezá por los `flow-*` y un par de skills, no todo a la vez.

</Callout>
