## Qué es

Setting decorativo que reemplaza los spinner verbs (los textos animados que Claude Code muestra mientras procesa) con el clásico recordatorio "Don't Deploy On Fridays".

Es un guiño humorístico al folklore de la industria: nunca deployees a producción un viernes a la tarde porque si rompe algo, te arruinás el fin de semana.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting environment/friday-deploy-warning
```

Para Claude:

```
Instalé el setting environment/friday-deploy-warning. Hacé una tarea
y mostrame qué aparece en el spinner.
```

## Qué podés hacer con esto

### Cultura del equipo

Si tu equipo respeta la regla, este setting es un toque cómplice.

### Anti-impulso de viernes

A las 5pm del viernes alguien propone "rápido este deploy". El spinner te recuerda que mejor lunes.

### Para combo con [Vercel Auto Deploy](/boveda/hook-vercel-auto-deploy)

Si tenés auto-deploy activo, este reminder mantiene awareness.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si trabajás con timezone donde viernes no es "el día":** la cultura es US-centric. En otros países el patrón puede ser distinto.
- **Si tu CI/CD es serio:** un buen pipeline con feature flags + rollback automatic permite deploy cualquier día.
- **Para uso individual sin equipo:** menos divertido.
- **Solo 32 installs:** muy nicho.

</Callout>
