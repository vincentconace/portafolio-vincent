## What it is

A statusline focused on Neon (serverless Postgres). It shows:
- **Project name**
- **Active branch** (Neon supports git-like branching of DBs)
- **Compute state** (active, suspended, etc.)
- **Autoscaling CU range** (compute units)

It uses **only the Neon REST API** — it doesn't open constant SQL connections, so it's lightweight.

One of 3 Neon statuslines in the family: this one (Context), [Dev](/boveda/setting-neon-database-dev), [Resources](/boveda/setting-neon-database-resources).

## What you can do with it

### Neon branch awareness

If you're working on a Neon dev branch, the statusline reminds you — so you don't push migrations to main by mistake.

### Quick cost awareness

Compute state tells you whether you're paying for active or you're suspended. Handy for monitoring costs.

### Combo with [Neon Database Dev](/boveda/setting-neon-database-dev)

Context shows cluster info. Dev shows runtime info. They combine well (but only one at a time as a statusline).

### Combo with [Neon MCP](/boveda/mcp-neon)

Statusline for constant context. MCP for on-demand ops. A natural combo.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **If you don't use Neon:** doesn't apply.
- **If you already have another statusline:** you can only have one, so choose.
- **Only 26 installs:** small community. Validate the implementation.

</Callout>
