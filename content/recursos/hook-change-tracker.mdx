## Qué es

La versión simple del [Change Logger](/boveda/hook-change-logger): registra qué archivos se modificaron y cuándo. Sin CSV estructurado, sin details — solo el log básico.

1,127 installs lo hacen el hook más adoptado de la categoría `development-tools`. Razón: cubre el 80% de uso casual sin complejidad extra.

Hook event: `PostToolUse`.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook development-tools/change-tracker
```


## Qué podés hacer con esto

### Tracking básico

Saber qué archivos Claude tocó en la sesión actual. Útil cuando perdés el hilo.

### Combinación con [Conventional Commits](/boveda/hook-conventional-commits)

Si commiteás cada sesión, el log te ayuda a armar mensaje de commit completo recordando qué cambió.

### Para sesiones autónomas

Lanzás Claude con `/goal`, te vas. Al volver, el log te dice qué tocó.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si querés más estructura:** usá [Change Logger](/boveda/hook-change-logger) con CSV.
- **Si git ya te alcanza:** `git status` + `git log` tienen lo mismo y más.
- **Para audit serio:** este hook es informativo, no compliance-grade.

</Callout>
