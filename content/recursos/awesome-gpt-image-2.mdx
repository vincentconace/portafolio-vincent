## El workflow ganador — no copy-paste directo

El error típico: copiar prompt del repo → pegar en ChatGPT → output mediocre.

El truco: **personalizar con Claude antes** de pegar.

```
Prompt del repo →  Claude (lo personaliza a tu caso) →  ChatGPT (genera)
```

### Por qué Claude en el medio

El prompt del repo es genérico ("foto de producto, fondo blanco, iluminación de estudio"). Tu producto / marca tiene specs propias (paleta, ángulo preferido, mood). Claude **adapta** el prompt genérico a tu contexto antes de mandarlo a GPT.

### Ejemplo del workflow

#### Paso 1 — agarrás el prompt base del repo

Categoría: "Foto de producto Amazon-style"

```
Photo of {product} on white background, professional studio lighting,
soft shadow, sharp focus, photorealistic, commercial photography style
```

#### Paso 2 — Claude personaliza

```
[En Claude]
> Personalizame este prompt para mi producto: zapatillas running.
  
  Mi marca: minimalist, eco-friendly, audience runners 25-40.
  Producto: zapatilla unisex, paleta verde + crema.
  Necesito 3 variantes para A/B test.

Claude:
  Variante 1 (Amazon-style standard):
  "Photo of minimalist eco-friendly running shoe, green and cream colorway,
  on pure white background, professional studio lighting from upper-left,
  soft shadow under shoe, 45-degree angle, sharp focus, photorealistic,
  commercial photography for e-commerce, 4K"

  Variante 2 (editorial):
  "[same shoe] photographed on textured natural canvas background, golden
  hour lighting, slightly elevated angle showing sole detail, magazine
  editorial style, depth of field, premium feel"

  Variante 3 (action/lifestyle):
  "[same shoe] photographed on running trail at dawn, slight motion blur
  suggesting movement, soft natural lighting, environmental context,
  authentic lifestyle photography, Nike Sandtrap meets Allbirds style"
```

3 variantes con personalidad propia desde un prompt genérico.

#### Paso 3 — Pegás en ChatGPT

Cada variante se genera por separado en GPT Image 2. Comparás output. Iterás la que más se acerque.

## Anti-patrones

### 1. Copy-paste directo sin personalizar

El prompt del repo es **base**. Sin Claude personalizando, te queda imagen "que se parece a marca de stock photo".

### 2. Probar 1 prompt y rendirte

GPT Image 2 (como cualquier image gen) **requiere iteración**. El primer output de un prompt nuevo rara vez es el final. Iterá 3-5 veces.

### 3. No guardar lo que funciona

Cuando encontrás prompt que te da output perfecto, **guardalo**. Para uso recurrente, mantené tu propio `mis-prompts/` con los ganadores.

### 4. Ignorar la diferencia entre modelos

Un prompt que funciona en GPT Image 2 puede no funcionar igual en Midjourney o Flux. **El prompt es específico al modelo**. Para Claude / Anthropic: tienen tags distintas, mejor pedirle a Claude que adapte.

## El componente Claude que aporta valor

Lo más importante del workflow no es el repo (4400 prompts es mucho pero podés encontrarlos en otros lados). **Es Claude personalizándolos**.

Sin Claude:
- Aplicás el prompt genérico → output genérico

Con Claude:
- Le pasás contexto de tu marca, audiencia, restricciones
- Te genera 3-5 variantes pensando en tu caso específico
- Comparás y elegís la mejor base
- Iterás más fácil porque tenés un punto de partida fuerte

**El 80% del valor está en este step intermedio**, no en el repo ni en GPT.
