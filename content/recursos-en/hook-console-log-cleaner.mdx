## What it is

When Claude edits a file and you're on a production branch (`main`, `master`), the hook detects whether the change adds a `console.log` and warns you about it. Hook event: `PreToolUse`.

Typical case: Claude adds a `console.log` to debug something, you forget to remove it, and it ends up in prod. This hook prevents that.

## How to install it

```bash
npx claude-code-templates@latest --hook pre-tool/console-log-cleaner
```

For Claude:

```
I installed the pre-tool/console-log-cleaner hook. Try it: I'm on main.
Ask Claude to add a console.log for debugging — the hook
should warn.
```

## What you can do with it

### Basic prevention of debug code in prod

It alerts you when you're about to merge debug code.

### For teams with strict CI

Some teams have lint rules that block console.log in prod. This hook warns you before CI rejects it.

### For autonomous sessions

If Claude is running with `/goal` and modifies production branches, this hook adds a safety net.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If your project USES `console.log` legitimately:** some backends (CLIs, scripts) use console.log as normal output. False positive.
- **If your lint already covers this:** ESLint with `no-console` blocks it automatically. The hook is redundant.
- **If you don't use JS/TS:** it doesn't apply.
- **For Python (`print`):** this hook is JS-specific. For Python you'd need a different one.

</Callout>
