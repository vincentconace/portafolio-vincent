## El modelo

No necesitás dinero para empezar. Solo:

1. **Tiempo**
2. **Ganas de aprender**
3. **Herramientas correctas** — todas con tier gratis

El flujo:

```
1. Construís con tier gratis
2. Mostrás demo funcional al cliente
3. Cliente paga
4. Sus pagos cubren upgrades si necesitás escalar
```

**Cero inversión inicial**. Solo tu tiempo.

## El flujo paso a paso

### 1. Aprendé

Practicá con las herramientas hasta sentirte cómodo. Hacé 2-3 proyectos personales primero.

### 2. Construí un demo

Hacé proyecto real con el stack — algo **que puedas mostrar**. No "voy a hacer" — terminado y funcionando.

### 3. Presentá al cliente

Mostrá el demo. **Un demo real vale más que mil propuestas**.

### 4. Escalá con ingresos

Cuando el cliente paga, ese ingreso cubre upgrades si los necesitás. Vercel Pro, Supabase Pro, Stripe en producción real, etc.

## Un proyecto real de principio a fin

### Caso — landing para cliente local

```
1. Cliente: "necesito web para mi consultora"
2. Diseñás en Figma (gratis)
3. Construís con Claude Code + Next.js
4. BD en Supabase para formulario de contacto (gratis)
5. Auth con Clerk si necesita login (gratis)
6. Deploy en Vercel (gratis)
7. Dominio + DNS en Cloudflare (DNS gratis, dominio aparte)
8. Cobrás al cliente con Stripe
9. Con el pago cubrís upgrades si escalás
```

Tu costo en tools durante el proyecto: **$0**.

## Honesto sobre el modelo

### Funciona si

- Tenés tiempo para invertir en aprender (~3-6 meses)
- Estás dispuesto a hacer 2-3 proyectos gratis al principio para portfolio
- Sabés vender (o aprendés)
- Entendés que "gratis" significa **tier limitado** — al escalar pagás

### NO funciona si

- Buscás ingresos inmediatos esta semana
- No tenés paciencia para aprender
- Esperás clientes sin esfuerzo de outreach
- No querés tocar código (con AI igual hay que entender)

---

## Anti-patrones

### 1. Saltarte el "demo real"

Vender sin demo = vender humo. **Construí el demo primero** aunque sea genérico.

### 2. Cobrar por hora desde día 1

Cobrá **por proyecto** al principio. Por hora castiga tu velocidad con IA.

### 3. Pagar pro de todo antes de cliente

No upgradees nada hasta que **tengas un cliente que justifique** el upgrade.

### 4. Stack incompleto

Si elegís Supabase, **dominalo**. No saltes a Firebase a mitad de proyecto. Cada stack tiene su quirk.

### 5. Ofrecer "todo"

Al principio, **un servicio bien hecho** > 10 servicios mal hechos. Especializate primero.
