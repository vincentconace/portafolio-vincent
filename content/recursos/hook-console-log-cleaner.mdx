## Qué es

Cuando Claude edita un archivo y vos estás en una branch productiva (`main`, `master`), el hook detecta si el cambio agrega `console.log` y te lo avisa. Hook event: `PreToolUse`.

Caso típico: Claude agrega `console.log` para debuggear, vos te olvidás de sacarlo, termina en prod. Este hook lo previene.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook pre-tool/console-log-cleaner
```

Para Claude:

```
Instalé el hook pre-tool/console-log-cleaner. Probá: estoy en main.
Pedile a Claude que agregue un console.log para debug — el hook
debería warn.
```

## Qué podés hacer con esto

### Prevención básica de debug code en prod

Te alerta cuando estás a punto de mergeear debug code.

### Para teams con CI estricto

Algunos teams tienen lint rules que bloquean console.log en prod. Este hook te avisa antes de que CI te lo tire.

### Para sesiones autónomas

Si Claude está corriendo con `/goal` y modifica branches productivas, este hook agrega red de seguridad.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu proyecto USA `console.log` legítimamente:** algunos backends (CLIs, scripts) usan console.log como output normal. Falso positivo.
- **Si tu lint ya cubre esto:** ESLint con `no-console` bloquea automáticamente. El hook duplica.
- **Si no usás JS/TS:** no aplica.
- **Para Python (`print`):** este hook es JS-specific. Para Python necesitarías otro.

</Callout>
