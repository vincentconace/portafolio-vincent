## Qué es

Setting que configura un statusline custom para proyectos con asset pipelines pesados:

- **Texture processing** (compression, format conversion)
- **Model optimization** (3D meshes, LOD generation)
- **Audio compression** (ogg, mp3, opus)
- **Platform-specific variants** (PC vs mobile vs console)

Muestra queue de procesamiento, status actual y métricas.

Nicho: game dev, multimedia apps, AR/VR.

## Cómo se instala

```bash
npx claude-code-templates@latest --setting statusline/asset-pipeline-controller-statusline
```

Para Claude:

```
Instalé el setting statusline/asset-pipeline-controller-statusline.
Mostrame el comando que ejecuta para generar el statusline y
verificá que tengo Python3 disponible (el setting usa python).
```

## Qué podés hacer con esto

### Visibility de pipeline durante development

Mientras codeás, ves en statusline cuántas textures faltan procesar, qué modelo está optimizándose.

### Para projects con muchos assets

Game dev típicamente tiene cientos de textures + modelos. El statusline te da awareness sin abrir herramientas externas.

### Multi-platform tracking

Si compilás para PC + mobile + Switch, el statusline puede mostrar progress por target.

### Combo con [Context Timeline Hook](/boveda/hook-context-timeline)

Statusline en editor + Context Timeline en browser = doble visibility de lo que pasa.

### Comparado con [Claude HUD](/boveda/plugin-claude-hud)

Claude HUD es genérico (context, tools, todos). Asset Pipeline es nicho game dev. Si tu uso es game dev, este aporta más value.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no hacés game dev / multimedia:** este statusline no tiene caso.
- **Si tu pipeline está en otro lado** (CI/CD): el statusline puede no reflejarlo.
- **Solo 84 installs:** muy nicho. Validá implementación.
- **Para proyectos JS/TS web:** [Claude HUD](/boveda/plugin-claude-hud) es más útil.

</Callout>
