## Qué es

Statusline focused en Git Flow. Muestra:
- **Branch type icon** según prefix:
  - 🌿 feature/*
  - 🚀 release/*
  - 🔥 hotfix/*
- **Sync status** con remote
- **Change indicators** (uncommitted)

Es la versión visual de [Git Flow Settings](/boveda/setting-git-flow-settings) (el bundle all-in-one). Si solo querés el statusline sin permissions/hooks, este.


## Qué podés hacer con esto

### Identificación rápida de branch type

Sin leer el nombre completo, el icon te dice tipo de branch.

### Para teams con muchas branches activas

Si trabajás 5 features en paralelo, los icons ayudan a contexto rápido.

### Combo con [Git Flow Settings](/boveda/setting-git-flow-settings)

Si querés el bundle completo (incluye este statusline), usá [Git Flow Settings](/boveda/setting-git-flow-settings).

### Combo con [Validate Branch Name](/boveda/hook-validate-branch-name)

Hook enforce naming. Statusline muestra el icon resultante. Combo natural.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás Git Flow:** otros workflows no tienen los prefixes.
- **Si tu terminal no muestra emojis:** los icons aparecen como ?? raros.
- **Si tu equipo usa convention distinta:** no matchea.

</Callout>
