## Qué es

`forceLoginMethod: claudeai` — Claude Code solo permite login con cuentas Claude.ai (Pro, Team, Max plans). Console accounts (API billing) quedan rejected.

Útil para enterprises que centralizan en Claude.ai subscription, no en API consumption.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `authentication` |
| Installs | 86 |
| Setting key | `forceLoginMethod: "claudeai"` |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting authentication/force-claudeai-login
```

Para Claude:

```
Instalé el setting authentication/force-claudeai-login. Verificá que
estoy autenticado con account Claude.ai (Pro/Team) y no con API key
de Console.
```

## Qué podés hacer con esto

### Centralizar billing en Claude.ai plan

Tu Team plan paga uso. Console API te genera bill separado. Este setting enforce un solo path.

### Para teams con Claude.ai Team / Enterprise

Plan compartido = billing predictable. Setting evita que devs accidentalmente usen API.

### Compliance billing

Si tu org tiene contract con Anthropic via Claude.ai, este setting ensures todo va por ahí.

### Anti vs [Force Console Login](/boveda/setting-force-console-login)

Estos dos son opuestos. Solo instalá uno.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si usás API direct:** este setting te bloquea login.
- **Para devs con account personal Console:** override no posible — el setting es duro.
- **Si tu org permite ambas modalidades:** elegí flexibility.

</Callout>
