## Qué es

Por default, Claude Code opera sobre el directorio del proyecto actual. Si necesitás que también lea/escriba en otras carpetas (un monorepo con paths cross-package, una shared library en `~/code/shared/`, docs en `~/Notes/`), tenés que declarar esos paths.

Este setting configura el array `additionalDirectories` en tu config de Claude Code para autorizar paths extras.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `permissions` |
| Installs | 530 |
| Setting key | `additionalDirectories` |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting permissions/additional-directories
```

Después editá tu config con las paths reales:

```json
{
  "permissions": {
    "additionalDirectories": [
      "~/code/shared",
      "~/Notes/proyecto-x",
      "/Users/me/Documents/specs"
    ]
  }
}
```

Para Claude:

```
Instalé el setting permissions/additional-directories. Decime dónde
queda mi config y mostrame cómo agregar las paths reales.
```

## Qué podés hacer con esto

### Monorepos con cross-package work

Trabajás en `packages/api` pero necesitás ver `packages/shared`. Sin esto, Claude solo ve `api`.

### Shared libraries

Tu `~/code/shared/` viene en varios proyectos. Una sola config y siempre alcanzable.

### Docs / specs externas

Carpeta de notas en `~/Notes/` o `~/Documents/specs/`. Accesible para ground answers en docs reales.

### Combo con [Filesystem Access MCP](/boveda/mcp-filesystem-access)

Settings da permiso. Filesystem Access MCP da las operations granulares. Combo natural.

## Cuándo NO conviene

<Callout type="danger" title="Antes de agregar paths sensibles">

- **No autorices `~/` entero:** Claude tendría acceso a todo home incluyendo secrets.
- **Cuidado con `.ssh/`, `.aws/`, `.env`:** nunca agregues directorios con credentials.
- **Para writes:** asegurate que los paths permiten escritura solo donde lo necesitás.
- **Backup primero:** antes de habilitar write en paths con contenido valioso.

</Callout>
