## Qué es

`CLAUDE_CODE_ENABLE_TELEMETRY=1` — Claude Code manda usage analytics a Anthropic.

Para users que opt-in explícito. Anthropic usa esa data para identificar bugs, patterns de uso, y feature improvements.

Anti vs [Disable Telemetry](/boveda/setting-disable-telemetry).

## Cómo se instala

```bash
npx claude-code-templates@latest --setting telemetry/enable-telemetry
```

Para Claude:

```
Instalé el setting telemetry/enable-telemetry. Confirma que telemetry
está activo y mostrame qué tipo de data exactamente se envía (alta level).
```

## Qué podés hacer con esto

### Contribuir a improvements

Tu uso pattern ayuda a Anthropic a priorizar features.

### Explicitness sobre defaults

Si tu policy es "opt-in todo explicit", este setting documenta que telemetry está activo intencionalmente.

### Para Anthropic API users sin issues de privacy

Si tu workflow no tiene constraints de privacy, telemetry es positivo.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu workflow requires privacy:** usá [Disable Telemetry](/boveda/setting-disable-telemetry).
- **Si tu org bloquea data sharing:** este setting choca con policy.
- **Para data altamente sensible:** ni activar.

</Callout>
