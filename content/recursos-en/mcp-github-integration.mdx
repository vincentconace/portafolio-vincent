## What it is

If you work with GitHub (almost everyone does), this MCP is one of the ones you'll use most. It gives Claude Code access to the GitHub API: reading repos, creating issues, managing PRs, reviewing code, handling releases, searching.

With 2,867 installs it stands as **the most adopted MCP in the integration category** of the aitmpl catalog. That signals it's the de facto standard, not an experiment.

Compared to [mcp-github-official](/boveda/mcp-github-official) (the "official" version in the catalog), this is the one the community chose for consistency. There are minor differences; for most cases, either one works.

## How to install it

```bash
npx claude-code-templates@latest --mcp integration/github-integration
```

Prerequisites: GitHub account + Personal Access Token with the appropriate scopes (repo, issues, pull_request).

For Claude:

```
I installed the integration/github-integration MCP. After setting
GITHUB_TOKEN, list my 5 most active repos by commits in the last month
and the open PRs in each one.
```

## What you can do with this

### Issue triage

"List the 20 oldest issues with no activity in the last month across my repos. For each one, tell me whether it makes sense to close them or whether they need attention." Handy for periodic cleanup.

### Complete PR workflow

"Create a branch from main, add this change, commit with message X, open a PR with description Y, assign reviewers Z." The whole flow in one prompt.

### Assisted code review

"Go to PR #234, read the diff and give me a review per file: bugs, style, edge cases. If it's OK, leave an approval comment with a summary."

### Cross-repo search

"Find all uses of the `legacyAuth()` function across my repos and put together a migration plan." GitHub Search API through the MCP.

### Release management

"Cut a v1.2.0 release with a changelog from the last 30 commits, grouped by type (feat/fix/chore)."

### Sync with [Claude + Codex as a team](/boveda/claude-codex-equipo)

If the pattern is "Claude implements, Codex reviews," the GitHub MCP makes the review and the PR work in a single session without context switching.

## Prerequisites

- **Active GitHub account**
- **Personal Access Token** with the correct scopes (check the MCP's repo for the exact list)
- **Access to the repo / org** you're going to query
- **2FA** enabled recommended (not required for the MCP)

## When it's NOT a good idea

<Callout type="danger" title="Before enabling writes on important repos">

- **Tokens with broad scopes:** the token inherits whatever your user can do. If you have admin over 100 orgs, so does Claude. Restrict scopes to the minimum.
- **Destructive operations:** deleting branches, force-pushing, closing issues by mistake. ALWAYS use approval hooks on production repos.
- **Audit log:** everything Claude does is recorded under your user. Check the audit logs if you overdo it.
- **GitHub Enterprise Cloud or Server:** verify compatibility. Most of it works, some GHES features don't.
- **Rate limits:** the API has quotas. Bulk operations can block you temporarily.

</Callout>
