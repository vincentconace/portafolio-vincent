## What it is

This is the **official** version of the Sentry MCP in local mode. It runs via `npx` without having to set up any extra infrastructure. Compared to the [Sentry MCP (community)](/boveda/mcp-sentry) (270 installs), this one is more recent and officially backed by Sentry — only 21 installs so far.

The "local" distinction usually refers to:
- It runs as a local Node process
- It connects to your Sentry SaaS or self-hosted instance (configurable URL)
- It requires no additional setup compared to versions that need their own server

## How to install it

```bash
npx claude-code-templates@latest --mcp devtools/sentry-local
```

Prerequisite: a Sentry account + access token.

For Claude:

```
I installed the devtools/sentry-local MCP. Configure SENTRY_AUTH_TOKEN and
SENTRY_URL if self-hosted. List the most recent issues and top
errors in the project.
```

## What you can do with it

The same operations as with the [Sentry MCP community](/boveda/mcp-sentry):

- Issue triage
- Assisted debugging with stack traces
- Release analysis
- Correlation with deploys
- Performance triage

Detailed use cases are in [Sentry MCP](/boveda/mcp-sentry) — they apply just the same.

## Differences from the community version

- **Maintained directly by Sentry** — releases synced with the product
- **Fewer installs (21 vs 270)** — a smaller community trying it out
- **Official support** if you have an enterprise plan
- **Predictable updates**

## When to choose which

| Case | Recommendation |
| --- | --- |
| Official support matters | **Sentry Local (official)** |
| More community trying it | **Sentry (community)** |
| No preference | **Start with community for maturity** |
| Critical enterprise stack | **Official** |

## Prerequisites

- **Sentry account** + access token
- **Node.js** to run it via npx
- **Don't install it alongside the community one** (collision)

## When it's not a good fit

<Callout type="warning" title="Before you install">

- **If you already have the [`sentry`](/boveda/mcp-sentry) community one working:** don't force a migration. Test it in a sandbox.
- **21 installs is low:** fewer publicly known issues.
- **For self-hosted, on-prem Sentry:** check that the MCP supports your setup.

</Callout>
