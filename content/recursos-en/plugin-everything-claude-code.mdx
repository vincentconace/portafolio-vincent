## What it is

113k stars: it's the most-starred plugin in the aitmpl.com catalog. Above Claude Mem (42k), above Claude HUD (14k), above any marketplace. When a single piece pulls together that much backing, it's worth paying attention to.

The author's subtitle nails it: "The agent harness performance optimization system." It's not a marketplace or a complete workflow — it's a **system** that optimizes how the agent harness (the layer connecting the LLM with the tools and the environment) operates. It brings five pillars:

- **Skills** — specific capabilities
- **Instincts** — default behaviors that avoid typical mistakes
- **Memory** — smart persistence across sessions
- **Security** — guards and validations
- **Research-first development** — the pattern of investigating before implementing

It works in Claude Code, Codex, Opencode, Cursor "and beyond." The author (affaan-m) maintains a dedicated site at [ecc.tools](https://ecc.tools) with detailed docs.

## What you can do with this

### Get a Claude Code that "thinks before acting"

The **research-first** pillar is probably the most impactful: before it starts editing, Claude investigates the area of the code it's about to touch. Less "it implemented something that doesn't connect to the rest" and more "it understood the context, then acted."

### Persistent memory without fighting the setup

Memory across sessions, just like [Claude Mem](/boveda/plugin-claude-mem) but integrated with the other 4 pillars. If you're only going to install one, compare them first — both solve the same problem with different philosophies.

### Instincts that cover the typical mistakes

The concept of "instincts" is interesting: default behaviors Claude applies without asking, based on what the community has already identified as common mistakes. Things like "before deleting a file, check whether it's under version control," or "before pushing, run the tests."

### Real cross-tool support

If tomorrow you switch from Claude Code to Cursor or Codex, the 5 pillars come with you. For mixed teams this is valuable.

### Performance optimization

The author calls it a "performance optimization system" for a reason. The system cuts down useless iterations, avoids re-reading files, prioritizes relevant skills, and so on. Fewer tokens consumed for the same result.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If you're just getting started with Claude Code:** installing a "complete system" without understanding Claude Code first can bury the basic learning. Start with the official [Claude Code Setup](/boveda/claude-code-setup) and a couple of skills.
- **Overlap with `claude-mem`:** both cover memory. Don't install both right away — pick one.
- **Overlap with your own security hooks:** if you've already set up security hooks by hand, the "instincts" may duplicate rules.
- **It's not a toolkit, it's an opinionated system:** you'll inherit the author's decisions. If you disagree, you'll have to tune it.
- **Token cost of research-first:** investigating before acting spends more. Efficiency wins in the long run, but the first run is more expensive.

</Callout>
