## Qué es

Cada vez que Claude Code termina de responder (Stop event), este hook dispara una notificación nativa del sistema operativo (macOS, Linux, Windows).

A diferencia de [Discord Notifications](/boveda/hook-discord-notifications) (que va a un canal remoto) o [Claude Notifications Go](/boveda/plugin-claude-notifications-go) (que es plugin completo), este es la versión más simple: una notification del sistema, una por response.

Hook event: `Stop`.

## Cómo se instala

```bash
npx claude-code-templates@latest --hook monitoring/desktop-notification-on-stop
```

Para Claude:

```
Instalé el hook monitoring/desktop-notification-on-stop. Pedile a Claude
que haga algo que tarde 20 segundos. Cuando termine debería sonar el
notification de mi SO.
```

## Qué podés hacer con esto

### Para sesiones largas

Lanzás una tarea de 5 min, te vas a hacer otra cosa. Cuando vuelve el notification, ya está lista.

### Sin webhooks ni servicios externos

A diferencia de Discord/Slack notifications, este es solo SO. Sin internet, sin webhook URLs.

### Combo con focus mode

Si tenés focus mode activado en macOS, las notificaciones van a tu summary igual.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si tu workflow es interactivo continuo:** no aporta — siempre estás mirando.
- **Si tu SO bloquea notifications:** algunos setups las bloquean por default. Habilítalas primero.
- **Si necesitás notifications remotas** (avisarle a otra persona o que llegue al cel): mejor [Claude Notifications Go](/boveda/plugin-claude-notifications-go) o [Discord Notifications](/boveda/hook-discord-notifications).
- **Para tareas cortas:** notification llega después de vos haber visto el output.

</Callout>
