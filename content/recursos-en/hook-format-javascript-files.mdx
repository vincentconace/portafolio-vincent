## What it is

After Claude edits a `.js`, `.jsx`, `.ts`, or `.tsx` file, this hook runs prettier on it automatically. Hook event: `PostToolUse`.

Result: your code is always formatted according to your `.prettierrc`. No more noisy diffs from whitespace or trailing commas.

## How to install it

```bash
npx claude-code-templates@latest --hook post-tool/format-javascript-files
```

Prerequisite: `prettier` installed (`npm i -D prettier` or global).

For Claude:

```
I installed the post-tool/format-javascript-files hook. Try it: edit a
TS file with inconsistent formatting — it should come out formatted
after the edit.
```

## What you can do with it

### Automatic consistent formatting

Your `.prettierrc` is always applied. You don't have to ask Claude to respect formatting — the hook does it afterward.

### Clean diffs

Any changes Claude makes will come out formatted. Your diffs (`git diff --staged`) stay focused on real changes, not whitespace.

### Combo with [Lint On Save](/boveda/hook-lint-on-save)

Format first, lint after. The lint catches what formatting can't fix.

### Combo with [Auto Git Add](/boveda/hook-auto-git-add)

Auto Git Add stages the changes. The formatting from this hook also affects the stage. Combined, your workflow becomes: edit → format → automatic stage.

## When it's NOT a good fit

<Callout type="warning" title="Before installing">

- **Without prettier installed:** the hook fails silently.
- **Without a .prettierrc in the repo:** prettier uses defaults that may not match your team.
- **If your team doesn't use prettier:** get everyone aligned before imposing it.
- **For non-JS/TS files:** this hook is specific. For Python use [Format Python Files](/boveda/hook-format-python-files).

</Callout>
