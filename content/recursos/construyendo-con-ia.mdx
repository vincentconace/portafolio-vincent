## Para quién es

No necesitás experiencia técnica previa. Solo:
- Laptop
- Ganas de experimentar
- Claridad para describir lo que querés crear

**No buscamos convertirte en ingeniero en una tarde**. Buscamos que construyas algo real y **lo pongas en internet HOY**.

---

## Qué vas a obtener hoy

- ✅ Una app funcional en internet
- ✅ Link público listo para compartir
- ✅ Un sistema **repetible** para crear más productos

---

## Filosofía

> **Aprender construyendo, no acumulando teoría.**
>
> **Terminar algo hoy** > planear perfecto una semana.

Avanzá rápido, publicá rápido, iterá con feedback real.

---

## Lo único que necesitás saber

| Skill | Nivel |
|-------|-------|

## Setup en una línea

```bash
# Mac / Linux
curl -fsSL https://raw.githubusercontent.com/construyeconia/setup/main/install.sh | bash

# Windows PowerShell
irm https://raw.githubusercontent.com/construyeconia/setup/main/install.ps1 | iex
```

Esto instala:
- Codex CLI
- Dependencias base (git, node, npm)
- Proyecto inicial
- Servidor local
- Túnel cloudflared para URL pública

5-10 min después estás listo para construir.

## El flow — 4 fases

### Fase 1 — Define tu idea (15 min)

Antes de tocar Codex, respondé:
- **Qué hace tu app** (1 línea)
- **Para quién es** (audiencia)
- **Qué es lo MÁS importante** que tiene que funcionar
- **Cómo se ve** (referencia visual si tenés)

Si te cuesta responder, usá [VibeCoding 101](/boveda/vivecoding-101) prompt para PRD.

### Fase 2 — Construir MVP (1-3h)

```
> Quiero construir [DESCRIBÍ TU APP].

  Brief:
  - Audiencia: [...]
  - Features MUST: [3 max]
  - Estilo visual: [referencia]
  - Stack: cualquiera que funcione bien

  Construime MVP funcionando localmente.
  Mostrame cuando esté listo para probar.
```

Codex genera código, te muestra preview, iterás.

### Fase 3 — Iterar (30-60 min)

Probás. Algo no convence → decís específico:

```
> El botón Hero es muy chico, hacelo prominente.
> El color del background es muy oscuro, hacelo más claro.
> El form de email no manda confirmación.
```

Codex ajusta.

### Fase 4 — Publicar (15-30 min)

```
> Está listo. Publicalo:
  1. Crear repo en mi GitHub
  2. Deploy a Vercel
  3. Configurar dominio
  4. Dame URL pública final
```

URL en mano.

## Casos de uso

### 1. Landing para validar idea

```
Idea: Servicio de coaching para [TARGET]
MVP: Landing + form de contacto + calendar embed
Tiempo: 2h total
Resultado: URL para compartir y validar interés
```

### 2. Tool simple

```
Idea: Calculadora de pricing para freelancers
MVP: Form interactivo + cálculo + share button
Tiempo: 3h
Resultado: Tool que podés monetizar después
```

### 3. Portfolio personal

```
Idea: Portfolio con casos
MVP: 5 proyectos + sobre mí + contacto
Tiempo: 1-2h
Resultado: URL para LinkedIn / aplicaciones
```

### 4. App de utility chica

```
Idea: Generador de [X] que automatiza tarea que hacés a mano
MVP: Input + procesamiento + output
Tiempo: 2-4h
Resultado: Tool propia, podés side-monetizarla
```

## Anti-patrones

### 1. Querer Spotify en v1

Tu primera app **no necesita features de productos consolidados**. 3 features bien hechas > 15 mediocres.

### 2. Pulir feo

Si pulís 1 semana, perdés momentum. **Mostralo feo y mejoralo después**.

### 3. No publicar "porque no está perfecto"

Perfecto = nunca lo publicás. **Publicá hoy. Iterá esta semana**.

### 4. Cambiar stack a mitad

Empezaste Codex con Next.js. **No cambies** a SvelteKit a mitad. Quedate.

### 5. Sin feedback real

Mostrale a 5-10 personas la URL. Su feedback **vale más que tu opinión**.
