## Qué es

Antes de cada tool use (Bash, Edit, Write, Read, etc.), este hook agrega una entrada al log: timestamp + tool name + args básicos. Hook event: `PreToolUse`.

Útil para entender el comportamiento de Claude: ¿qué herramientas usa? ¿en qué orden? ¿hay patterns repetitivos que podrías automatizar?

npx claude-code-templates@latest --hook development-tools/command-logger

## Qué podés hacer con esto

### Audit de tools

Saber exactamente qué herramientas usó Claude durante la sesión. Para compliance o para entender comportamiento.

### Debugging cuando algo sale mal

Si Claude tuvo un comportamiento raro, mirás el log y reconstruís qué hizo paso a paso.

### Detectar patterns para optimizar

Si ves que Claude usa siempre la misma secuencia Read → Edit → Bash → Read, podés armar una skill que lo haga en un solo paso.

### Combo con [Change Tracker](/boveda/hook-change-tracker)

Command Logger registra qué tools se usaron (incluyendo lecturas). Change Tracker registra solo las mutaciones. Combinalos para audit completo.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Para sesiones muy intensas:** muchos tool calls = log grande. Rotación de logs recomendada.
- **Si el contenido de tool args es sensible:** algunos args pueden incluir paths o snippets sensibles. Validá qué se registra.
- **Si ya usás [Context Timeline](/boveda/hook-context-timeline):** hay solapamiento (Timeline también muestra tool usage).

</Callout>
