## What it is

A setting that points the statusline to a custom shell script (`~/.claude/statusline.sh` by default). The script receives the session context via JSON stdin and returns the string to display.

For anyone who wants full control: you build your own statusline with whatever info is relevant to you.

The JSON stdin includes: model name, current directory, git info, etc.

## How to install it

```bash
npx claude-code-templates@latest --setting statusline/command-statusline
```

Then edit `~/.claude/statusline.sh` with your logic.

## What you can do with this

### Your own statusline

You combine exactly the info you want. No gimmicks, no one else's decisions.

### For teams with specific preferences

Your org may have specific conventions (ticket number in the branch → show it in the statusline). A custom script makes that possible.

### Mix with your tools

If you have health checks for your local app, you can drop them into the statusline.

### A solid starting point

Other statuslines are shell scripts too. You can take [Colorful Statusline](/boveda/setting-colorful-statusline) as a base and modify it.

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **For those not comfortable with shell:** maintaining the script can be friction.
- **If the existing statuslines are enough for you:** [Context Monitor](/boveda/setting-context-monitor) covers 80% of cases.
- **If your script is slow:** the statusline renders frequently. Slow = laggy experience.
- **Without testing:** a bug in the script can silently break the statusline.

</Callout>
