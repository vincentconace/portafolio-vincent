## What it is

After each use of the Edit tool, the hook writes an entry to the project's `.claude/edit-log.txt` with a timestamp + the path of the modified file. Hook event: `PostToolUse`.

The log is **project-local** (not global) — it lives in `.claude/` inside the project. If you work across multiple projects, each one has its own trail.

```

```

## What you can do with it

### Cross-session tracking per project

Each project has its own `edit-log.txt`. Useful when you rotate between several and want to see recent activity without opening git log.

### Version the audit (or not)

You can decide whether to commit the log (so the team can see it) or ignore it in `.gitignore`.

### Compared with [Change Tracker](/boveda/hook-change-tracker)

Change Tracker is more popular (1,127 installs). Both do simple tracking. Either one works — pick based on your path/format preference.
