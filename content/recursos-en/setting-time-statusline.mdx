## What it is

Simple statusline with a clock. It shows:
- **Model** name
- Project **directory**
- **Current time**

Useful for long sessions where you lose track of time, or for tracking duration.

## How to install it

```bash
npx claude-code-templates@latest --setting statusline/time-statusline
```

For Claude:

```
I installed the statusline/time-statusline setting. Show me the current
time so it's visible.
```

## What you can do with it

### Anti time-sink-from-deep-work

If you get into flow and hours slip by, the clock helps.

### Session duration tracking

Combined with the start time, you can compute how long you've been working.

### For scheduled meetings

It reminds you to check the time before a meeting.

### Combo with [Deadline Countdown](/boveda/setting-deadline-countdown)

Time shows the present moment. Deadline shows how long until the cutoff. They're alternatives (one statusline).

## When it's NOT worth it

<Callout type="warning" title="Before installing">

- **If your OS already shows a clock:** mac/Linux/Windows have a system clock. Redundant.
- **If you want more info:** [Context Monitor](/boveda/setting-context-monitor) has a clock + more.
- **For deep focus:** seeing the time can break flow.

</Callout>
