## Qué es

Hooks SubagentStart + SubagentStop que loguean cada subagent invocation con timestamp. Cada vez que Claude spawnea o finaliza un subagent, queda JSON entry.

Para multi-agent workflows donde necesitás audit:
- Qué subagents corrieron
- Cuándo arrancaron / terminaron
- Cuánto tardaron
- Para qué propósito

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `hooks` |
| Installs | 7 |
| Hook events | `SubagentStart` + `SubagentStop` |
| Output | JSON entries timestamped |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting hooks/subagent-lifecycle-logger
```

Para Claude:

```
Instalé el setting hooks/subagent-lifecycle-logger. Lanzá una tarea
que use subagents y mostrame el log JSON generated.
```

## Qué podés hacer con esto

### Audit de multi-agent flows

Si tu workflow lanza 8 subagents para un goal, el log te muestra qué hizo cada uno.

### Performance analysis

Identificar subagents lentos analyzing los timestamps.

### Compliance trail

Para orgs que requieren documentación de AI agent invocations.

### Combo con [Context Timeline Hook](/boveda/hook-context-timeline)

Subagent Logger para JSON estructurado. Context Timeline para vista visual.

### Para sesiones autónomas

`/goal` largos que lanzan agents: el log es esencial post-mortem.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si no usás subagents:** el log queda vacío.
- **Solo 7 installs:** muy nicho.
- **Si querés log de ALL tool calls:** [Command Logger Hook](/boveda/hook-command-logger) es más amplio.

</Callout>
