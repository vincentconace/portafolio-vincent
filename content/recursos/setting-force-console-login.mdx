## Qué es

`forceLoginMethod: console` — Claude Code solo permite login con cuentas Anthropic Console (API billing). Claude.ai accounts (Pro/Team) quedan rejected.

Anti vs [Force Claude.ai Login](/boveda/setting-force-claudeai-login). Si tu org centraliza en API billing en vez de Claude.ai plans.

## Tabla resumen

| Campo | Valor |
| --- | --- |
| Categoría aitmpl | `authentication` |
| Installs | 40 |
| Setting key | `forceLoginMethod: "console"` |

## Cómo se instala

```bash
npx claude-code-templates@latest --setting authentication/force-console-login
```

Para Claude:

```
Instalé el setting authentication/force-console-login. Confirmá que
autenticado con API key de Console y no con account Claude.ai.
```

## Qué podés hacer con esto

### Centralizar en API billing

Tu org tiene Anthropic API contract con billing centralized. Este setting enforce.

### Para developers con API access dedicated

Si cada dev tiene API key del org en vez de personal Claude.ai, este setting evita confusion.

### Para integración con tooling existente

Si tu org usa API direct para más cosas (apps internas, etc.), centralizar en Console hace sense.

### Anti vs [Force Claude.ai Login](/boveda/setting-force-claudeai-login)

Estos dos son opposites. Solo instalá uno.

## Cuándo NO conviene

<Callout type="warning" title="Antes de instalar">

- **Si usás Claude.ai Pro/Team:** este setting te bloquea.
- **Solo 40 installs:** menor adoption. La mayoría prefiere Claude.ai.
- **Para devs con account personal Pro:** override no posible.

</Callout>
